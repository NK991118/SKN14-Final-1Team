Cloud Firestore는 사용하는 클라이언트 라이브러리에 따라 서로 다른 두 가지 메서드를 통해 강력한 액세스 관리 및 인증을 제공합니다.

모바일 및 웹 클라이언트 라이브러리의 경우 Firebase Authentication 및 Cloud Firestore Security Rules을 사용하여 서버리스 인증, 승인, 데이터 유효성 검사를 처리합니다. Cloud Firestore Security Rules [https://firebase.google.com/docs/firestore/security/get-started?hl=ko]으로 Android, Apple, 웹 클라이언트 라이브러리의 데이터를 보호하는 방법을 알아보세요.

앱 체크 [https://firebase.google.com/docs/app-check?hl=ko]를 사용하면 사용자의 앱에서만 Cloud Firestore 데이터에 액세스할 수 있습니다.

Cloud Storage for Firebase를 사용하는 앱의 경우 Cloud Firestore를 사용하여 Cloud Storage Security Rules으로 액세스 [https://firebase.google.com/docs/storage/security/rules-conditions?hl=ko#enhance_with_firestore]할 수 있는 데이터베이스 문서에서 Cloud Storage 리소스에 대한 액세스 조건을 정의합니다.

서버 클라이언트 라이브러리의 경우 Identity and Access Management(IAM)를 사용하여 데이터베이스 액세스를 관리합니다. IAM [https://cloud.google.com/firestore/docs/security/iam?hl=ko]으로 자바, Python, Node.js, Go 클라이언트 라이브러리의 데이터에 보안을 적용하는 방법을 알아보세요.
