소비자 경험
소비자 SDK를 통해 소비자가 차량 공유, 음식 배달, 택배 배송 또는 서비스 방문 현황을 모니터링할 수 있도록 주문 위치와 진행 상황을 소비자에게 공유합니다.
주문형 이동을 위한 소비자 SDK
실시간에 가까운 위치 업데이트와 주문형 이동의 정확한 도로 위치를 활용하여 소비자 모바일 앱을 강화하세요.

예약된 작업을 위한 소비자 SDK
소비자 웹 앱을 개선하여 배송 소비자가 배송 기사가 공유하는 여정과 택배 또는 예정된 서비스 상태를 확인할 수 있도록 하세요.
JavaScript [https://developers.google.com/maps/documentation/mobility/journey-sharing/scheduled/shipment-tracking/setup?hl=ko]
소비자 SDK를 통합하는 방법
이 단계에서는 Fleet Engine에서 작동하도록 소비자 SDK를 소비자 앱에 통합하기 위한 대략적인 워크플로를 설명합니다.
airport_shuttle [https://developers.google.com/maps/documentation/mobility/journey-sharing/on-demand?hl=ko]
주문형 이동에 대한 소비자 경험 이해 [https://developers.google.com/maps/documentation/mobility/journey-sharing/on-demand?hl=ko]
주문형 이동 데이터를 모델링하고 공유하고, Fleet Engine에서 이동 데이터에 액세스하여 소비자가 이동 경로를 확인할 수 있도록 합니다.
local_shipping [https://developers.google.com/maps/documentation/mobility/journey-sharing/scheduled?hl=ko]
예약된 작업에 대한 소비자 환경 이해 [https://developers.google.com/maps/documentation/mobility/journey-sharing/scheduled?hl=ko]
예약된 작업 데이터를 모델링하고 공유하고, Fleet Engine의 이동 데이터에 액세스하여 소비자가 작업 진행 상황을 추적할 수 있도록 합니다.
directions [https://developers.google.com/maps/documentation/mobility/journey-sharing/on-demand/android/setup?hl=ko]
소비자 SDK 설정 [https://developers.google.com/maps/documentation/mobility/journey-sharing/on-demand/android/setup?hl=ko]
Consumer SDK는 Fleet Engine에서 실시간 위치 업데이트를 가져오므로 이동성 솔루션에서 여정 및 작업 진행 상황을 공유할 수 있습니다.

Fleet Engine을 통한 여정 관리 [https://developers.google.com/maps/documentation/mobility/fleet-engine?hl=ko]
Fleet Engine은 Driver SDK와 자체 백엔드 서비스 간의 상호작용을 처리합니다. 백엔드 서비스는 REST 또는 gRPC를 호출하여 Fleet Engine과 통신할 수 있습니다.
