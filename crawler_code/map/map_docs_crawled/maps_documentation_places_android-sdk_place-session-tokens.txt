세션 토큰
세션 토큰은 Autocomplete (New) [https://developers.google.com/maps/documentation/places/android-sdk/place-autocomplete?hl=ko] 호출을 세션으로 추적하는 사용자 생성 문자열 (또는 객체)입니다. 자동 완성 (신규)은 세션 토큰을 사용하여 사용자 자동 완성 검색의 쿼리 및 선택 단계를 결제 목적의 개별 세션으로 그룹화합니다.
사용자는 각 세션의 세션 토큰을 만들어야 합니다. 세션 토큰은 URL 및 파일 이름 안전 base64 문자열이어야 합니다. 사용자는 어떤 방식으로든 세션 토큰을 생성할 수 있지만 Google에서는 세션 토큰에 버전 4 범용 고유 식별자 (UUID) [https://tools.ietf.org/html/rfc4122]를 사용하는 것이 좋습니다.
세션은 Autocomplete (New) 호출로 시작하고 Place Details (New) [https://developers.google.com/maps/documentation/places/android-sdk/details-place?hl=ko] 또는 Address Validation [https://developers.google.com/maps/documentation/address-validation/overview?hl=ko] 호출로 종료됩니다. 각 세션에는 여러 개의 Autocomplete (New) 쿼리가 포함될 수 있으며, 그 뒤에 Place Details (New) 또는 Address Validation에 대한 요청이 하나 포함됩니다. 세션 내 각 요청에 사용되는 API 키는 동일한 Google Cloud 콘솔 프로젝트에 속해야 합니다.
참고: 세션 토큰은 Places API (기존) 엔드포인트와 Places API (신규) 또는 Address Validation 엔드포인트 간에 교환할 수 없습니다. 예를 들어 Place Autocomplete (레거시) [https://developers.google.com/maps/documentation/places/android-sdk/autocomplete?hl=ko] 요청은 동일한 세션에서 Place Details (레거시) 요청으로 이어질 수 있지만 동일한 세션에서 Place Details (신규) 요청이나 Address Validation 요청으로 이어질 수는 없습니다.
세션 토큰은 다음과 같은 방식으로 종료됩니다.
Place Details (New)가 호출됩니다.
주소 유효성 검사가 호출됩니다.
세션이 중단됩니다.
참고: 요청에서 Autocomplete (New) 세션 토큰을 생략할 수 있습니다. 세션 토큰이 생략되면 각 요청에 대해 별도로 요금이 청구되며 SKU: Autocomplete Requests [https://developers.google.com/maps/billing-and-pricing/sku-details?hl=ko#places_autocomplete-request-new-ess-sku]에 따라 요금이 청구됩니다.
세션이 종료되면 토큰이 더 이상 유효하지 않습니다. 앱에서 각 새 세션에 대해 새 토큰을 생성해야 합니다. sessionToken 매개변수가 생략되거나 세션 토큰을 재사용하는 경우 새 세션에 대해 세션 토큰이 제공되지 않은 것처럼 요금이 청구됩니다 (각 요청에 대해 별도로 요금이 청구됨).
Autocomplete (New) 요청에 대한 요금이 청구되는 방식에 대한 자세한 내용은 Autocomplete (New) 및 세션 가격 책정 [https://developers.google.com/maps/documentation/places/android-sdk/session-pricing?hl=ko]을 참고하세요.
