역 지오코딩

  유럽 경제 지역 (EEA) 개발자
역 지오코딩은 위치의 지리 좌표를 사람이 읽을 수 있는 상세 주소로 변환하는 것을 말합니다. iOS용 Maps SDK에는 GMSGeocoder [https://developers.google.com/maps/documentation/ios-sdk/reference/objc/Classes/GMSGeocoder?hl=ko] 클래스가 포함되어 있습니다. 이 클래스에는 변환을 실행하는 데 사용할 수 있는 reverseGeocodeCoordinate [https://developers.google.com/maps/documentation/ios-sdk/reference/objc/Classes/GMSGeocoder?hl=ko#-reversegeocodecoordinate:completionhandler:] 멤버 함수가 포함되어 있습니다. 이 메서드는 CLLocationCoordinate2D [https://developer.apple.com/documentation/corelocation/cllocationcoordinate2d?language=objc] 객체의 인스턴스에서 위치 좌표를 가져와 GMSAddress [https://developers.google.com/maps/documentation/ios-sdk/reference/objc/Classes/GMSAddress?hl=ko] 클래스의 인스턴스에서 사람이 읽을 수 있는 도로 주소를 반환합니다.
참고: iOS용 Maps SDK는 지오코딩을 직접 지원하지 않습니다. 애플리케이션에서 Geocoding API [https://developers.google.com/maps/documentation/geocoding?hl=ko]를 직접 사용하여 사람이 읽을 수 있는 주소를 지도 좌표로 변환하는 것이 좋습니다.
언어 환경설정의 영향
지오코더는 사람이 읽을 수 있는 상세 주소를 제공하도록 최적화되어 있습니다. 이를 위해 현지 언어로 된 거리 주소를 반환하며, 필요한 경우 사용자가 읽을 수 있는 텍스트로 음역합니다. 다른 모든 주소는 기본 언어로 반환됩니다.
주소 구성요소는 첫 번째 구성요소에서 선택한 동일한 언어로 반환됩니다.
선호하는 언어로 이름을 사용할 수 없는 경우 지오코더는 가장 일치하는 이름을 사용합니다.
주소 구성요소 관련 보장
Google은 주소 구성요소를 보장하지 않습니다. 주소 구조는 국가마다, 심지어 국가 내에서도 다릅니다.
주소 구성요소에는 우편 주소와 관련된 내용만 포함되며 그 외의 내용은 거의 포함되지 않습니다.
특히 locality는 항상 표시되지 않을 수 있으며 항상 city를 나타내지는 않습니다.
주소 구성요소의 예는 Place Autocomplete 주소 양식 [https://developers.google.com/maps/documentation/javascript/examples/places-autocomplete-addressform?hl=ko]을 참고하세요.
결과 정렬
결과는 거리순으로 정렬되지 않으며 순서가 변경될 수 있습니다.
특정 순서는 보장되지 않습니다.
첫 번째 결과가 무엇인지 보장할 수 없습니다.
역 지오코딩은 추정치입니다.
지오코더는 특정 오차 범위 내에서 주소를 가진 가장 가까운 위치를 찾으려고 시도합니다.
지오코더가 일치하는 항목을 찾을 수 없으면 결과가 반환되지 않습니다.
자세한 내용은 주소 지오코딩 권장사항 [https://developers.google.com/maps/documentation/geocoding/best-practices?hl=ko] 및 지오코딩 FAQ [https://developers.google.com/maps/documentation/geocoding/faq?hl=ko]를 참고하세요.
