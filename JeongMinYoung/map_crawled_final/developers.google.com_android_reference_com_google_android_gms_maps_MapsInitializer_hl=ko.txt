public final class MapsInitializer extends Object [https://developer.android.com/reference/java/lang/Object.html]

Use this class to initialize the Google Maps SDK for Android if features need to be used
before obtaining a map. It must be called because some classes such as
BitmapDescriptorFactory and CameraUpdateFactory need to be initialized.
Do not use this class if you obtain a valid GoogleMap [https://developers.google.com/android/reference/com/google/android/gms/maps/GoogleMap]
reference using the onMapReady(GoogleMap map) [https://developers.google.com/android/reference/com/google/android/gms/maps/OnMapReadyCallback#onMapReady(com.google.android.gms.maps.GoogleMap)] callback. Instead provide the callback to either
MapFragment#getMapAsync() [https://developers.google.com/android/reference/com/google/android/gms/maps/MapFragment#getMapAsync(com.google.android.gms.maps.OnMapReadyCallback)] or MapView#getMapAsync() [https://developers.google.com/android/reference/com/google/android/gms/maps/MapView#getMapAsync(com.google.android.gms.maps.OnMapReadyCallback)]. For examples, see the sample application.

public static synchronized int
initialize (Context [https://developer.android.com/reference/android/content/Context.html] context,
MapsInitializer.Renderer [https://developers.google.com/android/reference/com/google/android/gms/maps/MapsInitializer.Renderer]
preferredRenderer, OnMapsSdkInitializedCallback [https://developers.google.com/android/reference/com/google/android/gms/maps/OnMapsSdkInitializedCallback]
callback)

Specifies which MapsInitializer.Renderer [https://developers.google.com/android/reference/com/google/android/gms/maps/MapsInitializer.Renderer]
type you prefer to use to initialize the Google Maps SDK for Android, and provides a
callback to receive the actual MapsInitializer.Renderer [https://developers.google.com/android/reference/com/google/android/gms/maps/MapsInitializer.Renderer]
type. This call will initialize the Google Maps SDK for Android, so that its classes
are ready for use. The callback will be triggered when the Maps SDK is initialized.
The Maps SDK only initializes once per Application lifecycle. Only the first call of
this method or initialize(Context) [https://developers.google.com/android/reference/com/google/android/gms/maps/MapsInitializer#initialize(android.content.Context)] takes effect. To make renderer preference meaningful,
you must call this method before initialize(Context) [https://developers.google.com/android/reference/com/google/android/gms/maps/MapsInitializer#initialize(android.content.Context)], and before MapFragment.onCreate(Bundle) [https://developers.google.com/android/reference/com/google/android/gms/maps/MapFragment#onCreate(android.os.Bundle)] and MapView.onCreate(Bundle) [https://developers.google.com/android/reference/com/google/android/gms/maps/MapView#onCreate(android.os.Bundle)]. It's recommended to do this in Application.onCreate() [https://developer.android.com/reference/android/app/Application.html#onCreate()].
Do not use this method if you obtain a valid GoogleMap [https://developers.google.com/android/reference/com/google/android/gms/maps/GoogleMap]
reference using the onMapReady(GoogleMap map) [https://developers.google.com/android/reference/com/google/android/gms/maps/OnMapReadyCallback#onMapReady(com.google.android.gms.maps.GoogleMap)] callback. Instead provide the callback to either
MapFragment#getMapAsync() [https://developers.google.com/android/reference/com/google/android/gms/maps/MapFragment#getMapAsync(com.google.android.gms.maps.OnMapReadyCallback)] or MapView#getMapAsync() [https://developers.google.com/android/reference/com/google/android/gms/maps/MapView#getMapAsync(com.google.android.gms.maps.OnMapReadyCallback)]. This call will then have no effect other than
triggering the callback for the initialized MapsInitializer.Renderer [https://developers.google.com/android/reference/com/google/android/gms/maps/MapsInitializer.Renderer].
Note the following:

Use LEGACY to request the legacy renderer. No action is necessary if
you prefer to use the latest renderer, as the latest render is the default.
The latest renderer may not always be returned due to various reasons, including
not enough memory, unsupported Android version, or routine downtime.

Parameters
| context | Required to fetch the necessary SDK resources and code. Must not be null. |
| --- | --- |
| preferredRenderer | Which MapsInitializer.Renderer type you prefer to use for your application. If null is provided, the default preference is taken. |
| callback | The callback that the Maps SDK triggers when it informs you about which renderer type was actually loaded. You can define what you want to do differently according to the maps renderer that is loaded. |

Returns

A ConnectionResult error code.
