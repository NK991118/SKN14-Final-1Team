Title: Fleet Engine 서비스란 무엇인가요?

Fleet Engine 서비스란 무엇인가요?

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

Fleet Engine은 고객의 운송 서비스를 관리하는 데 사용하는 시스템과 통합되는 백엔드 서비스입니다. 다양한 API 및 SDK와 상호 운용하여 매핑, 라우팅, 위치 관리를 개선합니다.

Fleet Engine은 실제 차량을 기반으로 모델링된 차량 엔티티의 라우팅 및 상태 업데이트를 제공합니다. 시스템이 차량 진행 상황 업데이트를 Fleet Engine에 전송하면 Fleet Engine은 Driver SDK에서 제공하는 실시간 차량 위치 및 경로 정보를 사용하여 차량 여정(예상 도착 시간, 교통 정보, 차량 가용성)에 관한 정확한 보고서를 제공합니다.

Fleet Engine 서비스는 다음 두 가지 모빌리티 서비스 제품과 함께 작동합니다.

주문형 여정: 이 제품에서 Fleet Engine은 출발지에서 정류장을 거쳐 하차 지점까지의 여정의 수명 주기를 모델링합니다. 여정 수명 주기에서 차량은 소비자가 예약한 여정을 완료하기 위한 수단으로 픽업 위치, 중간 경유지, 하차 위치를 통과합니다. 주문형 이동에 관한 자세한 내용은 주문형 이동 [https://developers.google.com/maps/documentation/mobility/fleet-engine/journeys/trips?hl=ko] 문서를 참고하세요.
예약된 작업: 이 제품에서 Fleet Engine은 배송 또는 서비스 작업의 수명 주기를 모델링합니다. 여정의 각 정류장에서 운전자는 하나 이상의 예약된 작업을 완료해야 합니다. 작업에는 패키지 배송 또는 서비스 호출 완료가 포함될 수 있습니다. 예약된 작업에 대한 자세한 내용은 예약된 작업 [https://developers.google.com/maps/documentation/mobility/fleet-engine/journeys/tasks?hl=ko] 문서를 참고하세요.

Fleet Engine으로 할 수 있는 작업

Fleet Engine 구성요소를 사용하여 다음 작업을 할 수 있습니다.

소비자 여정 추적 사용 설정: 주문형 여정의 경우 Fleet Engine의 소비자 환경 기능을 사용하고 소비자 SDK를 모바일 또는 웹 앱에 통합할 수 있습니다. 이렇게 하면 소비자가 동일한 앱을 사용하여 이동 요청을 하고 이동 경로를 추적할 수 있습니다.
고객에게 실시간 패키지 정보 제공: 예약된 작업의 경우 Fleet Engine의 소비자 환경 기능을 사용하고 JavaScript Consumer SDK를 웹 앱에 통합할 수 있습니다. 이렇게 하면 소비자에게 패키지 상태에 관한 거의 실시간 업데이트를 제공할 수 있습니다. SDK는 소비자가 주문과 관련된 정보만 볼 수 있도록 내장 필터와 데이터 개인 정보를 제공합니다.
Fleet 운영자 솔루션 설계: 예약된 작업의 경우 JavaScript Fleet 추적 라이브러리를 사용하여 Fleet 운영자가 하루 종일 차량 할당을 더 잘 조정하고 관리할 수 있도록 합니다.
운전자 환경 개선: Driver SDK는 모바일 앱 툴킷이자 Fleet Engine의 기본 구성요소입니다. 차량의 기능을 사용합니다. 이 SDK를 사용하면 운전 앱에서 작업 할당을 관리하고 운전자 탐색 및 경로 지정 기능을 사용 설정할 수 있습니다. 기본적으로 SDK는 이벤트 리스너를 사용하여 운전자가 운전을 시작할 때 위치 업데이트를 Fleet Engine에 전송합니다. 또한 차량 상태에 관한 신호를 Fleet Engine에 전송합니다. 이 SDK를 사용하면 운전자는 할당을 관리하고 Google 지도 소비자 버전을 사용하는 것처럼 탐색할 수 있지만 다른 앱으로 전환할 필요는 없습니다.

Fleet Engine 아키텍처

이 섹션에서는 다양한 Fleet Engine 제품의 시스템 아키텍처를 제공합니다. 두 서비스 모두 다음을 사용합니다.

차량: 주문형 이동 및 예약된 작업은 운영을 수행하는 데 사용되는 비즈니스의 실제 차량을 모델링하는 수단으로 차량을 사용합니다. Fleet Engine에서 차량 법인은 Driver SDK와 통합된 운전자 앱을 통해 연결된 실제 차량에서 제공하는 위치 정보를 수신합니다. Fleet Engine 차량은 두 서비스 간에 대부분 유사한 항목이지만 몇 가지 차이점이 있습니다.
차량 소개 [https://developers.google.com/maps/documentation/mobility/fleet-engine/essentials/vehicles-intro?hl=ko]를 읽고 Fleet Engine의 차량을 이해하세요.
Driver SDK: 주문형 작업과 예약된 작업 모두에서 Driver SDK는 기본 구성요소입니다. Driver SDK는 운전자 앱과 통합되어 Fleet Engine이 여정의 실시간 상태와 ETA, 교통 데이터 등의 기타 정보를 보고하는 데 사용하는 경로 정보와 함께 위치 업데이트 스트림을 Fleet Engine에 제공합니다.
소비자 SDK: 주문형 작업과 예약된 작업 모두의 주요 이점은 소비자에게 차량, 음식 배달 또는 예약된 패키지 배송 상태를 보고하는 데 사용되는 소비자 SDK입니다.

Fleet Engine의 리소스 이름 지정

Fleet Engine에서는 모든 Google Cloud 기반 서비스와 마찬가지로 리소스에 문자열 형식으로 제공되는 names라는 고유 식별자가 있습니다. 리소스 이름의 일부는 생성 요청을 실행할 때 제공하는 리소스 ID입니다.
하지만 Fleet Engine 리소스에는 ID 필드가 없고 리소스 ID가 포함된 출력 전용 이름 필드가 있습니다.

Fleet Engine은 상대 리소스 이름을 사용합니다. 이러한 ID는 계층 구조를 나타내는 방식으로 구조화된 컬렉션 ID와 리소스 ID로 구성됩니다.

컬렉션 ID: 컬렉션은 리소스 또는 기타 컬렉션을 보유합니다.

[TAB_GROUP_PLACEHOLDER_4]

리소스 ID: 참고 문서의 리소스 식별자는 이름 필드에 중괄호로 표시됩니다. 이는 생성 요청을 실행할 때 제공하는 ID의 변수입니다.

[TAB_GROUP_PLACEHOLDER_5]

자세한 내용은 Google Cloud API 문서의 리소스 이름 [https://cloud.google.com/apis/design/resource_names?hl=ko]을 참고하세요.

Fleet Engine용 Google Cloud 로그

Fleet Engine은 API 요청과 응답 페이로드를 저장할 수 있는 기본 로깅 서비스를 제공합니다. 이러한 로그를 사용하면 통합을 디버그하고, 모니터링 측정항목을 만들고, 트래픽 패턴을 분석할 수 있습니다.

자세한 내용은 Cloud Logging [https://developers.google.com/maps/documentation/mobility/operations/cloud-logging?hl=ko]을 참고하세요.

Fleet Engine 사용 방법

| 1 | Cloud 프로젝트를 설정합니다. | 이 단계에서는 관련 API를 사용 설정하는 것 외에도 다양한 서비스 계정 역할을 만듭니다. Fleet Engine API 및 SDK를 사용하려면 Cloud 콘솔에서 생성된 서비스 계정을 사용하여 서명된 JSON 웹 토큰 (JWT)을 사용해야 합니다. Fleet Engine 프로젝트 만들기의 안내를 따릅니다. 이러한 역할을 더 잘 이해하려면 서비스 계정 역할을 참고하세요. |
| --- | --- | --- |
| 2 | 설정 확인하기 | 서비스 계정을 만든 후 설정이 완료되었으며 차량을 만들 수 있는지 확인합니다. 이 단계를 통해 프로젝트 설정 중에 발생할 수 있는 일반적인 승인 문제를 해결할 수 있습니다. 설정 확인의 안내를 따르세요. |
| 3 | 첫 번째 여정 만들기 | 여기에는 주문형 이동 또는 예약된 작업을 따라 정류장을 나타내는 좌표 집합이 있는 단일 차량을 만드는 작업이 포함됩니다. 자세한 내용은 주문형 이동 또는 예약된 작업의 개요 콘텐츠를 참고하세요. |
| 4 | 통합 테스트 | Fleet Engine 서비스와 구성요소에는 차량의 실시간 데이터가 필요합니다. 드라이버의 기기에 앱을 설계, 개발, 배포하는 데 시간이 걸릴 수 있습니다. 따라서 먼저 모의 데이터를 사용하여 통합을 살펴본 후 대규모 배포를 진행하는 것이 좋습니다. 자세한 내용은 Fleet Engine 설정을 참고하세요. |
