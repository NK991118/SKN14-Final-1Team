경우에 따라 단일 주소에 대해 Address Validation API를 여러 번 호출해야 할 수 있습니다. 예를 들어 고객이 첫 번째 유효성 검사 결과를 확인한 후 주소를 변경할 수 있습니다. 그런 다음 업데이트된 주소에 대해 두 번째 유효성 검사를 실행합니다.

이 문서에서는 Address Validation API에 동일한 주소에 관한 일련의 요청을 보낼 때 취할 수 있는 두 가지 주요 작업을 설명합니다. 이러한 작업은 엄격한 사용 요구사항은 아니지만 전반적인 데이터 품질과 시스템 응답을 개선하는 데 도움이 됩니다.

provideValidationFeedback 엔드포인트에 요청 전송

provideValidationFeedback 요청 전송

일련의 유효성 검사가 끝나면 Provide Validation Feedback API를 호출하여 모든 유효성 검사의 최종 결과를 표시하는 것이 좋습니다. 요청은 다음 두 필드를 설정합니다.

conclusion: 서비스에서 사용한 유효성 검사 버전을 나타냅니다. 사용자가 제공했는지 또는 Address Validation API에서 반환했는지 여부와 관계없이 유효성 검사 버전을 나타냅니다.
responseId: Address Validation API의 첫 번째 응답의 응답 ID를 나타냅니다. 아래의 previousResponseId 필드 설정 [https://developers.google.com/maps/documentation/address-validation/handle-updated-address?hl=ko#followup-requests]을 참고하세요.

provideValidationFeedback [https://developers.google.com/maps/documentation/address-validation/reference/rest/v1/TopLevel/provideValidationFeedback?hl=ko] 메서드 (REST) 또는 ProvideValidationFeedback [https://developers.google.com/maps/documentation/address-validationreference/rpc/google.maps.addressvalidation.v1?hl=ko#google.maps.addressvalidation.v1.AddressValidation.ProvideValidationFeedback] 메서드 (gRPC)를 사용합니다.

다음 예에서는 결론을 정의하는 요청에 JSON 본문 [https://developers.google.com/maps/documentation/address-validation/reference/rest/v1/TopLevel/validateAddress?hl=ko#request-body]을 전달합니다.

curl -X POST -d '{
"conclusion": "VALIDATED_VERSION_USED",
"responseId": "de22bed8-7f52-44cb-8526-faceac57150a"

VALIDATED_VERSION_USED: Address Validation API의 주소의 검증된 버전입니다.
USER_VERSION_USED: 사용자가 제공한 원본 버전입니다.
UNVALIDATED_VERSION_USED: 사용자에게 메시지를 표시한 결과 생성된 버전이지만 Address Validation API로 다시 검증하지 않았습니다.
UNUSED: 트랜잭션을 포기했습니다.

| 참조 가이드의 유효성 검사 결과를 참고하세요. |
| --- |

responseId 필드는 의견을 제공할 주소 유효성 검사 결과를 지정합니다. 이 값은 validateAddress [https://developers.google.com/maps/documentation/address-validation/reference/rest/v1/TopLevel/validateAddress?hl=ko] 메서드의 첫 번째 호출에서 반환된 responseId 값과 동일해야 합니다.

| 검증 의견 참조 가이드에서 요청 본문을 참고하세요. |
| --- |

previousResponseID 필드 설정

특정 주소에 대해 Address Validation API에 후속 요청을 보낼 때는 previousResponseId 필드를 설정합니다.

previousResponseId 값의 경우 첫 번째 검증 요청 후 API에서 반환한 responseId의 값을 사용합니다.
시리즈가 완료될 때까지 모든 후속 요청에 동일한 값을 사용합니다.
