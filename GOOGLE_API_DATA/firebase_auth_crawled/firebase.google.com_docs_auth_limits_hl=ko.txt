이 페이지의 내용결제 수단이 없는 일일 사용량 한도 [https://firebase.google.com/docs/auth/limits?hl=ko#daily_instrumentless_usage_limits]계정 생성 및 삭제 한도 [https://firebase.google.com/docs/auth/limits?hl=ko#account_creation_and_deletion_limits]계정 한도 [https://firebase.google.com/docs/auth/limits?hl=ko#account_limits]이메일 전송 한도 [https://firebase.google.com/docs/auth/limits?hl=ko#email_sending_limits]이메일 링크 생성 한도 [https://firebase.google.com/docs/auth/limits?hl=ko#email_link_generation_limits]전화번호 로그인 한도 [https://firebase.google.com/docs/auth/limits?hl=ko#phone-auth]인증 SMS 전송 한도 [https://firebase.google.com/docs/auth/limits?hl=ko#verification_sms_sending_limits]Identity Toolkit API 한도 [https://firebase.google.com/docs/auth/limits?hl=ko#identity_toolkit_api_limits]Token Service API 한도 [https://firebase.google.com/docs/auth/limits?hl=ko#token_service_api_limits]

다음 인증 작업은 실행 빈도가 제한되어 있습니다. 몇 주 전에 미리 Firebase에 문의하여 특별한 사용 사례를 논의해 보세요.
참고: 제한사항은 고객별로 적용되며 예고 없이 변경될 수 있습니다. 의심스러운 트래픽 패턴을 보이는 계정에 경고 없이 악용 방지 기능이 사용 설정될 수 있습니다.
결제 수단이 없는 일일 사용량 한도

다음은 Firebase Authentication with Identity Platform 사용자에게 무료 Spark 요금제가 없을 경우 일일 사용량 한도입니다.

이 사용량 한도는 Google Cloud 가격 책정 등급 [https://cloud.google.com/identity-platform/pricing?hl=ko]과 바로 일치합니다.

| 사용 | 결제 수단이 없을 경우 한도 |
| --- | --- |
| 등급 1 일일 활성 사용자 | 3,000명/일 |
| 등급 2 일일 활성 사용자 | 2명/일 |

계정 생성 및 삭제 한도

| 작업 | 한도 |
| --- | --- |
| 새 계정 생성 | IP 주소당 계정 100개/시간 |
| 계정 삭제 | 계정 10개/초 |
| 계정 일괄 삭제 | 요청 1회/초 |
| 계정 구성 업데이트 | 요청 10회/초 |

참고: Firebase Console에서 계정 생성 한도를 일시적으로 늘리도록 예약할 수 있습니다.

계정 한도

| 계정 유형 | 한도 |
| --- | --- |
| 익명 사용자 계정 | 1억 개 |
| 등록된 사용자 계정 | 무제한 |

이메일 전송 한도

이 섹션에 나열된 할당량은 사용자 수에 따라 확장됩니다.

| 작업 | Spark 요금제 한도 | Blaze 요금제 한도 |
| --- | --- | --- |
| 주소 확인 이메일 | 이메일 1,000개/일 | 이메일 100,000개/일 |
| 주소 변경 이메일 | 이메일 1,000개/일 | 이메일 10,000개/일 |
| 비밀번호 재설정 이메일 | 이메일 150개/일 | 이메일 10,000개/일 |
| 이메일 링크 로그인 이메일 | 이메일 5개/일 | 이메일 25,000개/일 |
참고: 이메일 링크 로그인 이메일의 한도가 최근에 변경되었습니다. 이메일 링크 로그인 이메일의 한도 5개를 초과하는 경우 결제 수단을 추가하세요.

이메일 링크 생성 한도

이 섹션에 나열된 할당량은 사용자 수에 따라 확장됩니다.

| 작업 | Spark 요금제 한도 | Blaze 요금제 한도 |
| --- | --- | --- |
| 주소 확인 링크 | 링크 10,000개/일 | 링크 1,000,000개/일 |
| 비밀번호 재설정 링크 | 링크 1,500개/일 | 링크 100,000개/일 |
| 로그인 링크 | 링크 20,000개/일 | 링크 250,000개/일 |

전화번호 로그인 한도

| 작업 | 한도 |
| --- | --- |
| 사용자 로그인 | 1,600건/분, 가격 책정 페이지에 지정된 가격 및 한도 |
| 인증 코드 SMS 메시지 | 사용한 만큼만 지불(Blaze) 요금제만 해당. Firebase Authentication: 전송된 SMS 3,000개/일 한도 Firebase Authentication with Identity Platform: 한도 없음 |
| 인증 요청 | 요청 150회/IP 주소/시간 |

인증 SMS 전송 한도

| 작업 | 한도 |
| --- | --- |
| 인증 SMS 전송. | 1,000개 전송/분 |
| IP 주소당 전송된 인증 SMS | 50개 전송/분, 500개 전송/시간 |

또한 일정 시간 내에 프로젝트가 단일 전화번호로 전송할 수 있는 인증 SMS 메시지 수에 한도가 있습니다. 가상 숫자 [https://firebase.google.com/docs/auth/android/phone-auth?hl=ko#test-with-fictional-phone-numbers]로 테스트하거나 여러 기기에서 테스트하여 프로젝트가 이러한 한도를 초과하지 않도록 할 수 있습니다.

또한 프로젝트에서 활동 로깅 [https://cloud.google.com/identity-platform/docs/activity-logging?hl=ko]을 사용 설정한 경우 전화번호별로 전송된 인증 코드를 추적할 수 있습니다.
