Google 판매자 센터 오프라인 판매점 인벤토리 표
bookmark_border
개요
판매자는 로컬 인벤토리 데이터를 사용하여 자신의 로컬 제품 제안을 BigQuery로 내보낼 수 있습니다. 이 데이터에는 수령 및 제품 매장 위치에 대한 정보와 함께 제품의 가격 책정, 재고, 수량이 포함됩니다.
오프라인 판매점 인벤토리 데이터 형식은 주로 Content API의 localinventory [https://developers.google.com/shopping-content/reference/rest/v2.1/localinventory?hl=ko] 리소스의 관련 필드 형식을 따릅니다.
사용하는 판매자 계정 유형에 따라 데이터는 다음 테이블 중 하나에 기록됩니다.
개별 판매자 ID를 사용하는 경우 데이터는 LocalInventories_MERCHANT_ID 테이블에 기록됩니다.
MCA 계정을 사용하는 경우 데이터가 LocalInventories_AGGREGATOR_ID 테이블에 기록됩니다.
스키마
LocalInventories_ 테이블에는 다음 스키마가 있습니다.
열 BigQuery 데이터 유형 설명 예시 데이터
product_id STRING 다음과 같은 형식의 Content API의 상품 REST ID: channel:content_language:feed_label:offer_id. 이 필드는 기본 키입니다. online:en:AU:666840730
merchant_id INTEGER 판매자 계정 ID 이 필드는 기본 키입니다.
aggregator_id INTEGER 멀티 클라이언트 계정(MCA) [https://support.google.com/merchants/answer/188487?hl=ko]의 ID입니다.
store_code STRING 이 오프라인 판매점 인벤토리 리소스의 판매점 코드
price RECORD 상품의 오프라인 가격
price.value NUMERIC 상품의 오프라인 가격 99
price.currency STRING 상품의 오프라인 가격 통화 CHF
sale_price RECORD 상품의 오프라인 할인가
sale_price.value NUMERIC 상품의 오프라인 할인가 49
sale_price.currency STRING 상품의 오프라인 할인가 통화 CHF
sale_price_effective_start_date TIMESTAMP 상품 판매 시작 날짜 및 시간 2021-03-30 00:00:00 UTC
sale_price_effective_end_date TIMESTAMP 상품 판매 종료 날짜 및 시간 2021-04-14 00:00:00 UTC
availability STRING 상품의 오프라인 재고 상태 재고 없음
quantity INTEGER 상품 수량 500
pickup_method STRING 상품의 지원되는 수령 방법 매장 배송
pickup_sla STRING 주문 날짜와 상품 수령 준비 날짜 간의 예상 경과 시간 3일
instore_product_location STRING 상품의 매장 내 위치
