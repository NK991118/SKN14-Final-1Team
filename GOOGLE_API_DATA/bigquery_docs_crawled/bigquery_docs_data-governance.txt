BigQuery의 데이터 거버넌스 소개
bookmark_border
BigQuery에는 데이터 및 AI 애셋을 탐색, 관리, 모니터링, 제어, 사용하는 방식을 간소화하는 기본 제공 거버넌스 기능이 있습니다.
관리자, 데이터 관리 책임자, 데이터 거버넌스 관리자, 데이터 관리자는 BigQuery의 거버넌스 기능을 사용하여 다음 작업을 할 수 있습니다.
데이터를 검색합니다.
데이터를 선별합니다.
메타데이터를 수집하고 보강합니다.
데이터 품질을 관리합니다.
데이터가 조직 정책을 준수하여 일관되게 사용되도록 합니다.
안전한 방식으로 대규모 데이터를 공유합니다.
BigQuery 거버넌스 기능은 조직의 모든 데이터 애셋에 대한 중앙 집중식 인벤토리인 Dataplex Universal Catalog [https://cloud.google.com/dataplex/docs/catalog-overview?hl=ko]를 기반으로 합니다. Dataplex Universal Catalog는 모든 데이터의 비즈니스, 기술, 운영 메타데이터를 보유합니다. 인공지능과 머신러닝을 적용하여 메타데이터의 관계와 시맨틱을 파악할 수 있습니다.
BigLake metastore를 사용하면 데이터 중복 없이 단일 스키마로 데이터의 단일 복사본을 쿼리하는 데 여러 데이터 처리 엔진을 사용할 수 있습니다. 사용할 수 있는 데이터 처리 엔진에는 BigQuery, Apache Spark, Apache Flink, Apache Hive가 있습니다. 데이터는 BigQuery 스토리지 테이블, BigQuery의 Apache Iceberg용 BigLake 테이블 또는 BigLake 외부 테이블과 같은 위치에 저장할 수 있습니다.
BigQuery는 탐색에서 데이터 사용에 이르기까지 엔드 투 엔드 데이터 수명 주기를 지원합니다. 거버넌스 기능은 Dataplex Universal Catalog에서도 사용할 수 있습니다.
데이터 검색
BigQuery는 데이터가 BigQuery, Spanner, Cloud SQL, Pub/Sub 또는 Cloud Storage에 있는지 여부와 관계없이 Google Cloud조직 전체에서 데이터를 검색합니다. 메타데이터는 자동으로 추출되어 Dataplex Universal Catalog에 저장됩니다. 예를 들어 Cloud Storage에서 정형 및 비정형 데이터의 메타데이터를 추출하고 쿼리 준비가 된 BigLake 테이블을 대규모로 자동 생성할 수 있습니다. 이를 통해 데이터 중복 없이 오픈소스 엔진으로 분석을 실행할 수 있습니다.
맞춤 커넥터를 사용하여 서드 파티 데이터 소스에서 메타데이터를 추출하고 분류할 수도 있습니다.
BigQuery는 다음과 같은 데이터 검색 기능을 제공합니다.
검색. 프로젝트 및 조직 전반에서 데이터 및 AI 리소스를 검색합니다. Google Cloud 콘솔의 BigQuery 내에서 시맨틱 검색 [https://cloud.google.com/bigquery/docs/search-resources?hl=ko](프리뷰 [https://cloud.google.com/products?hl=ko#product-launch-stages])을 사용하여 일상적인 언어로 리소스를 검색합니다. 또는 Dataplex Universal Catalog에서 키워드 검색 [https://cloud.google.com/dataplex/docs/search-assets?hl=ko]을 사용하여 리소스를 찾습니다.
Cloud Storage 데이터 자동 검색 [https://cloud.google.com/bigquery/docs/automatic-discovery?hl=ko]. Cloud Storage 버킷에서 데이터를 스캔하여 메타데이터를 추출하고 분류합니다. 자동 탐색은 정형 데이터와 비정형 데이터 모두에 대한 테이블을 만듭니다.
메타데이터 가져오기 [https://cloud.google.com/dataplex/docs/managed-connectivity-overview?hl=ko]. 서드 파티 시스템에서 Dataplex Universal Catalog로 대규모 메타데이터를 가져옵니다. 맞춤 커넥터를 빌드하여 데이터 소스에서 데이터를 추출한 다음 메타데이터 가져오기 워크플로를 오케스트레이션하는 관리형 연결 파이프라인을 실행할 수 있습니다.
메타데이터 내보내기 [https://cloud.google.com/dataplex/docs/export-metadata?hl=ko]. Dataplex Universal Catalog에서 대규모로 메타데이터를 내보냅니다. BigQuery로 내보낸 메타데이터를 분석하거나 메타데이터를 맞춤 애플리케이션 또는 프로그래매틱 처리 워크플로에 통합할 수 있습니다.
큐레이션 및 데이터 관리 책임
데이터의 검색 가능성과 유용성을 개선하기 위해 데이터 관리자와 관리자는 BigQuery를 사용하여 메타데이터를 검토, 업데이트, 분석할 수 있습니다. BigQuery 데이터 큐레이션 및 관리 기능을 사용하면 데이터가 정확하고 일관되며 조직의 정책에 부합하는지 확인할 수 있습니다.
BigQuery는 다음과 같은 데이터 선별 및 관리 기능을 제공합니다.
비즈니스 용어집 [https://cloud.google.com/dataplex/docs/create-glossary?hl=ko] (프리뷰 [https://cloud.google.com/products?hl=ko#product-launch-stages]). 용어집에서 조직의 용어를 정의하여 컨텍스트, 공동작업, 검색을 개선하세요. 용어의 데이터 관리자를 식별하고 용어를 데이터 애셋 필드에 연결합니다.
데이터 통계 [https://cloud.google.com/bigquery/docs/data-insights?hl=ko]. Gemini는 메타데이터를 사용하여 테이블에 관한 자연어 질문과 이에 답변하는 SQL 쿼리를 생성합니다. 이러한 데이터 통계를 통해 패턴을 파악하고, 데이터 품질을 평가하고, 통계 분석을 수행할 수 있습니다.
데이터 프로파일링 [https://cloud.google.com/bigquery/docs/data-profile-scan?hl=ko]. BigQuery 테이블에 있는 열의 일반적인 통계 특성을 식별하여 데이터를 더 효과적으로 이해하고 분석할 수 있습니다.
데이터 품질 [https://cloud.google.com/bigquery/docs/data-quality-scan?hl=ko]. BigQuery 및 Cloud Storage의 테이블 간에 데이터 품질 검사를 정의하고 실행하며 BigQuery 환경에 정기적이고 지속적인 데이터 컨트롤을 적용합니다.
데이터 계보 [https://cloud.google.com/dataplex/docs/about-data-lineage?hl=ko]. 데이터가 시스템을 통해 이동하는 방식, 즉 데이터의 출처, 데이터가 전달되는 위치, 데이터에 적용되는 변환을 추적할 수 있습니다. BigQuery는 테이블 및 열 수준에서 데이터 계보를 지원합니다.
큐레이션 및 데이터 관리 책임을 위한 다음 단계
다음 표에서는 큐레이션 및 데이터 관리 책임 기능에 대해 자세히 알아볼 수 있는 다음 단계를 설명합니다.
경험 수준 학습 과정
신규 클라우드 사용자
데이터 프로필 스캔 [https://cloud.google.com/bigquery/docs/data-profile-scan?hl=ko]을 실행하여 데이터 한도 또는 평균 등 데이터에 대한 유용한 정보를 수집합니다.
숙련된 클라우드 사용자
BigQuery 프로젝트에서 데이터 계보 [https://cloud.google.com/dataplex/docs/about-data-lineage?hl=ko#auto-lineage-bq-support]를 사용 설정하여 로드, 복사, 데이터 수정과 같은 BigQuery 작업의 계보 정보를 자동으로 기록합니다.
반복되는 데이터 품질 스캔 [https://cloud.google.com/bigquery/docs/data-quality-scan?hl=ko]을 설정하여 사전 정의된 스캔 규칙 [https://cloud.google.com/dataplex/docs/auto-data-quality-overview?hl=ko#predefined-rules]을 사용하여 발생 가능한 데이터 문제를 알립니다.
스캔이 특정 니즈에 맞도록 데이터 품질 스캔의 커스텀 데이터 규칙 [https://cloud.google.com/dataplex/docs/auto-data-quality-overview?hl=ko#supported-custom-sql-rule-types]을 설정합니다.
보안 및 액세스 제어
데이터 액세스 관리는 데이터에 액세스할 수 있는 사용자를 관리하는 규칙과 정책을 정의, 적용, 모니터링하는 프로세스입니다. 액세스 관리는 데이터에 액세스할 수 있도록 승인된 사용자만 데이터에 액세스할 수 있도록 합니다.
BigQuery는 다음과 같은 보안 및 액세스 제어 기능을 제공합니다.
Identity and Access Management(IAM) [https://cloud.google.com/bigquery/docs/access-control?hl=ko]. IAM을 사용하면 프로젝트, 데이터 세트, 테이블, 뷰와 같은 BigQuery 리소스에 액세스할 수 있는 사용자를 제어할 수 있습니다. 사용자, 그룹, 서비스 계정에 IAM 역할을 부여할 수 있습니다. 이러한 역할은 리소스로 수행할 수 있는 작업을 정의합니다.
열 수준 액세스 제어 [https://cloud.google.com/bigquery/docs/column-level-security-intro?hl=ko] 및 행 수준 액세스 제어 [https://cloud.google.com/bigquery/docs/row-level-security-intro?hl=ko]. 열 수준 및 행 수준 액세스 제어를 사용하면 사용자 속성이나 데이터 값을 기반으로 액세스를 테이블의 특정 열과 행으로 제한할 수 있습니다. 이 제어를 통해 무단 액세스로부터 민감한 정보를 보호하도록 세분화된 액세스를 구현할 수 있습니다.
데이터 전송 관리 [https://cloud.google.com/bigquery/docs/vpc-sc?hl=ko]. VPC 서비스 제어를 사용하면 Google Cloud리소스 주위에 경계를 만들고 조직 정책에 따라 이러한 리소스에 대한 액세스를 제어할 수 있습니다.
감사 로그 [https://cloud.google.com/bigquery/docs/introduction-audit-workloads?hl=ko]. 감사 로그는 조직 내에서의 사용자 활동과 시스템 이벤트에 대한 자세한 기록을 제공합니다. 이러한 로그는 데이터 거버넌스 정책을 적용하고 잠재적인 보안 위험을 식별하는 데 도움이 됩니다.
데이터 마스킹 [https://cloud.google.com/bigquery/docs/column-data-masking-intro?hl=ko]. 데이터 마스킹을 사용하면 승인된 사용자가 주변 데이터에 액세스하도록 허용하면서 테이블에서 민감한 정보를 숨길 수 있습니다. 데이터 마스킹은 민감한 정보 패턴과 일치하는 데이터를 가려 실수로 인한 데이터 공개를 방지할 수도 있습니다.
암호화 [https://cloud.google.com/bigquery/docs/encryption-at-rest?hl=ko]. BigQuery는 자동으로 모든 저장 데이터와 전송 중 데이터를 암호화하고 특정 요구사항에 맞게 암호화 설정을 맞춤설정할 수 있게 해줍니다.
보안 및 액세스 제어를 위한 다음 단계
다음 표에서는 액세스 제어 기능에 대해 자세히 알아볼 수 있는 다음 단계를 설명합니다.
경험 수준 학습 과정
신규 클라우드 사용자
BigQuery에서 사전 정의된 역할 [https://cloud.google.com/iam/docs/understanding-roles?hl=ko#predefined]을 살펴보고 최소 권한의 원칙 [https://wikipedia.org/wiki/Principle_of_least_privilege]에 따라 역할을 할당하는 방법을 알아봅니다.
Google에서 기본적으로 저장 데이터 [https://cloud.google.com/docs/security/encryption/default-encryption?hl=ko]와 전송 중 데이터 [https://cloud.google.com/docs/security/encryption-in-transit?hl=ko]를 암호화하는 방법을 알아봅니다.
숙련된 클라우드 사용자
권한 관리의 유연성과 세분성을 높이려면 니즈에 맞는 커스텀 역할을 만드는 [https://cloud.google.com/iam/docs/creating-custom-roles?hl=ko] 것이 좋습니다.
행 [https://cloud.google.com/bigquery/docs/row-level-security-intro?hl=ko]과 열 제어 [https://cloud.google.com/bigquery/docs/column-level-security-intro?hl=ko]를 추가하여 테이블의 특정 행과 열에 대한 액세스를 제어합니다.
Google Cloud리소스 주위에 액세스 경계를 설정하려면 VPC 서비스 제어를 설정 [https://cloud.google.com/vpc-service-controls/docs/set-up-service-perimeter-verify-access?hl=ko]하세요.
열 수준 데이터 마스킹 [https://cloud.google.com/bigquery/docs/column-data-masking?hl=ko]을 테이블에 추가하여 민감한 정보를 공개하지 않고 조직을 통해 정보를 공유할 수 있습니다.
Sensitive Data Protection [https://cloud.google.com/sensitive-data-protection/docs/data-profiles?hl=ko]을 사용하여 개인 식별 정보(PII), 금융 데이터, 건강 정보와 같은 민감하고 위험성이 높은 정보의 데이터에 스캔합니다.
공유 데이터 및 통계
BigQuery를 사용하면 조직 내 및 조직 간에 대규모로 데이터와 유용한 정보를 공유할 수 있습니다. 내장된 데이터 교환 플랫폼을 통해 강력한 보안 및 개인 정보 보호 프레임워크를 갖추고 있습니다. BigQuery Sharing [https://cloud.google.com/bigquery/docs/analytics-hub-introduction?hl=ko]을 사용하면 다양한 데이터 제공업체가 선별한 데이터 라이브러리를 찾아 액세스하고 사용할 수 있습니다.
BigQuery는 다음과 같은 공유 기능을 제공합니다.
데이터 이상의 정보 공유 [https://cloud.google.com/bigquery/docs/analytics-hub-introduction?hl=ko]. BigQuery 데이터 세트, 테이블, 뷰, Pub/Sub 주제가 있는 실시간 스트림, SQL 저장 프로시저, BigQuery ML 모델 등 다양한 데이터 및 AI 애셋을 공유할 수 있습니다.
Google 데이터 세트에 액세스 [https://cloud.google.com/datasets?hl=ko]. 검색 트렌드, DeepMind WeatherNext 모델, Google Maps Platform, Google Earth Engine 등의 Google 데이터 세트로 분석 및 ML 이니셔티브를 보강하세요.
데이터 거버넌스 원칙과 통합 [https://cloud.google.com/bigquery/docs/analytics-hub-grant-roles?hl=ko]. 데이터 소유자는 데이터를 계속 제어할 수 있으며 액세스 및 사용을 제한하는 규칙 또는 정책을 정의하고 구성할 수 있습니다.
실시간 제로 카피 데이터 공유 [https://cloud.google.com/bigquery/docs/analytics-hub-introduction?hl=ko#linked_resources]. 통합, 데이터 이동 또는 복제가 필요 없이 데이터가 제자리에서 공유되므로 최신 정보를 기반으로 분석할 수 있습니다. 생성된 연결된 데이터 세트는 공유 애셋에 대한 실시간 포인터입니다.
보안 상황 개선 [https://cloud.google.com/bigquery/docs/analytics-hub-vpc-sc-rules?hl=ko]. 액세스 제어를 사용하여 기본 제공 VPC 서비스 제어 지원을 비롯한 과도한 액세스 프로비저닝을 줄일 수 있습니다.
제공업체 사용량 측정항목으로 가시성을 증대 [https://cloud.google.com/bigquery/docs/analytics-hub-monitor-listings?hl=ko]. 데이터 게시자는 실행된 작업 수, 스캔된 총 바이트 수, 각 조직의 구독자 수와 같은 공유 애셋의 사용량을 확인하고 모니터링할 수 있습니다.
데이터 클린룸을 사용하여 민감한 정보에 대해 공동작업 [https://cloud.google.com/bigquery/docs/data-clean-rooms?hl=ko]. 데이터 클린룸은 여러 관계자가 기본 데이터를 이동하거나 공개하지 않고도 데이터 애셋을 공유, 조인, 분석할 수 있는 보안 강화 환경을 제공합니다.
BigQuery 기반 [https://cloud.google.com/analytics-hub?hl=ko]. BigQuery의 확장성과 대규모 처리 기능을 기반으로 대규모 공동작업을 지원할 수 있습니다.
