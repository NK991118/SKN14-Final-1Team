BigQuery와 같은 데이터 웨어하우스에서 위치 정보는 일반적이며 중요한 비즈니스 결정에 영향을 줄 수 있습니다. 지리정보 분석을 사용하면 GEOGRAPHY 데이터 유형 [https://cloud.google.com/bigquery/docs/reference/standard-sql/data-types?hl=ko#geography_type]과 GoogleSQL 지리 함수 [https://cloud.google.com/bigquery/docs/reference/standard-sql/geography_functions?hl=ko]를 사용하여 BigQuery에서 지리공간 데이터를 분석하고 시각화할 수 있습니다.
예를 들면 시간의 경과에 따라 배송 차량 또는 패키지의 위도와 경도를 기록할 수 있습니다. 고객 트랜잭션을 기록하고 데이터를 점포 위치 데이터가 있는 다른 테이블에 조인할 수도 있습니다. 이러한 유형의 위치 데이터를 사용하여 다음을 수행할 수 있습니다.
패키지가 도착할 것으로 예상되는 시점을 추정합니다.
특정한 상점 위치의 우편물을 수신할 고객을 정합니다.
위성 이미지의 나무 덮개 비율과 데이터를 결합하여 항공 드론 배송이 가능한지 판단합니다.
제한사항
지리정보 분석에는 다음과 같은 제한사항이 적용됩니다.
지리 함수 [https://cloud.google.com/bigquery/docs/reference/standard-sql/geography_functions?hl=ko]는 GoogleSQL에서만 사용할 수 있습니다.
Python용 BigQuery 클라이언트 라이브러리에서만 GEOGRAPHY 데이터 유형이 지원됩니다. 다른 클라이언트 라이브러리의 경우 ST_ASTEXT 또는 ST_ASGEOJSON 함수를 사용하여 GEOGRAPHY 값을 문자열로 변환합니다. ST_ASTEXT를 사용하여 텍스트로 변환하면 하나의 값만 저장되며, WKT로 변환하면 데이터가 GEOGRAPHY 유형 대신 STRING 유형으로 주석 처리됩니다.
할당량
지리정보 분석의 할당량 및 한도는 지리정보 데이터가 포함된 테이블에 실행할 수 있는 여러 작업 유형에 적용되며, 여기에는 다음 작업 유형이 포함됩니다.
데이터 로드 [https://cloud.google.com/bigquery/quotas?hl=ko#load_jobs](로드 작업)
데이터 내보내기 [https://cloud.google.com/bigquery/quotas?hl=ko#export_jobs](내보내기 작업)
데이터 쿼리 [https://cloud.google.com/bigquery/quotas?hl=ko#query_jobs](쿼리 작업)
테이블 복사 [https://cloud.google.com/bigquery/quotas?hl=ko#copy_jobs](복사 작업)
모든 할당량 및 한도에 대한 자세한 내용은 할당량 및 한도 [https://cloud.google.com/bigquery/quotas?hl=ko]를 참조하세요.
가격 책정
지리정보 분석 사용 요금은 다음 요소를 기준으로 합니다.
지리정보 데이터가 포함된 테이블에 저장된 데이터의 양
데이터에 대하여 실행하는 쿼리
스토리지 가격 책정에 대한 자세한 내용은 스토리지 가격 책정 [https://cloud.google.com/bigquery/pricing?hl=ko#storage]을 참조하세요.
쿼리 가격 책정에 대한 자세한 내용은 분석 가격 책정 모델 [https://cloud.google.com/bigquery/pricing?hl=ko#analysis_pricing_models]을 참조하세요.
데이터 로드, 복사, 내보내기를 포함한 여러 테이블 작업이 무료입니다. 이러한 작업은 무료이더라도 BigQuery의 할당량 및 한도 [https://cloud.google.com/bigquery/quotas?hl=ko]가 적용됩니다. 모든 무료 작업에 대한 자세한 내용은 가격 책정 페이지의 무료 작업 [https://cloud.google.com/bigquery/pricing?hl=ko#free]을 참조하세요.
다음 단계
지리정보 분석을 시작하려면 지리정보 분석 시작하기 [https://cloud.google.com/bigquery/docs/geospatial-get-started?hl=ko]를 참조하세요.
지리정보 분석의 시각화 옵션에 대한 자세한 내용은 지리정보 데이터 시각화 [https://cloud.google.com/bigquery/docs/geospatial-visualize?hl=ko]를 참조하세요.
지리정보 데이터를 사용하는 작업에 대한 자세한 내용은 지리정보 데이터 작업 [https://cloud.google.com/bigquery/docs/geospatial-data?hl=ko]을 참조하세요.
래스터 데이터 작업에 대해 자세히 알아보려면 래스터 데이터 작업 [https://cloud.google.com/bigquery/docs/raster-data?hl=ko]을 참고하세요.
Google Earth Engine 지리 공간 데이터를 BigQuery에 통합하는 방법에 대한 자세한 내용은 Google Earth Engine 지리 공간 데이터 로드 [https://cloud.google.com/bigquery/docs/geospatial-data?hl=ko#load-ee]를 참조하세요.
지리 공간 분석의 GoogleSQL 함수에 대한 문서는 GoogleSQL의 지리 함수 [https://cloud.google.com/bigquery/docs/reference/standard-sql/geography_functions?hl=ko]를 참조하세요.
다양한 그리드 시스템에 대해 알아보려면 공간 분석을 위한 그리드 시스템 [https://cloud.google.com/bigquery/docs/grid-systems-spatial-analysis?hl=ko]을 참고하세요.
지리 공간 데이터 세트, 지리 공간 분석, AI에 대해 자세히 알아보려면 지리 공간 분석 [https://cloud.google.com/solutions/geospatial?hl=ko]을 참고하세요.
