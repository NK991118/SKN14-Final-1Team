Google Cloud Ready - BigQuery 파트너
bookmark_border
BigQuery 검증 프로그램을 이용해 파트너는 BigQuery와의 통합을 검증하여 최상의 통합을 보장할 수 있습니다. BigQuery 검증 프로그램은 다음과 같은 3단계로 진행됩니다.
평가: Google팀이 샌드박스 프로덕션 환경에서 일련의 데이터 통합 테스트를 실행하고 결과를 권장사항 및 성능 벤치마크와 비교합니다.
강화: 평가 단계의 결과를 얻고 부족한 부분이 있는 경우 파트너와 협력하여 이를 채웁니다.
지원: 파트너와 함께 공통 고객이 Google 제품을 성공적으로 사용하는 데 필요한 모든 것을 갖출 수 있도록 기존 문서를 수정합니다.
이 문서에는 Google Cloud Ready - BigQuery 프로그램 [https://cloud.google.com/bigquery/docs/bigquery-ready-overview?hl=ko]의 일환으로 검증된 파트너 솔루션이 나와 있습니다. 서드 파티 소프트웨어가 BigQuery가 의도한 대로 작동한다고 판단되면 Cloud Customer Care에서 지원되지 않습니다. 솔루션은 다음 카테고리로 구성됩니다.

Stitch
솔루션 Stitch [https://www.stitchdata.com/]
카테고리 ETL 및 데이터 통합
설명
Stitch는 신속한 데이터 이동을 위한 클라우드 중심의 오픈소스 플랫폼입니다. 간단하고 강력한 ETL 서비스인 Stitch는 MySQL과 같은 데이터베이스에서 모든 데이터 소스에 연결됩니다.
Stitch의 목적은 유용한 원시 형식으로 데이터 소스에서 대상으로 데이터를 가져오는 것입니다. 여기에서 유용한 형식은 데이터를 쉽게 사용할 수 있게 해주는 유형 및 구조를 의미하고, 원시 형식은 가능한 한 원본과 가까운 형식을 유지하는 것을 의미합니다. 즉, Stitch가 복제 프로세스 중 변환을 수행하지 않는 것은 아닙니다. Stitch는 임의적인 변환을 수행하지 않을 뿐입니다. Stitch에서는 로드된 데이터가 유용하고 사용자의 대상과 일치하도록 보장하는 데 필요한 변환만 수행됩니다. Stitch에서 수행되는 변환은 선택한 대상에 따라 달라집니다.
