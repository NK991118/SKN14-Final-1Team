데이터 변환 소개
bookmark_border
이 문서에서는 BigQuery 테이블에서 데이터를 변환할 수 있는 다양한 방법을 설명합니다.
데이터 통합에 관한 자세한 내용은 데이터 로드, 변환, 내보내기 소개 [https://cloud.google.com/bigquery/docs/load-transform-export-intro?hl=ko]를 참조하세요.
데이터 변환 방법
다음 방법으로 BigQuery의 데이터를 변환할 수 있습니다.
BigQuery 테이블의 데이터를 변환하려면 DML [https://cloud.google.com/bigquery/docs/transform-intro?hl=ko#transform-with-dml]을 사용합니다.
성능 및 효율성 향상을 위해 쿼리 결과를 자동으로 캐시하려면 구체화된 뷰 [https://cloud.google.com/bigquery/docs/transform-intro?hl=ko#transform-with-mvs]를 사용합니다.
수신되는 데이터를 실시간으로 분석하고 출력 행을 BigQuery 테이블에 지속적으로 삽입하거나 Pub/Sub 또는 Bigtable로 내보내려면 연속 쿼리 [https://cloud.google.com/bigquery/docs/transform-intro?hl=ko#transform-with-continuous-queries]를 사용합니다.
BigQuery에서 파이프라인을 개발, 테스트, 버전 제어, 예약하려면 BigQuery 파이프라인 [https://cloud.google.com/bigquery/docs/transform-intro?hl=ko#transform-with-bq-pipelines] 또는 Dataform [https://cloud.google.com/bigquery/docs/transform-intro?hl=ko#transform-with-dataform]을 사용합니다.
분석을 위해 데이터를 정리하려면 컨텍스트를 인식하는 AI 생성 변환 추천이 포함된 데이터 준비 [https://cloud.google.com/bigquery/docs/transform-intro?hl=ko#data-preparation]를 사용합니다. 데이터 준비는 Dataform API [https://cloud.google.com/dataform/reference/rest?hl=ko]를 기반으로 합니다.
다음 표에서는 각 변환 방법의 서로 다른 특징을 보여줍니다.

DML을 사용한 데이터 변환
데이터 조작 언어(DML) [https://cloud.google.com/bigquery/docs/data-manipulation-language?hl=ko]를 사용하여 BigQuery 테이블의 데이터를 변환할 수 있습니다. DML 문은 기존 테이블 데이터를 조작하여 행을 추가 또는 삭제하거나 기존 행의 데이터를 수정하거나 다른 테이블의 값과 데이터를 병합하는 GoogleSQL 쿼리입니다. DML 변환은 파티션을 나눈 테이블 [https://cloud.google.com/bigquery/docs/using-dml-with-partitioned-tables?hl=ko]에서도 지원됩니다.
여러 DML 문을 동시에 실행할 수 있으며, 여기에서 BigQuery는 데이터를 하나씩 변환하는 여러 DML 문을 큐에 추가합니다. BigQuery는 변환 유형에 따라 동시 DML 문 실행 방법 [https://cloud.google.com/bigquery/docs/data-manipulation-language?hl=ko#concurrent_jobs]을 관리합니다.
구체화된 뷰로 데이터 변환
구체화된 뷰 [https://cloud.google.com/bigquery/docs/materialized-views-intro?hl=ko]는 성능 및 효율성 향상을 위해 SQL 쿼리 결과를 주기적으로 캐시하는 미리 계산된 뷰입니다. BigQuery는 구체화된 뷰에서 미리 계산된 결과를 활용하고 가능한 모든 경우에 기본 테이블에서 변경사항만 읽어 최신 결과를 계산합니다.
기본 테이블이 변경되면 구체화된 뷰가 백그라운드에서 미리 계산됩니다. 기본 테이블의 모든 증분 데이터 변경사항은 사용자가 작업하지 않아도 구체화된 뷰에 자동으로 추가됩니다.
연속 쿼리로 데이터 변환
프리뷰
이 기능에는 서비스별 약관 [https://cloud.google.com/terms/service-terms?hl=ko#1]의 일반 서비스 약관 섹션에 있는 'GA 이전 제공 서비스 약관'이 적용됩니다. GA 이전 기능은 '있는 그대로' 제공되며 지원이 제한될 수 있습니다. 자세한 내용은 출시 단계 설명 [https://cloud.google.com/products?hl=ko#product-launch-stages]을 참조하세요.
연속 쿼리 [https://cloud.google.com/bigquery/docs/continuous-queries-introduction?hl=ko]는 지속적으로 실행되는 SQL 문입니다. 연속 쿼리를 사용하면 BigQuery에서 수신 데이터를 실시간으로 분석할 수 있습니다. 연속 쿼리에서 생성된 출력 행을 BigQuery 테이블에 삽입하거나 Pub/Sub 또는 Bigtable로 내보낼 수 있습니다.
Dataform으로 데이터 변환
Dataform을 사용하면 데이터 통합을 위한 ELT(추출, 로드 및 변환) 프로세스에서 데이터 변환을 관리할 수 있습니다. 소스 시스템에서 원시 데이터를 추출하여 BigQuery로 로드한 후 Dataform을 사용하여 구성되고 테스트되고 문서화된 테이블 제품군으로 변환할 수 있습니다. DML에서는 데이터를 정확히 어떻게 변환할지 BigQuery에 지시하는 명령형 접근 방식을 취하는 반면, Dataform에서는 선언적 문을 작성하면 Dataform이 해당 상태를 달성하는 데 필요한 변환을 결정합니다.
Dataform에서는 데이터 소스 선언부터 출력 테이블, 뷰 또는 구체화된 뷰까지 데이터 변환을 위한 SQL 워크플로 [https://cloud.google.com/dataform/docs/sql-workflows?hl=ko]를 개발, 테스트, 버전 제어할 수 있습니다. Dataform 코어 또는 순수 자바스크립트로 SQL 워크플로를 개발할 수 있습니다. Dataform 코어 [https://cloud.google.com/dataform/docs/overview?hl=ko#dataform-core]는 SQLX 및 자바스크립트로 SQL을 확장하는 오픈소스 메타 언어입니다. Dataform 코어를 사용하여 종속 항목을 관리하고 자동화된 데이터 품질 테스트를 설정하며 코드 내에서 테이블 또는 열 설명을 문서화할 수 있습니다.
Dataform은 SQL 워크플로 코드를 저장소 [https://cloud.google.com/dataform/docs/create-repository?hl=ko]에 저장하고 Git를 사용하여 파일 변경사항을 추적합니다. Dataform의 개발 작업공간을 사용하면 같은 저장소에서 작업하는 다른 사람의 작업에 영향을 주지 않고 저장소의 콘텐츠에 따라 작업할 수 있습니다. Dataform 저장소를 Azure DevOps Services, Bitbucket, GitHub, GitLab을 포함한 서드 파티 Git 제공업체에 연결할 수 있습니다.
Dataform 출시 구성 및 워크플로 구성을 사용하여 SQL 워크플로를 실행하거나 예약할 수 있습니다. 또는 Cloud Composer 또는 Workflows 및 Cloud Scheduler를 사용하여 실행을 예약할 수 있습니다. 실행 중에 Dataform은 SQL 워크플로에서 객체 종속 항목 순서대로 BigQuery의 SQL 쿼리를 실행합니다. 실행 후에는 정의된 테이블과 뷰를 BigQuery에서 분석으로 사용할 수 있습니다.
Dataform에서 데이터 변환 SQL 워크플로를 만드는 방법에 대한 자세한 내용은 Dataform 개요 [https://cloud.google.com/dataform/docs/overview?hl=ko] 및 Dataform 기능 [https://cloud.google.com/dataform/docs/overview?hl=ko#features]을 참조하세요.
BigQuery 파이프라인으로 데이터 변환
BigQuery 파이프라인은 Dataform을 기반으로 하며 추출, 로드, 변환(ELT) 또는 추출, 변환, 로드(ETL) 프로세스에서 데이터 변환을 만들고 관리할 수 있습니다.
BigQuery Studio에서 시각적인 방식으로 BigQuery 파이프라인을 만들고 관리할 수 있습니다.
BigQuery 파이프라인 만들기를 자세히 알아보려면 파이프라인 만들기 [https://cloud.google.com/bigquery/docs/create-pipelines?hl=ko]를 참조하세요.
