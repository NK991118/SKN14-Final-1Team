파이프라인 관리
bookmark_border
이 문서에서는 파이프라인을 예약하고 삭제하는 방법을 비롯하여 BigQuery 파이프라인 [https://cloud.google.com/bigquery/docs/pipelines-introduction?hl=ko]을 관리하는 방법을 설명합니다.
이 문서에서는 Dataplex Universal Catalog [https://cloud.google.com/dataplex/docs/introduction?hl=ko]에서 파이프라인 메타데이터를 보고 관리하는 방법도 설명합니다.
파이프라인은 Dataform [https://cloud.google.com/dataform/docs/overview?hl=ko]을 기반으로 합니다.
시작하기 전에
BigQuery 파이프라인을 만듭니다 [https://cloud.google.com/bigquery/docs/create-pipelines?hl=ko].
Dataplex Universal Catalog에서 파이프라인 메타데이터를 관리하려면 Google Cloud 프로젝트에서 Dataplex API [https://cloud.google.com/dataplex/docs/enable-api?hl=ko]가 사용 설정되어 있어야 합니다.
필요한 역할
파이프라인을 관리하는 데 필요한 권한을 얻으려면 관리자에게 다음 IAM 역할을 부여해 달라고 요청하세요.
파이프라인 삭제: 파이프라인에 대한 Dataform 관리자 [https://cloud.google.com/iam/docs/roles-permissions/dataform?hl=ko#dataform.Admin](roles/dataform.Admin)
파이프라인 보기 및 실행: 프로젝트에 대한 Dataform 뷰어 [https://cloud.google.com/iam/docs/roles-permissions/dataform?hl=ko#dataform.Viewer](roles/dataform.Viewer)
역할 부여에 대한 자세한 내용은 프로젝트, 폴더, 조직에 대한 액세스 관리 [https://cloud.google.com/iam/docs/granting-changing-revoking-access?hl=ko]를 참조하세요.
커스텀 역할 [https://cloud.google.com/iam/docs/creating-custom-roles?hl=ko]이나 다른 사전 정의된 역할 [https://cloud.google.com/iam/docs/roles-overview?hl=ko#predefined]을 통해 필요한 권한을 얻을 수도 있습니다.
Dataplex Universal Catalog에서 파이프라인 메타데이터를 관리하려면 필요한 Dataplex Universal Catalog 역할 [https://cloud.google.com/dataplex/docs/iam-roles?hl=ko]이 있어야 합니다.
Dataform IAM에 대한 자세한 내용은 IAM으로 액세스 제어 [https://cloud.google.com/dataform/docs/access-control?hl=ko]를 참조하세요.
참고: 파이프라인을 만들면 BigQuery에서 해당 파이프라인에 대한 Dataform 관리자(roles/dataform.admin [https://cloud.google.com/iam/docs/understanding-roles?hl=ko#dataform.admin]) 역할을 부여합니다. Google Cloud 프로젝트에 대한 Dataform 관리자(roles/dataform.admin) 역할 [https://cloud.google.com/dataform/docs/access-control?hl=ko#dataform.admin]이 부여된 모든 사용자는 프로젝트에서 생성된 모든 파이프라인에 대한 소유자 액세스 권한을 갖습니다.

탐색기 창에서 more_vert를 클릭하고 파이프라인을 펼칩니다.

선택한 파이프라인의 이전 수동 실행을 보려면 다음 단계를 따르세요.
Google Cloud 콘솔에서 BigQuery 페이지로 이동합니다.
BigQuery로 이동 [https://console.cloud.google.com/bigquery?hl=ko]
탐색기 창에서 프로젝트와 파이프라인 폴더를 펼친 후 파이프라인을 선택합니다.
실행을 클릭합니다.

실패한 파이프라인 실행에 대한 알림 구성
각 파이프라인에는 해당하는 Dataform 저장소 ID가 있습니다. 각 BigQuery 파이프라인 실행은 해당 Dataform 저장소 ID를 사용하여 Cloud Logging [https://cloud.google.com/logging/docs?hl=ko]에 로깅됩니다. Cloud Monitoring을 사용하여 Cloud Logging 로그에서 BigQuery 파이프라인 실행 추세를 관찰하고 사용자가 지정한 조건이 발생하면 알림을 받을 수 있습니다.
BigQuery 파이프라인 실행이 실패할 경우 알림을 수신하려면 해당 Dataform 저장소 ID에 로그 기반 알림 정책을 만들면 됩니다. 자세한 내용은 실패한 워크플로 호출에 대한 알림 구성 [https://cloud.google.com/dataform/docs/monitor-runs?hl=ko#configure-alerts-failed-workflow-invocations]을 참조하세요.
파이프라인의 Dataform 저장소 ID를 찾으려면 다음 단계를 따르세요.
Google Cloud 콘솔에서 BigQuery 페이지로 이동합니다.
BigQuery로 이동 [https://console.cloud.google.com/bigquery?hl=ko]
탐색기 창에서 프로젝트와 파이프라인 폴더를 펼친 후 파이프라인을 선택합니다.
설정을 클릭합니다.
파이프라인의 Dataform 저장소 ID가 설정 탭 하단에 표시됩니다.

Google Cloud 콘솔에서 BigQuery 페이지로 이동합니다.
BigQuery로 이동 [https://console.cloud.google.com/bigquery?hl=ko]

파이프라인 옆에 있는 more_vert 작업 보기를 클릭한 후 삭제를 클릭합니다.
삭제를 클릭합니다.
Dataplex Universal Catalog에서 메타데이터 관리
Dataplex Universal Catalog를 사용하면 파이프라인의 메타데이터를 저장하고 관리할 수 있습니다. Dataplex Universal Catalog에서는 기본적으로 추가 구성 없이 파이프라인을 사용할 수 있습니다.
Dataplex Universal Catalog를 사용하여 모든 파이프라인 위치 [https://cloud.google.com/bigquery/docs/locations?hl=ko]의 파이프라인을 관리할 수 있습니다. Dataplex Universal Catalog에서 파이프라인을 관리하는 경우 Dataplex Universal Catalog 할당량 및 한도 [https://cloud.google.com/dataplex/docs/quotas?hl=ko]와 Dataplex Universal Catalog 가격 책정 [https://cloud.google.com/dataplex/pricing?hl=ko]이 적용됩니다.
Dataplex Universal Catalog는 파이프라인에서 다음 메타데이터를 자동으로 검색합니다.
데이터 애셋 이름
데이터 애셋 상위 항목
데이터 애셋 위치
데이터 애셋 유형
해당 Google Cloud 프로젝트
Dataplex Universal Catalog는 파이프라인을 다음과 같은 항목 값을 사용해 항목 [https://cloud.google.com/dataplex/docs/ingest-custom-sources?hl=ko#entries]으로 로깅합니다.
시스템 항목 그룹
파이프라인의 시스템 항목 그룹 [https://cloud.google.com/dataplex/docs/ingest-custom-sources?hl=ko#entry-groups]은 @dataform입니다. Dataplex Universal Catalog에서 파이프라인 항목의 세부정보를 보려면 dataform 시스템 항목 그룹을 확인해야 합니다. 항목 그룹의 모든 항목 목록을 보는 방법에 관한 안내는 Dataplex Universal Catalog 문서의 항목 그룹 세부정보 보기 [https://cloud.google.com/dataplex/docs/ingest-custom-sources?hl=ko#entry-group-details]를 참조하세요.
시스템 항목 유형
파이프라인의 시스템 항목 유형 [https://cloud.google.com/dataplex/docs/ingest-custom-sources?hl=ko#entry-types]은 dataform-code-asset입니다. 파이프라인의 세부정보를 보려면 dataform-code-asset 시스템 항목 유형을 확인하고, 관점 기반 필터로 결과를 필터링하고, dataform-code-asset 관점 내의 type 필드를 WORKFLOW로 설정 [https://cloud.google.com/dataplex/docs/search-syntax?hl=ko#aspect-search]해야 합니다. 그런 다음 선택한 파이프라인의 항목을 선택합니다. 선택한 항목 유형의 세부정보를 보는 방법에 관한 안내는 Dataplex Universal Catalog 문서의 항목 유형의 세부정보 보기 [https://cloud.google.com/dataplex/docs/ingest-custom-sources?hl=ko#entry-type-details]를 참조하세요. 선택한 항목의 세부정보를 보는 방법에 관한 안내는 Dataplex Universal Catalog 문서의 항목 세부정보 보기 [https://cloud.google.com/dataplex/docs/search-assets?hl=ko#view-entry-details]를 참조하세요.
시스템 관점 유형
파이프라인의 시스템 관점 유형 [https://cloud.google.com/dataplex/docs/enrich-entries-metadata?hl=ko#aspect-types]은 dataform-code-asset입니다. 관점 [https://cloud.google.com/dataplex/docs/enrich-entries-metadata?hl=ko#aspects]으로 데이터 파이프라인 항목에 주석을 추가하여 Dataplex Universal Catalog의 파이프라인에 추가 컨텍스트를 제공하려면 dataform-code-asset 관점 유형을 확인하고 관점 기반 필터로 결과를 필터링한 다음 dataform-code-asset 관점 내의 type 필드를 WORKFLOW로 설정 [https://cloud.google.com/dataplex/docs/search-syntax?hl=ko#aspect-search]합니다. 관점으로 항목에 주석을 추가하는 방법에 관한 안내는 Dataplex Universal Catalog 문서의 관점 관리 및 메타데이터 보강 [https://cloud.google.com/dataplex/docs/enrich-entries-metadata?hl=ko]을 참조하세요.
유형
데이터 캔버스의 유형은 WORKFLOW입니다. 이 유형을 사용하면 관점 기반 필터 [https://cloud.google.com/dataplex/docs/search-syntax?hl=ko#aspect-search]에서 aspect:dataplex-types.global.dataform-code-asset.type=WORKFLOW 쿼리를 사용하여 dataform-code-asset 시스템 항목 유형 및 dataform-code-asset 관점 유형의 파이프라인을 필터링할 수 있습니다.
Dataplex Universal Catalog에서 애셋을 검색하는 방법에 관한 안내는 Dataplex Universal Catalog 문서의 Dataplex Universal Catalog에서 데이터 애셋 검색 [https://cloud.google.com/dataplex/docs/search-assets?hl=ko]을 참고하세요.
