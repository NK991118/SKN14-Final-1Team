BigQuery Data Transfer Service 데이터 소스 변경 로그
bookmark_border
이 페이지에서는 BigQuery Data Transfer Service 데이터 소스 스키마 및 스키마 매핑의 변경사항에 관해 자세히 설명합니다. BigQuery Data Transfer Service 커넥터의 예정된 변경사항에 대한 자세한 내용은 이 페이지에서 Google Ads API 또는 Display & Video 360 API과 같은 데이터 소스나 특정 테이블 이름 또는 값을 검색하세요.
Campaign Manager 360
Campaign Manager 360 커넥터용 BigQuery Data Transfer Service는 새로운 열, 지원 중단된 열, 이전된 열을 지원하기 위해 정기적으로 업데이트됩니다. Campaign Manager 360 커넥터용 BigQuery Data Transfer Service는 Campaign Manager 360 데이터 전송 파일 [https://developers.google.com/doubleclick-advertisers/dtv2/reference/release-notes?hl=ko]에서 데이터를 가져옵니다.
다음 섹션에서는 변경사항을 간략하게 설명합니다. 변경사항은 출시일별로 정리되어 있으며 각 항목에는 Campaign Manager 360에서 데이터를 계속 수신하기 위해 적용해야 하는 변경사항에 관한 정보가 제공됩니다.
2025년 7월 7일
Campaign Manager 360은 크로스 플랫폼 데이터 표준에 맞게 브라우저, 운영체제, 모바일 제조사 및 모델, ISP 데이터의 기준 ID를 업데이트한다고 발표 [https://support.google.com/campaignmanager/answer/16320235?hl=ko#111]했습니다. 이전 후에는 Campaign Manager 360에서 지원 중단된 열의 값을 채우지 않고 새 열을 채우기 시작합니다. 영향을 받는 열은 다음과 같습니다.

Display & Video 360 API
Display & Video 360 커넥터용 BigQuery Data Transfer Service는 새로운 열을 지원하고 Display & Video 360 API [https://developers.google.com/display-video/api/release-notes?hl=ko]의 새로운 버전을 통해 도입되는 변경사항에 맞게 조정하기 위해 정기적으로 업데이트됩니다. Display & Video 360용 BigQuery Data Transfer Service 커넥터는 지원되는 API 버전을 사용하여 구성 데이터 [https://cloud.google.com/bigquery/docs/display-video-transfer?hl=ko#supported_configuration_data]를 가져옵니다.
다음 섹션에서는 새로운 Display & Video 360 API 버전으로 업데이트할 때 발생하는 변경사항을 간략하게 설명합니다. 변경사항은 출시일별로 정리되며 각 항목에는 Display & Video 360에서 계속 데이터를 수신하려면 취해야 할 변경사항에 관한 정보가 제공됩니다.
2025년 8월 26일
Display & Video 360 커넥터 [https://cloud.google.com/bigquery/docs/display-video-transfer?hl=ko]는 구성 데이터를 검색하는 데 사용되는 Display & Video 360 API 버전 [https://developers.google.com/display-video/api/release-notes?hl=ko]을 v3 [https://developers.google.com/display-video/api/reference/rest/v3?hl=ko]에서 v4 [https://developers.google.com/display-video/api/reference/rest/v4?hl=ko]로 업데이트할 계획입니다. API 업그레이드의 변경사항은 다음 섹션에 나와 있습니다. 자세한 내용은 Display & Video 360 API v3에서 v4로의 이전 가이드 [https://developers.google.com/display-video/api/v4-migration-guide?hl=ko]를 참조하세요.
Display & Video 360 커넥터의 이번 업데이트는 2025년 8월 26일에 시작될 예정입니다.

Google Ads API
Google Ads용 BigQuery Data Transfer Service는 새로운 열을 지원하고 Google Ads API [https://developers.google.com/google-ads/api/docs/release-notes?hl=ko]의 새로운 버전을 통해 도입되는 변경사항에 맞게 조정하기 위해 정기적으로 업데이트됩니다. Google Ads 커넥터용 BigQuery Data Transfer Service는 Google Ads 커넥터에서 지원되는 API 버전 [https://cloud.google.com/bigquery/docs/google-ads-transfer?hl=ko#supported_reports]을 사용합니다.
다음 섹션에서는 새로운 Google Ads API 버전으로 업데이트할 때 발생하는 변경사항을 간략하게 설명합니다. 변경사항은 출시일별로 정리되며 각 항목에는 Google Ads에서 계속 데이터를 수신하려면 어떤 변경사항을 적용해야 하는지 관한 정보가 제공됩니다.
Google Ads API 출시 일정에 대한 자세한 내용은 일정 [https://developers.google.com/google-ads/api/docs/sunset-dates?hl=ko#timetable]을 참조하세요.

Google Ads 이전 [https://cloud.google.com/bigquery/docs/google-ads-transfer?hl=ko]에서 Google Ads API 버전 [https://developers.google.com/google-ads/api/docs/release-notes?hl=ko]을 v18 [https://developers.google.com/google-ads/api/reference/rpc/v18/overview?hl=ko]에서 v20 [https://developers.google.com/google-ads/api/reference/rpc/v20/overview?hl=ko]으로 업데이트할 계획입니다. Google Ads API v14에서는 segments_product_category_level1 및 segments_product_category_level2 같은 새 열이 BigQuery 테이블 스키마에 추가되었지만 null로 채워졌습니다. Google Ads API v16으로 업데이트하면 이러한 새 열이 새 값으로 채워집니다. segments_product_bidding_category_level1 및 segments_product_bidding_category_level2 같은 지원 중단된 열은 null로 채워지지만 테이블 스키마에 계속 남아 있습니다.
각 열 쌍에서 하나의 열만 Google Ads API의 값으로 채워지고 다른 열은 null로 채워집니다. 업데이트 후에도 기존 쿼리가 계속 작동하도록 하려면 쿼리를 업데이트하여 두 열 중 하나를 선택하세요. 예를 들어 SQL 쿼리에 다음 문이 있다고 가정해 보겠습니다.
segments_product_bidding_category_level1
올바른 열을 지정하는 다음 문으로 바꿉니다.
IFNULL(segments_product_category_level1, segments_product_bidding_category_level1)
2024년 6월 24일 이후에 생성된 전송 구성에는 항상 새 열이 사용됩니다. 지원 중단된 열은 테이블 스키마에 계속 유지되지만 null로 채워집니다.
Google Analytics API
Google 애널리틱스 커넥터용 BigQuery Data Transfer Service는 새로운 열을 지원하고 Google Analytics Data API [https://developers.google.com/analytics/devguides/reporting/data/v1/changelog?hl=ko]의 새로운 버전을 통해 도입되는 변경사항에 맞게 조정하기 위해 정기적으로 업데이트됩니다. Google 애널리틱스 커넥터용 BigQuery Data Transfer Service는 지원되는 최신 API 버전을 사용하여 보고 데이터를 가져옵니다.
다음 섹션에서는 새 Google 애널리틱스 Data API 버전으로 업데이트할 때 발생하는 변경사항을 간략하게 설명합니다. 변경사항은 출시일별로 정리되며 각 항목에는 Google 애널리틱스에서 계속 데이터를 수신하려면 취해야 할 변경사항에 관한 정보가 제공됩니다.
2025년 9월 22일
참고: 이 변경사항은 2025년 4월 25일 이전에 Google 애널리틱스 커넥터를 사용한 사용자에게만 영향을 미칩니다.
Google 애널리틱스 커넥터 [https://cloud.google.com/bigquery/docs/google-analytics-4-transfer?hl=ko]는 Google Analytics Data API v1 [https://developers.google.com/analytics/devguides/reporting/data/v1/changelog?hl=ko]의 변경사항을 반영하기 위해 테이블을 지원 중단하고 스키마를 업데이트할 계획입니다. 이러한 변경사항은 다음 섹션에 나와 있습니다.
Google 애널리틱스 커넥터의 이 업데이트는 2025년 9월 22일에 시작될 예정입니다.

업데이트된 테이블 스키마
새 테이블 스키마는 Google 애널리틱스 보고서 변환 [https://cloud.google.com/bigquery/docs/google-analytics-4-transformation?hl=ko] 페이지에서 확인할 수 있습니다.
스키마 변경사항을 요약하면 다음과 같습니다.
수정된 스키마: 트래픽 확보, 사용자 확보, 방문 페이지 보고서의 스키마가 수정되었습니다. 예를 들어 이전에는 트래픽 확보 보고서와 사용자 확보 보고서의 스키마가 서로 바뀌어 있었고 방문 페이지 보고서에 landingPage 측정기준이 누락되었습니다.
필드 이름 변경 및 지원 중단: 현재 Google 애널리틱스 용어와 일치하도록 모든 보고서에서 전환 필드의 이름이 keyEvents로 변경됩니다. 따라서 '전환' 보고서 자체가 지원 중단됩니다.
데이터 유형 변경: API에서 반환된 부동 소수점 마이크로 값을 정확하게 나타내기 위해 BigQuery에서 수익 필드가 INTEGER에서 FLOAT로 변경됩니다.
새로운 테이블 및 필드 이름 지정 규칙: 새 테이블의 필드 이름은 Google Analytics API와 일관성을 유지하기 위해 이전의 snake_case(예: event_count) 대신 camelCase(예: eventCount)를 사용합니다.
YouTube Reporting API
YouTube 콘텐츠 소유자 커넥터 및 YouTube 채널 커넥터용 BigQuery Data Transfer Service는 YouTube Reporting API [https://developers.google.com/youtube/reporting?hl=ko]에서 도입된 새 보고서를 지원하고 이전 보고서를 지원 중단하기 위해 정기적으로 업데이트됩니다.
다음 섹션에서는 YouTube Reporting API에서 새 보고서가 도입될 때의 변경사항을 간략하게 설명합니다. 변경사항은 출시일별로 정리되며 각 항목에는 YouTube에서 계속 데이터를 수신하려면 취해야 할 변경사항에 관한 정보가 제공됩니다.
2025년 9월 22일
YouTube 콘텐츠 소유자 커넥터 [https://cloud.google.com/bigquery/docs/youtube-content-owner-transfer?hl=ko] 및 YouTube 채널 커넥터 [https://cloud.google.com/bigquery/docs/youtube-channel-transfer?hl=ko]는 YouTube Shorts 조회수 변경사항 [https://support.google.com/youtube/thread/333869549/a-change-to-how-we-count-views-on-shorts?hl=ko]을 반영하기 위해 새로운 보고서를 도입하고 기존 보고서를 지원 중단할 계획입니다. 이러한 변경사항은 다음 섹션에 나와 있습니다.
