다른 Google 및 Google Cloud 서비스에서 데이터 로드
bookmark_border
다양한 Google Cloud 서비스를 사용하여 BigQuery로 데이터를 로드한 후 추가 분석을 실행할 수 있습니다. 이러한 서비스는 일반적으로 서비스의 각 콘솔 또는 API에서 내보내기 작업을 시작해야 합니다. 사용 설정하면 서비스의 내보내기 작업에 정의된 Cadence에 따라 데이터가 BigQuery에 로드됩니다. 일부 내보내기 작업은 실시간으로 실행되고, 다른 작업은 일괄 데이터 로드를 제공합니다.
Google Drive 및 Google Sheets를 비롯한 Google Cloud 데이터베이스 및 서비스의 경우 데이터 쿼리는 BigQuery에서 시작됩니다. 자세한 내용은 외부 데이터 소스 [https://cloud.google.com/bigquery/external-data-sources?hl=ko]를 참조하세요.
서비스가 목록에 없더라도 서비스에서 데이터를 내보낼 수는 있지만 추가 기능을 사용해야 할 수 있습니다. 커스텀 내보내기를 설정하는 방법 또는 BigQuery에서 로드 작업 및 쿼리를 만드는 방법에 관한 자세한 내용은 데이터 로드 소개 [https://cloud.google.com/bigquery/docs/loading-data?hl=ko]를 참고하세요.
데이터 내보내기를 지원하는 클라우드 서비스
탄소 발자국
탄소 발자국 [https://cloud.google.com/carbon-footprint?hl=ko] 내보내기를 수행하면 선택한 청구 계정의 Google Cloud 서비스 사용과 관련된 총 추정 온실가스 배출량이 캡처됩니다.
데이터 분석을 수행하거나 커스텀 대시보드 및 보고서를 만들기 위해 탄소 발자국 데이터를 BigQuery로 내보낼 수 있습니다.
탄소 발자국 데이터 내보내기를 설정하려면 탄소 발자국 내보내기 [https://cloud.google.com/carbon-footprint/docs/export?hl=ko]를 참고하세요.
Google Security Operations
Google Security Operations [https://chronicle.security/] 보안 로그를 BigQuery로 내보내 추가 데이터 조인 및 분석을 수행할 수 있습니다.
Google Security Operations 보안 로그의 내보내기를 설정하려면 Google Security Operations 지원팀 [https://cloud.google.com/chronicle/docs/getting-support?hl=ko]에 문의하여 이를 설정하세요.
Cloud 애셋 인벤토리
Cloud 애셋 인벤토리 [https://cloud.google.com/asset-inventory?hl=ko]를 사용하면 조직, 폴더 또는 프로젝트의 애셋 메타데이터를 BigQuery 테이블로 내보낸 다음 인벤토리에 대한 데이터 분석을 실행할 수 있습니다.
Cloud 애셋 인벤토리 데이터 내보내기를 설정하려면 BigQuery로 내보내기 [https://cloud.google.com/asset-inventory/docs/exporting-to-bigquery?hl=ko]를 참고하세요.
Cloud Billing
Cloud Billing [https://cloud.google.com/billing?hl=ko] BigQuery로 내보내기를 사용하면 자세한Google Cloud 결제 데이터(예: 사용량, 예상 비용, 가격 데이터)를 하루 동안 자동으로 내보낼 수 있습니다.
타이밍이 중요합니다. 분석 요구에 맞게 보다 포괄적인 Google Cloud 결제 데이터 세트에 액세스하려면 Cloud Billing 계정을 만들 때 동시에 BigQuery로 Cloud Billing 데이터 내보내기를 사용 설정하는 것이 좋습니다.
Cloud Billing 데이터 내보내기를 설정하려면 BigQuery로 Cloud Billing 데이터 내보내기 [https://cloud.google.com/billing/docs/how-to/export-data-bigquery?hl=ko]를 참고하세요.
Cloud Logging
추가 분석 및 조인을 위해 Cloud Logging [https://cloud.google.com/logging?hl=ko]에서 BigQuery 테이블로 로그를 라우팅할 수 있습니다. Google Cloud 서비스의 경우 로그 데이터가 생성된 후 약 1분 후에 쿼리할 수 있습니다.
BigQuery를 로그 애널리틱스의 일부로 사용하려면 로그 애널리틱스 [https://cloud.google.com/logging/docs/analyze/query-and-view?hl=ko]를 참고하세요.
Cloud Logging 데이터 내보내기를 설정하려면 지원되는 싱크로 로그 라우팅하기 [https://cloud.google.com/logging/docs/export/configure_export_v2?hl=ko]를 참고하세요.
대화형 인사이트
대화형 인사이트 [https://cloud.google.com/solutions/ccai-insights?hl=ko]를 사용하면 자체 원시 쿼리를 수행할 수 있도록 대화형 인사이트 대화 및 분석 데이터를 BigQuery로 내보낼 수 있습니다.
대화형 인사이트 데이터 내보내기를 구성하려면 BigQuery로 대화 내보내기 [https://cloud.google.com/contact-center/insights/docs/export?hl=ko]를 참고하세요.
Dialogflow CX
Dialogflow CX [https://cloud.google.com/dialogflow?hl=ko]는 상담사와 고객 간의 대화 로그를 생성합니다.
Dialogflow CX에서 대화 내보내기를 구성하려면 BigQuery로 상호작용 로깅 내보내기 [https://cloud.google.com/dialogflow/cx/docs/concept/export-bq?hl=ko]를 참고하세요.
Firebase
Firebase [https://firebase.google.com/?hl=ko]에는 BigQuery로 전송할 수 있는 여러 애널리틱스 내보내기가 포함되어 있습니다. 예를 들면 다음과 같습니다.
애널리틱스
클라우드 메시징
Crashlytics
성능 모니터링
A/B 테스팅
원격 구성 맞춤설정
Firebase 데이터 내보내기를 구성하려면 BigQuery에 프로젝트 데이터 내보내기 [https://firebase.google.com/docs/projects/bigquery-export?hl=ko]를 참고하세요.
Google 애널리틱스 4
Google 애널리틱스 4 [https://developers.google.com/analytics/devguides/collection/ga4?hl=ko] 보고서 보기에서 BigQuery로 세션 데이터를 내보내는 방법을 알아보려면 애널리틱스 고객센터에서 BigQuery 내보내기 [https://support.google.com/analytics/answer/9358801?hl=ko], BigQuery 설정 내보내기 [https://support.google.com/analytics/answer/3416092?hl=ko]를 참조하세요. Google 애널리틱스 4 데이터가 BigQuery에 있으면 GoogleSQL을 사용하여 쿼리할 수 있습니다.
Google 애널리틱스 4 내보내기에 오류가 발생하면 연결 실패 이유 [https://support.google.com/analytics/answer/9823238?hl=ko#reasons&zippy=,in-this-article] 또는 내보내기 실패 이유 [https://support.google.com/analytics/answer/9823238?hl=ko#export&zippy=,in-this-article]를 참고하세요.
Google 애널리틱스 4 내보내기에서 데이터 지연이 발생하거나 내보내기 후 Google 애널리틱스 4 데이터가 누락되는 문제 또는 기타 문제가 발생하면 애널리틱스 지원팀 [https://support.google.com/analytics/?hl=ko]에 문의하세요.
결제와 같은 BigQuery 관련 문제는 Google Cloud 지원팀 [https://cloud.google.com/support/?hl=ko]에 문의하세요.
Google 애널리틱스 360
Google 애널리틱스 360 [https://marketingplatform.google.com/about/analytics-360/?hl=ko] 보고서 보기에서 BigQuery로 세션과 히트 데이터를 내보내는 방법을 알아보려면 애널리틱스 고객센터의 BigQuery 내보내기 [https://support.google.com/analytics/topic/3416089?hl=ko]를 참조하세요. Google 애널리틱스 360 데이터가 BigQuery에 있으면 GoogleSQL을 사용하여 쿼리할 수 있습니다.
BigQuery의 애널리틱스 데이터를 쿼리하는 예시에 대해서는 애널리틱스 도움말의 BigQuery 설명서 [https://support.google.com/analytics/answer/4419694?hl=ko]를 참조하세요.
BigQuery와 Google 애널리틱스 360 연결과 관련된 문제, 내보내기 후 Google 애널리틱스 360 데이터가 누락되는 문제 또는 Google 애널리틱스 360에서 내보낼 때 데이터 지연과 관련된 문제는 Google 애널리틱스 360 지원팀에 문의하세요. [https://support.google.com/analytics/answer/9026876?hl=ko]
결제와 같은 BigQuery 관련 문제는 Google Cloud 지원팀 [https://cloud.google.com/support/?hl=ko]에 문의하세요.
Google Search Console 데이터
Google Search [https://www.google.com/search/about/?hl=ko] Console 성능 데이터를 BigQuery로 매일 내보내는 일정을 예약하여 데이터에 대해 복잡한 쿼리를 실행할 수 있습니다.
데이터 내보내기를 설정하려면 BigQuery에 Search Console 데이터 대량 내보내기 정보 [https://support.google.com/webmasters/answer/12918484?hl=ko].
추천자
BigQuery Data Transfer Service를 사용해서 권장사항 [https://cloud.google.com/recommender/docs/whatis-activeassist?hl=ko]에 대한 일일 스냅샷을 예약할 수 있습니다. 권장사항은 Google Cloud 제품 및 리소스 사용 최적화에 대한 조언을 제공하고 리소스 사용 패턴에 대한 인사이트를 제공합니다.
BigQuery Data Transfer Service를 사용하여 데이터 스냅샷을 설정하려면 BigQuery에 권장사항 내보내기 [https://cloud.google.com/recommender/docs/bq-export/export-recommendations-to-bq?hl=ko]를 참조하세요.
Vertex AI 일괄 예측
Vertex AI [https://cloud.google.com/vertex-ai?hl=ko] 일괄 예측은 모델에 대한 입력을 기반으로 일련의 예측을 생성합니다. 이러한 결과를 BigQuery에 저장하여 추가 분석 및 조인을 수행할 수 있습니다.
일괄 예측 결과 내보내기를 구성하려면 일괄 예측 및 설명 가져오기 [https://cloud.google.com/vertex-ai/docs/tabular-data/classification-regression/get-batch-predictions?hl=ko#make-batch-request]를 참고하세요.
Vertex AI 예측
Vertex AI 예측을 사용하여 추가 분석을 위해 BigQuery에 온라인 엔드포인트의 예측 결과를 저장할 수 있습니다.
BigQuery와의 모델 예측 통합을 구성하려면 온라인 예측 로깅 [https://cloud.google.com/vertex-ai/docs/predictions/online-prediction-logging?hl=ko#enabling-and-disabling]을 참고하세요.
