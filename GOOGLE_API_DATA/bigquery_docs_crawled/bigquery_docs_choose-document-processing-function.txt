문서 처리 함수 선택
bookmark_border
이 문서에서는 BigQuery ML에서 사용할 수 있는 문서 처리 함수(ML.GENERATE_TEXT [https://cloud.google.com/bigquery/docs/reference/standard-sql/bigqueryml-syntax-generate-text?hl=ko], ML.PROCESS_DOCUMENT [https://cloud.google.com/bigquery/docs/reference/standard-sql/bigqueryml-syntax-process-document?hl=ko])를 비교합니다.
이 문서의 정보를 사용하여 함수에 겹치는 기능이 있는 경우에 사용할 함수를 결정할 수 있습니다.
간략한 이러한 함수의 차이점은 다음과 같습니다.
ML.GENERATE_TEXT는 일부 콘텐츠가 문서에 있는 자연어 처리(NLP) 태스크를 수행하는 데 적합합니다. 이 함수에는 다음과 같은 이점이 있습니다.
비용 절감
더 많은 언어 지원
처리량 증가
모델 튜닝 기능
멀티모달 모델 사용 가능
이 접근 방식이 가장 효과적인 문서 처리 태스크의 예는 Gemini API를 사용한 문서 처리 기능 살펴보기 [https://ai.google.dev/gemini-api/docs/document-processing?hl=ko]를 참조하세요.
ML.PROCESS_DOCUMENT는 문서 파싱 및 사전 정의된 구조화된 응답이 필요한 문서 처리 태스크를 수행하는 데 적합합니다.
지원되는 모델
지원되는 모델은 다음과 같습니다.
ML.GENERATE_TEXT: Vertex AI Gemini [https://cloud.google.com/vertex-ai/generative-ai/docs/learn/models?hl=ko#gemini-models] 모델의 하위 집합을 사용하여 텍스트를 생성할 수 있습니다. 지원되는 모델에 대한 자세한 내용은 ML.GENERATE_TEXT 구문 [https://cloud.google.com/bigquery/docs/reference/standard-sql/bigqueryml-syntax-generate-text?hl=ko#syntax]을 참조하세요.
ML.PROCESS_DOCUMENT: Document AI API [https://cloud.google.com/document-ai?hl=ko] 기본 모델을 사용합니다. Document AI API를 사용하면 인보이스 파서, 레이아웃 파서, 양식 파서와 같은 다양한 문서 프로세서에 액세스할 수 있습니다. 이러한 문서 프로세서를 사용하여 다양한 구조의 PDF 파일을 처리할 수 있습니다.
지원되는 태스크
지원되는 태스크는 다음과 같습니다.
ML.GENERATE_TEXT: 입력이 문서인 모든 NLP 태스크를 수행할 수 있습니다. 회사의 재무 문서를 예로 들면 What is the quarterly revenue for each division?과 같은 프롬프트를 제공하여 문서 정보를 검색할 수 있습니다.
ML.PROCESS_DOCUMENT: 인보이스, 세금 양식, 재무제표와 같은 다양한 문서 유형에 대한 특수 문서 처리를 수행할 수 있습니다. 문서 청크 처리도 수행할 수 있습니다. 이 태스크에 ML.PROCESS_DOCUMENT 함수를 사용하는 방법에 관한 자세한 내용은 검색 증강 생성 파이프라인에서 PDF 파싱 [https://cloud.google.com/bigquery/docs/rag-pipeline-pdf?hl=ko]을 참조하세요.
가격 책정
가격은 다음과 같습니다.
ML.GENERATE_TEXT: 이 함수와 함께 사용하는 Vertex AI 모델의 가격은 Vertex AI 가격 책정 [https://cloud.google.com/vertex-ai/generative-ai/pricing?hl=ko]을 참조하세요. 지원되는 모델의 지도 조정은 노드 시간당 달러로 청구됩니다. 자세한 내용은 Vertex AI 커스텀 학습 가격 책정 [https://cloud.google.com/vertex-ai/pricing?hl=ko#custom-trained_models]을 참조하세요.
ML.PROCESS_DOCUMENT: 이 함수와 함께 사용하는 Cloud AI 서비스의 가격은 Document AI API 가격 책정 [https://cloud.google.com/document-ai/pricing?hl=ko]을 참조하세요.
지도 조정
지도 조정 지원은 다음과 같습니다.
ML.GENERATE_TEXT: 일부 모델에서 지도 조정 [https://cloud.google.com/bigquery/docs/reference/standard-sql/bigqueryml-syntax-create-remote-model?hl=ko#supervised_tuning]이 지원됩니다.
ML.PROCESS_DOCUMENT: 지도 조정이 지원되지 않습니다.
분당 쿼리 수(QPM) 한도
QPM 한도는 다음과 같습니다.
ML.GENERATE_TEXT: gemini-1.5-pro 모델의 경우 기본 us-central1 리전에서 60QPM, gemini-1.5-flash 모델의 경우 기본 us-central1 리전에서 200QPM이 지원됩니다. 자세한 내용은 Vertex AI의 생성형 AI 할당량 [https://cloud.google.com/vertex-ai/generative-ai/docs/quotas?hl=ko]을 참조하세요.
ML.PROCESS_DOCUMENT: 프로세서 유형당 120QPM이며 프로젝트당 전체 한도는 600QPM입니다. 자세한 내용은 할당량 목록 [https://cloud.google.com/document-ai/quotas?hl=ko#quotas_list]을 참조하세요.
할당량을 늘리려면 할당량 조정 요청 [https://cloud.google.com/docs/quotas/help/request_increase?hl=ko]을 참조하세요.
토큰 한도
토큰 한도는 다음과 같습니다.
ML.GENERATE_TEXT: 입력 토큰 700개, 출력 토큰 8,196개가 지원됩니다.
ML.PROCESS_DOCUMENT: 토큰 한도가 없습니다. 하지만 이 함수는 사용하는 프로세서에 따라 페이지 한도가 다릅니다. 자세한 내용은 한도 [https://cloud.google.com/document-ai/limits?hl=ko]를 참조하세요.
지원 언어
지원되는 언어는 다음과 같습니다.
ML.GENERATE_TEXT: Gemini [https://cloud.google.com/vertex-ai/generative-ai/docs/learn/models?hl=ko#languages-gemini]와 동일한 언어를 지원합니다.
ML.PROCESS_DOCUMENT: 언어 지원은 문서 프로세서 유형에 따라 다릅니다. 대부분은 영어만 지원합니다. 자세한 내용은 프로세서 목록 [https://cloud.google.com/document-ai/docs/processors-list?hl=ko]을 참조하세요.
사용 가능한 리전
사용 가능한 리전은 다음과 같습니다.
ML.GENERATE_TEXT: 모든 Vertex AI용 생성형 AI 리전 [https://cloud.google.com/vertex-ai/generative-ai/docs/learn/locations?hl=ko#available-regions]에서 사용할 수 있습니다.
ML.PROCESS_DOCUMENT: 모든 프로세서의 EU 및 US 멀티 리전 [https://cloud.google.com/bigquery/docs/locations?hl=ko#multi-regions]에서 사용할 수 있습니다. 일부 프로세서는 특정 리전에만 제공됩니다. 자세한 내용은 리전 및 멀티 리전 지원 [https://cloud.google.com/document-ai/docs/regions?hl=ko]을 참조하세요.
