BigQuery Sharing 소개
bookmark_border
BigQuery Sharing(이전 명칭: Analytics Hub)는 강력한 보안 및 개인 정보 보호 프레임워크로 조직 경계에서 규모에 맞게 데이터와 통계를 공유할 수 있도록 하는 데이터 교환 플랫폼입니다. 공유를 사용하면 다양한 데이터 제공업체가 선별한 데이터 라이브러리를 찾아 액세스할 수 있습니다. 이 데이터 라이브러리에는 Google 제공 데이터 세트도 포함되어 있습니다.
예를 들어 공유를 사용하여 서드 파티 및 Google 데이터 세트로 분석 및 ML 이니셔티브를 보강할 수 있습니다.
Analytics Hub Identity and Access Management(IAM) 역할을 사용하면 다음 공유 작업을 실행할 수 있습니다.
Analytics Hub 게시자는 데이터를 파트너 네트워크와 공유하거나 조직 내부에 실시간으로 공유할 수 있습니다. 목록 [https://cloud.google.com/bigquery/docs/analytics-hub-introduction?hl=ko#listings]을 사용하면 공유 데이터를 복제하지 않고도 데이터를 공유할 수 있으며, Google Cloud Marketplace [https://cloud.google.com/bigquery/docs/analytics-hub-cloud-marketplace?hl=ko] 또는 자체 채널을 통해 수익을 창출할 수 있습니다. 적합한 대상에게 데이터를 전달할 수 있는 세분화된 권한으로 분석용 데이터 소스의 카탈로그를 빌드할 수 있습니다. 또한 구독을 관리하고 목록의 사용량 측정항목을 볼 수 있습니다.
Analytics Hub 구독자는 원하는 데이터를 탐색하고, 공유 데이터를 기존 데이터와 결합하고, BigQuery의 기본 제공 기능 [https://cloud.google.com/bigquery/docs/introduction?hl=ko#explore-bigquery]을 사용할 수 있습니다. 목록을 구독하면 프로젝트에 연결된 데이터 세트 [https://cloud.google.com/bigquery/docs/analytics-hub-introduction?hl=ko#linked_datasets] 또는 연결된 Pub/Sub 구독이 생성됩니다. 구독자에 대한 관련 정보를 저장하고 게시자와 구독자 사이의 연결을 나타내는 구독 리소스 [https://cloud.google.com/bigquery/docs/reference/analytics-hub/rest/v1/projects.locations.subscriptions?hl=ko]를 사용해서 구독을 관리할 수 있습니다.
Analytics Hub 뷰어는 공유에서 액세스할 수 있는 공유 리소스를 찾아보고 공유 데이터에 액세스하기 위해 게시자에게 요청할 수 있습니다. 공유 및 Cloud Marketplace에서 Cloud Marketplace 통합 상용 등록정보를 확인할 수 있습니다.
Analytics Hub 관리자는 데이터 공유를 사용 설정하는 데이터 교환 [https://cloud.google.com/bigquery/docs/analytics-hub-introduction?hl=ko#data_exchanges]을 생성한 다음 데이터 게시자와 구독자에게 이러한 데이터 교환에 액세스할 수 있는 권한을 부여할 수 있습니다.
자세한 내용은 Analytics Hub 역할 구성 [https://cloud.google.com/bigquery/docs/analytics-hub-grant-roles?hl=ko]을 참고하세요.

공유 데이터 세트
공유 데이터 세트는 BigQuery Sharing의 데이터 공유 단위인 BigQuery 데이터 세트입니다. BigQuery 아키텍처에서 컴퓨팅과 스토리지를 분리하기 때문에 데이터 게시자는 데이터의 여러 복사본을 만들 필요 없이 원하는 만큼 많은 구독자와 데이터 세트를 공유할 수 있습니다. 게시자는 다음과 같은 지원 객체를 구독자에게 전달하도록 프로젝트에서 기존 BigQuery 데이터 세트를 만들거나 사용합니다.

VPC 서비스 제어 및 공유와 관련된 다음 제한사항에 유의하세요.
게시자는 VPC 서비스 제어 경계 내의 프로젝트에 공유 데이터를 게시하지 않는 것이 좋습니다. 프로젝트의 공유 데이터 세트가 VPC 서비스 제어 경계 내에 있는 경우 교환 프로젝트(호스팅 목록)와 모든 구독자 프로젝트가 게시자의 목록을 성공적으로 구독하려면 적절한 인그레스 및 이그레스 규칙 [https://cloud.google.com/vpc-service-controls/docs/ingress-egress-rules?hl=ko]이 필요합니다.
교환 관리자는 교환 프로젝트를 VPC 서비스 제어 경계에 두지 않는 것이 좋습니다. 게시 워크플로가 중단될 수 있기 때문입니다. 게시자 프로젝트와 모든 구독자 프로젝트가 등록정보를 성공적으로 구독하려면 인그레스 및 이그레스 규칙 [https://cloud.google.com/vpc-service-controls/docs/ingress-egress-rules?hl=ko]이 필요합니다.
공유 주제(미리보기 [https://cloud.google.com/products?hl=ko#product-launch-stages])
공유 주제는 BigQuery의 스트리밍 데이터 공유 [https://cloud.google.com/bigquery/docs/analytics-hub-stream-sharing?hl=ko] 단위인 Pub/Sub 주제 [https://cloud.google.com/pubsub/docs/create-topic?hl=ko]입니다. 게시자는 프로젝트에서 Pub/Sub 주제를 만들거나 기존 Pub/Sub 주제를 사용하고 구독자에게 배포합니다.
데이터 교환
데이터 교환은 셀프서비스 데이터 공유를 사용 설정하는 컨테이너입니다. 여기에는 공유 리소스를 참조하는 목록이 포함됩니다. 게시자와 관리자는 교환 및 목록 수준에서 구독자에게 액세스 권한을 부여할 수 있습니다. 이 방법은 기본 공유 리소스에 대한 액세스 권한을 명시적으로 부여하지 않도록 도와줍니다. 구독자는 데이터 교환을 탐색하고, 액세스할 수 있는 데이터를 검색하고, 공유 리소스를 구독할 수 있습니다. 데이터 익스체인지를 만드는 [https://cloud.google.com/bigquery/docs/analytics-hub-manage-exchanges?hl=ko#create-exchange] 경우 기본 연락처 이메일을 할당할 수 있습니다. 기본 연락처 이메일은 사용자가 데이터 교환에 관한 질문이나 우려사항이 있을 때 데이터 교환 소유자에게 연락할 수 있는 방법을 제공합니다. 데이터 교환의 유형은 다음과 같습니다.
비공개 데이터 교환. 기본적으로 데이터 교환은 비공개이며 해당 교환에 액세스할 수 있는 사용자 또는 그룹만 목록을 보거나 구독할 수 있습니다.
공개 데이터 교환. 기본적으로 데이터 교환은 비공개이며 해당 교환에 액세스할 수 있는 사용자 또는 그룹만 목록을 보거나 구독할 수 있습니다. 하지만 데이터 교환을 공개로 설정할 수 있습니다. 공개 데이터 교환의 목록은 Google Cloud 사용자(allAuthenticatedUsers) [https://cloud.google.com/iam/docs/principals-overview?hl=ko#all-authenticated-users]가 검색 [https://cloud.google.com/bigquery/docs/analytics-hub-view-subscribe-listings?hl=ko#discover-listings]하고 구독 [https://cloud.google.com/bigquery/docs/analytics-hub-view-subscribe-listings?hl=ko#subscribe-listings]할 수 있습니다. 공개 데이터 교환에 대한 자세한 내용은 데이터 교환을 공개로 설정 [https://cloud.google.com/bigquery/docs/analytics-hub-manage-exchanges?hl=ko#make-data-exchange-public]을 참고하세요.
Analytics Hub 관리자 역할을 사용하면 여러 데이터 교환을 만들고 공유 작업을 수행하는 다른 사용자를 관리할 수 있습니다.
목록
목록은 게시자가 데이터 교환에 나열하는 공유 리소스를 참조합니다. 게시자는 목록을 만들고 리소스 설명, 실행할 샘플 쿼리 또는 샘플 메시지 데이터, 관련 문서 링크, 구독자가 공유 리소스를 사용하는 데 도움이 되는 추가 정보를 지정할 수 있습니다. 목록을 만들 때 기본 연락처 이메일, 제공자 이름 및 연락처, 게시자 이름 및 연락처를 할당할 수 있습니다. 기본 연락처 이메일은 사용자가 데이터 교환에 관한 질문이나 우려사항이 있을 때 목록 소유자에게 연락할 수 있는 방법을 제공합니다. 제공자 이름과 연락처는 원래 목록의 데이터를 제공한 기관의 정보입니다. 이 정보는 선택사항입니다. 게시자 이름과 연락처는 BigQuery Sharing에 사용할 데이터를 게시한 기관입니다. 이 정보는 선택사항입니다. 자세한 내용은 목록 관리 [https://cloud.google.com/bigquery/docs/analytics-hub-manage-listings?hl=ko]를 참조하세요.
목록은 목록에 설정된 Identity and Access Management(IAM) 정책과 목록을 포함하는 데이터 교환의 유형에 따라 다음 두 가지 유형 중 하나일 수 있습니다.
공개 목록. 모든 Google Cloud 사용자(allAuthenticatedUsers) [https://cloud.google.com/iam/docs/principals-overview?hl=ko#all-authenticated-users]와 공유됩니다. 공개 데이터 교환의 목록은 공개 목록입니다. 이 목록은 무료 공개 리소스 또는 상용 리소스의 참조일 수 있습니다. 목록이 상용 리소스인 경우 구독자는 데이터 제공업체에 직접 목록에 대한 액세스를 요청하거나 Google Cloud Marketplace 통합 상용 목록 [https://cloud.google.com/bigquery/docs/analytics-hub-cloud-marketplace?hl=ko]을 탐색하고 구매할 수 있습니다.
비공개 목록. 개인 또는 그룹과 직접 공유됩니다. 예를 들어 비공개 목록은 조직의 다른 내부 팀과 공유하는 마케팅 측정항목 데이터 세트를 참조할 수 있습니다.

연결된 리소스
연결된 리소스는 공유 목록을 구독할 때 생성되며, 구독자를 기본 공유 리소스에 연결합니다.
연결된 데이터 세트
연결된 데이터 세트는 공유 데이터 세트에 대한 포인터 또는 참조 역할을 하는 읽기 전용 BigQuery 데이터 세트입니다. 목록을 구독하면 데이터 세트의 사본이 아닌 프로젝트에 연결된 데이터 세트가 생성되므로 구독자는 데이터를 읽을 수 있지만 그 안에 객체를 추가하거나 업데이트할 수 없습니다. 연결된 데이터 세트를 통해 테이블 및 뷰와 같은 객체를 쿼리하면 공유 데이터 세트의 데이터가 반환됩니다. 연결된 데이터 세트에 대한 자세한 내용은 목록 보기 및 구독 [https://cloud.google.com/bigquery/docs/analytics-hub-view-subscribe-listings?hl=ko]을 참조하세요.
연결된 데이터 세트는 공유 데이터 세트의 테이블과 뷰에 액세스할 수 있는 권한이 있습니다. 연결된 데이터 세트가 있는 구독자는 추가 Identity and Access Management 승인 없이 공유 데이터 세트의 테이블과 뷰에 액세스합니다.
연결된 데이터 세트는 다음 객체를 지원합니다.
승인된 뷰 [https://cloud.google.com/bigquery/docs/authorized-views?hl=ko]
승인된 데이터 세트 [https://cloud.google.com/bigquery/docs/authorized-datasets?hl=ko]
승인된 루틴 [https://cloud.google.com/bigquery/docs/authorized-routines?hl=ko]
연결된 Pub/Sub 구독(미리보기 [https://cloud.google.com/products?hl=ko#product-launch-stages])
공유 주제가 있는 목록을 구독하면 구독자 프로젝트에 연결된 Pub/Sub 구독이 생성됩니다. 공유 주제 또는 메시지 데이터의 사본은 생성되지 않습니다. 연결된 Pub/Sub 구독 [https://cloud.google.com/pubsub/docs/subscription-overview?hl=ko]의 구독자는 공유 주제에 게시된 메시지에 액세스할 수 있습니다. 구독자는 추가 Identity and Access Management 승인 없이 공유 주제의 메시지 데이터에 액세스합니다. 게시자는 Pub/Sub에서 직접 또는 공유 구독 관리를 통해 구독을 관리할 수 있습니다. 연결된 Pub/Sub 구독에 대한 자세한 내용은 Pub/Sub로 스트림 공유 [https://cloud.google.com/bigquery/docs/analytics-hub-stream-sharing?hl=ko]를 참고하세요.
데이터 이그레스 옵션(BigQuery 공유 데이터 세트만 해당)
데이터 이그레스 옵션을 사용하면 게시자가 BigQuery 연결 데이터 세트 외부의 데이터 구독자에 따라 내보내기를 제한할 수 있습니다.
게시자는 목록, 쿼리 결과 또는 둘 다에 데이터 이그레스 제한을 사용 설정할 수 있습니다. 데이터 이그레스가 제한된 경우 다음 제한이 적용됩니다.
복사, 클론, 내보내기, 스냅샷 API가 사용 중지됩니다.
Google Cloud 콘솔에서 복사, 클론, 내보내기, 스냅샷 옵션이 사용 중지됩니다.
테이블 탐색기에 대한 제한된 데이터 세트 연결이 사용 중지됩니다.
BigQuery Data Transfer Service가 제한된 데이터 세트에서 사용 중지됩니다.
CREATE TABLE AS SELECT 문 [https://cloud.google.com/bigquery/docs/reference/standard-sql/data-definition-language?hl=ko#create_table_statement] 및 대상 테이블에 쓰기 [https://cloud.google.com/bigquery/docs/writing-results?hl=ko]가 사용 중지됩니다.
CREATE VIEW AS SELECT 문 [https://cloud.google.com/bigquery/docs/reference/standard-sql/data-definition-language?hl=ko#create_view_statement] 및 대상 뷰에 쓰기가 사용 중지됩니다.
목록을 만들 때 [https://cloud.google.com/bigquery/docs/analytics-hub-manage-listings?hl=ko#create_a_listing] 적합한 데이터 이그레스 옵션을 설정할 수 있습니다.
제한사항
공유에는 다음과 같은 제한사항이 있습니다.
공유 데이터 세트에는 최대 1,000개의 연결된 데이터 세트가 있을 수 있습니다.
공유 주제는 최대 [https://cloud.google.com/pubsub/quotas?hl=ko#resource_limits] 10,000개의 Pub/Sub 구독을 포함할 수 있습니다. 이 한도에는 연결된 Pub/Sub 구독과 공유 외부에서 생성된 Pub/Sub 구독(예: Pub/Sub에서 직접)이 포함됩니다.
목록을 만들 때 [https://cloud.google.com/bigquery/docs/analytics-hub-manage-listings?hl=ko#create_a_listing] 지원되지 않는 리소스가 있는 데이터 세트는 공유 데이터 세트로 선택할 수 없습니다. 공유가 지원하는 BigQuery 객체에 대한 자세한 내용은 이 문서의 공유 데이터 세트 [https://cloud.google.com/bigquery/docs/analytics-hub-introduction?hl=ko#shared_datasets]를 참고하세요.
연결된 데이터 세트 내 개별 테이블에 대한 IAM 역할 [https://cloud.google.com/bigquery/docs/access-control?hl=ko] 또는 IAM 정책 [https://cloud.google.com/config-connector/docs/reference/resource-docs/iam/iampolicy?hl=ko]을 설정할 수 없습니다. 대신 연결된 데이터 세트 수준에서 적용하세요.
연결된 데이터 세트 내 테이블에 IAM 태그 [https://cloud.google.com/bigquery/docs/tags?hl=ko]를 연결할 수 없습니다. 대신 연결된 데이터 세트 수준에서 적용하세요.
2023년 7월 25일 이전에 생성된 연결된 데이터 세트는 구독 리소스 [https://cloud.google.com/bigquery/docs/analytics-hub-manage-subscriptions?hl=ko]로 백필되지 않습니다. 2023년 7월 25일 이후에 생성된 구독만 API 메서드가 작동합니다.
게시자라면 다음 BigQuery 상호 운용성 제한사항이 적용됩니다.
구독자가 연결된 데이터 세트 내에서 뷰를 쿼리할 수 있게 하려면 구독자에게 소스 데이터 세트를 읽을 수 있는 명시적 권한을 부여해야 합니다. 뷰에 대한 액세스 권한을 부여하려면 게시자가 승인된 뷰를 만드는 [https://cloud.google.com/bigquery/docs/share-access-views?hl=ko] 것이 좋습니다. 승인된 뷰는 구독자에게 기본 소스 데이터에 대한 액세스 권한을 부여하지 않고도 뷰 데이터에 대한 액세스 권한을 부여할 수 있습니다.
쿼리 계획 [https://cloud.google.com/bigquery/docs/query-plan-explanation?hl=ko]에는 프로젝트 ID 및 승인된 뷰에 포함된 기타 데이터 세트를 포함하여 공유된 뷰 쿼리 및 루틴 쿼리가 표시됩니다. 민감한 것으로 간주되는 암호화 키와 같은 항목은 공유 뷰 또는 루틴 쿼리에 포함하지 마세요.
공유 데이터 세트는 Data Catalog [https://cloud.google.com/data-catalog/docs/concepts/overview?hl=ko](지원 중단됨) 및 Dataplex Universal Catalog [https://cloud.google.com/dataplex/docs/catalog-overview?hl=ko]에서 색인이 생성됩니다. 테이블 또는 뷰 추가와 같은 공유 데이터 세트에 대한 업데이트는 지연 없이 구독자에게 제공됩니다. 하지만 특정 시나리오에서 예를 들어 공유 데이터 세트에 구독자나 테이블이 100개 넘게 있는 경우 업데이트가 해당 서비스에서 색인을 생성하는 데 최대 18시간이 걸릴 수 있습니다. 색인 생성이 지연되므로 구독자는 Google Cloud 콘솔에서 업데이트된 이러한 리소스를 즉시 검색할 수 없습니다.
공유 주제는 Data Catalog(지원 중단됨) 및 Dataplex Universal Catalog에서 색인이 생성되지만 해당 리소스 유형을 특별히 필터링할 수는 없습니다.
나열된 테이블에 행 수준 보안 [https://cloud.google.com/bigquery/docs/row-level-security-intro?hl=ko] 또는 데이터 마스킹 [https://cloud.google.com/bigquery/docs/column-data-masking-intro?hl=ko] 정책을 설정한 경우 구독자는 Enterprise 또는 연결된 데이터 세트에서 쿼리 작업을 실행하기 위한 Enterprise Plus 고객이어야 합니다. 버전에 대한 자세한 내용은 BigQuery 버전 소개 [https://cloud.google.com/bigquery/docs/editions-intro?hl=ko]를 참조하세요.
구독자라면 다음과 같은 BigQuery 상호 운용성 제한사항이 적용됩니다.
연결된 데이터 세트의 테이블을 참조하는 구체화된 뷰는 지원되지 않습니다.
연결된 데이터 세트 테이블의 스냅샷 [https://cloud.google.com/bigquery/docs/table-snapshots-intro?hl=ko] 생성은 지원되지 않습니다.
연결된 데이터 세트와 JOIN 문이 1TB(물리적 스토리지)보다 큰 쿼리는 실패할 수 있습니다. 이 문제를 해결하려면 지원팀에 문의 [https://cloud.google.com/bigquery/docs/getting-support?hl=ko]하세요.
INFORMATION_SCHEMA 뷰에서 리전 한정자 [https://cloud.google.com/bigquery/docs/information-schema-intro?hl=ko#region_qualifier]를 사용하여 연결된 데이터 세트의 테이블 메타데이터를 볼 [https://cloud.google.com/bigquery/docs/analytics-hub-view-subscribe-listings?hl=ko#view-table-metadata] 수 없습니다.
연결된 데이터 세트에서 루틴 [https://cloud.google.com/bigquery/docs/routines?hl=ko]을 쿼리할 때는 사용자 정의 함수 [https://cloud.google.com/bigquery/docs/user-defined-functions?hl=ko](SQL 및 JavaScript UDF 모두) 및 테이블 함수 [https://cloud.google.com/bigquery/docs/table-functions?hl=ko] 루틴 유형만 쿼리할 수 있습니다. 지원되지 않는 루틴 유형을 쿼리하면 다음 오류 메시지가 표시됩니다. Querying routine type type is not yet supported on linked dataset dataset.
주의: 게시자는 지원되지 않는 루틴과 데이터 세트를 공유할 때 주의해야 합니다. 지원되지 않는 루틴은 이후에 지원을 받으면서 지원되지 않는 루틴에서 이전에 비활성화된 쿼리가 작동할 수 있습니다. 구독자에게 공유하려는 루틴만 공유 데이터 세트에 포함해야 합니다.
사용량 측정항목에는 다음과 같은 제한사항이 적용됩니다.
2023년 7월 20일 이전에 구독된 목록에 대해서는 사용량 측정항목을 가져올 수 없습니다.
num_rows_processed 및 total_bytes_processed 필드의 외부 테이블 [https://cloud.google.com/bigquery/docs/external-tables?hl=ko] 사용 측정항목은 부정확한 데이터를 포함할 수 있습니다.
소비에 대한 사용량 측정항목은 BigQuery 작업 [https://cloud.google.com/bigquery/docs/managing-jobs?hl=ko]을 사용한 사용량에만 지원됩니다. 다음 리소스를 사용하는 소비는 지원되지 않습니다.
BigQuery Storage Read API [https://cloud.google.com/bigquery/docs/reference/storage?hl=ko#read_from_a_session_stream]
tabledata.list [https://cloud.google.com/bigquery/docs/reference/rest/v2/tabledata/list?hl=ko]
BigQuery BI Engine 쿼리 [https://cloud.google.com/bigquery/docs/bi-engine-intro?hl=ko]
뷰 [https://cloud.google.com/bigquery/docs/views-intro?hl=ko]의 사용량 측정항목은 2024년 4월 22일 이후의 쿼리에 대해서만 채워집니다.
BigQuery에 연결된 Pub/Sub 구독에 대한 사용량 측정항목은 캡처되지 않습니다(Pub/Sub에서 직접 사용량을 계속 확인할 수 있음).
Salesforce Data Cloud 데이터를 구독할 때 다음 제한사항이 적용됩니다.
Data Cloud 데이터는 뷰로 공유됩니다. 구독자는 뷰가 참조하는 기본 테이블에 액세스할 수 없습니다.

사용 사례
이 섹션에서는 BigQuery에서 공유를 사용하는 방법의 예를 보여줍니다.
소매업체이고 조직에 예측이라는 Google Cloud 프로젝트에 실시간 수요 예측 데이터가 있다고 가정해 보겠습니다. 이 수요 예측 데이터를 공급망 시스템에 있는 수백 개의 공급업체와 공유하려고 합니다. BigQuery Sharing을 통해 공급업체와 데이터를 공유하는 방법은 다음과 같습니다.
관리자
예측 프로젝트 소유자는 먼저 API를 사용 설정한 다음 Analytics Hub 관리자 역할 [https://cloud.google.com/bigquery/docs/analytics-hub-grant-roles?hl=ko#ah-admin-role]을 프로젝트 내 데이터 교환을 관리하는 사용자에게 할당해야 합니다. Analytics Hub 관리자 역할이 있는 사용자를 Analytics Hub 관리자라고 합니다.
이 관리자는 다음 작업을 수행할 수 있습니다.
조직의 예측 프로젝트에서 데이터 교환을 생성, 업데이트, 삭제, 공유합니다.
Analytics Hub 관리자 역할이 있는 다른 관리자를 관리합니다.
조직 직원에게 Analytics Hub 게시자 역할을 부여하여 게시자를 관리합니다. 일부 직원의 경우에는 목록을 업데이트, 삭제, 공유할 수 있지만 만들 수는 없도록 하려면 Analytics Hub 목록 관리자 역할을 부여하면 됩니다.
모든 공급업체로 구성된 Google 그룹에 Analytics Hub 구독자 역할을 부여하여 구독자를 관리합니다. 일부 공급업체에게 사용 가능한 교환 및 목록에 대한 보기 액세스 권한만 부여하려면 Analytics Hub 뷰어 역할을 부여하면 됩니다. 이러한 공급업체는 목록을 구독할 수 없습니다.
자세한 내용은 데이터 교환 관리 [https://cloud.google.com/bigquery/docs/analytics-hub-manage-exchanges?hl=ko]를 참조하세요.
게시자
게시자는 예측 프로젝트 또는 다른 프로젝트에서 데이터 세트에 대해 다음 목록을 만듭니다.
목록 A: 수요 예측 데이터 세트 1
목록 B: 수요 예측 데이터 세트 2
목록 C: 수요 예측 데이터 세트 3
데이터 제공업체는 공유 데이터 세트에 대해 사용량 측정항목을 추적 [https://cloud.google.com/bigquery/docs/analytics-hub-monitor-listings?hl=ko#use-analytics-hub]할 수 있습니다. 사용량 측정항목에는 다음 세부정보가 포함됩니다.
공유 데이터 세트에 대해 실행되는 작업
구독자 프로젝트 및 조직의 공유 데이터 세트 소비 세부정보
작업에서 처리한 행 및 바이트 수
자세한 내용은 목록 관리 [https://cloud.google.com/bigquery/docs/analytics-hub-manage-listings?hl=ko]를 참조하세요.
구독자
구독자는 데이터 교환에서 액세스할 수 있는 목록을 탐색할 수 있습니다. 또한 연결된 데이터 세트를 만들어 이러한 목록을 구독하고 프로젝트에 데이터 세트를 추가할 수 있습니다. 공급업체는 연결된 데이터 세트에서 쿼리를 실행하고 실시간으로 결과를 검색할 수 있습니다.
자세한 내용은 목록 보기 및 구독 [https://cloud.google.com/bigquery/docs/analytics-hub-view-subscribe-listings?hl=ko]을 참조하세요.
가격 책정
데이터 교환 또는 목록을 관리하는 데 추가 비용이 들지 않습니다.
BigQuery 데이터 세트의 경우 게시자는 데이터 스토리지에 대해 비용이 청구되지만 구독자는 주문형 또는 용량 기반 가격 책정 모델에 따라 공유 데이터에 대해 실행되는 쿼리 비용을 지불합니다. 가격 책정에 대한 자세한 내용은 BigQuery 가격 책정 [https://cloud.google.com/bigquery/pricing?hl=ko]을 참조하세요.
Pub/Sub의 경우 주제 게시자에게는 공유 주제 및 네트워크 이그레스에 기록된 총 바이트 수(게시 처리량)에 대한 요금이 청구됩니다(해당하는 경우). 구독자에게는 연결된 구독에서 읽은 총 바이트 수(구독 처리량)와 네트워크 이그레스에 대한 요금이 청구됩니다(해당하는 경우). 자세한 내용은 Pub/Sub 가격 책정 [https://cloud.google.com/pubsub/pricing?hl=ko#pubsub]을 참조하세요.
할당량
BigQuery Sharing 할당량에 대한 자세한 내용은 할당량 및 한도 [https://cloud.google.com/bigquery/quotas?hl=ko#analytics-hub]를 참조하세요.

VPC 서비스 제어
게시자와 구독자가 VPC 서비스 제어 경계가 있는 프로젝트의 데이터에 액세스하는 데 필요한 인그레스 및 이그레스 규칙을 설정할 수 있습니다. 자세한 내용은 VPC 서비스 제어 규칙 공유 [https://cloud.google.com/bigquery/docs/analytics-hub-vpc-sc-rules?hl=ko]를 참조하세요.
