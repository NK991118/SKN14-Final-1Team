BigQuery API 종속 항목 관리
bookmark_border
이 문서에서는 BigQuery가 사용하는 Google Cloud 서비스와 API를 설명합니다. 또한 이러한 서비스를 사용 중지할 때 BigQuery 동작에 미치는 영향도 설명합니다. 프로젝트에서 서비스를 사용 설정하거나 중지하기 전에 이 문서를 검토하세요.
일부 서비스는 기본적으로 생성되는 모든 Google Cloud 프로젝트에서 사용 설정됩니다. BigQuery를 사용하는 모든 Google Cloud 프로젝트에 다른 API가 자동으로 사용 설정됩니다. 기능을 사용하기 전에 나머지 서비스를 명시적으로 사용 설정해야 합니다. 자세한 내용은 다음 리소스를 참조하세요.
기본적으로 사용 설정된 서비스 [https://cloud.google.com/service-usage/docs/enabled-service?hl=ko#default]
서비스 사용 설정 및 중지 [https://cloud.google.com/service-usage/docs/enable-disable?hl=ko]
이 문서는 관리자를 대상으로 합니다.

BigQuery Unified API로 사용 설정된 서비스
BigQuery Unified API(bigqueryunified.googleapis.com)에는 다양한 BigQuery 기능이 작동하는 데 필요한 선별된 서비스 컬렉션이 포함되어 있습니다. BigQuery Unified API를 사용 설정하면 이러한 모든 서비스가 동시에 활성화됩니다. Google은 이 컬렉션의 서비스를 업데이트할 수 있으며 이러한 서비스는 이 API가 사용 설정된 프로젝트에서 자동으로 사용 설정됩니다. 개별 서비스와 API를 사용 중지할 수 있습니다.
bigqueryunified.googleapis.com 사용 설정 방법은 서비스 사용 설정 및 중지 [https://cloud.google.com/service-usage/docs/enable-disable?hl=ko]를 참조하세요.

BigQuery 코드 애셋 수동 사용 설정
BigQuery에서 노트북, 저장된 쿼리와 같은 코드 애셋을 관리하려면 다음 API를 사용 설정해야 합니다.
Compute Engine API
Dataform API
Vertex AI API
2024년 3월 이전에는 이러한 API가 기본적으로 자동으로 사용 설정되지 않았습니다. 2024년 3월 이전에 이러한 API의 상태에 따라 달라지는 자동화 스크립트를 만든 경우 이 스크립트를 업데이트해야 할 수 있습니다. 이러한 API가 이미 사용 설정되어 있으면 BigQuery의 탐색기 창에 새 Notebooks 및 Queries 폴더가 표시됩니다.
시작하기 전에
코드 애셋 관리를 수동으로 사용 설정하려면 Identity and Access Management(IAM) 소유자(roles/owner) 역할 [https://cloud.google.com/iam/docs/understanding-roles?hl=ko#basic]이 있어야 합니다.
BigQuery 코드 애셋 수동 사용 설정
코드 애셋에 필요한 API 종속 항목을 사용 설정하려면 다음 단계를 수행합니다.
BigQuery 페이지로 이동합니다.
BigQuery로 이동 [https://console.cloud.google.com/bigquery?hl=ko]
Studio의 편집기 창 탭 표시줄에서 + 기호 옆에 있는 arrow_drop_down 화살표 드롭다운을 클릭하고 포인터를 노트북 위로 가져간 후 BigQuery 템플릿을 선택합니다.
편집기 창의 탭 표시줄 아래에 표시되는 배너에서 사용 설정을 클릭합니다.
배너가 표시되지 않으면 필요한 IAM 소유자 [https://cloud.google.com/iam/docs/understanding-roles?hl=ko#basic] 역할이 있는지 확인합니다.
기능 사용 설정 창의 핵심 기능 API 섹션에서 다음을 실행합니다.
버전 기록 및 코드 애셋 공유를 사용 설정하려면 버전 기록 및 공유 섹션에서 사용 설정을 클릭합니다.
노트북을 사용 설정하려면 Python 노트북에서 모두 사용 설정을 클릭합니다.
API가 사용 설정되면 다음을 클릭합니다.
(선택사항): 권한 섹션에서 사용자 권한을 설정합니다.
주 구성원에게 코드 애셋을 만들고 생성된 코드 애셋에 대한 읽기, 수정, 설정 권한을 부여하려면 생성자 필드에 사용자 이름이나 그룹 이름을 입력합니다.
주 구성원에게 공유된 모든 코드 애셋에 대한 읽기, 수정, 설정 권한을 부여하려면 소유자 필드에 사용자 이름이나 그룹 이름을 입력합니다.
다음을 클릭합니다.
선택사항: 추가 API 섹션에서 모두 사용 설정을 클릭하여 BigQuery DataFrames [https://cloud.google.com/python/docs/reference/bigframes/latest?hl=ko]를 사용해 BigQuery 원격 프로시저를 만드는 데 필요한 API를 사용 설정합니다.
추가 API를 사용 설정하지 않으면 닫기를 클릭하여 기능 사용 설정 창을 닫습니다.
코드 애셋에 대한 액세스 제한
리소스 서비스 사용량 제한 조직 정책 제약 조건 [https://cloud.google.com/resource-manager/docs/organization-policy/restricting-resources?hl=ko]을 설정하여 추가 API의 사용 설정을 방지할 수 있습니다. 언제든지 선택한 API를 사용 중지 [https://cloud.google.com/service-usage/docs/enable-disable?hl=ko#disabling]할 수 있습니다.
