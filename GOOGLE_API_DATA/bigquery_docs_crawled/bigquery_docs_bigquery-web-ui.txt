Google Cloud 콘솔 살펴보기
bookmark_border
Google Cloud 콘솔은 BigQuery 리소스를 생성 및 관리하고 SQL 쿼리를 실행하는 데 사용할 수 있는 그래픽 인터페이스를 제공합니다.
Google Cloud 콘솔에서 BigQuery를 사용해 보려면 빠른 시작 Google Cloud 콘솔에서 공개 데이터 세트 쿼리 [https://cloud.google.com/bigquery/docs/quickstarts/query-public-dataset-console?hl=ko]를 참조하세요.
시작하기 전에
Sign in to your Google Cloud account. If you're new to Google Cloud, create an account [https://console.cloud.google.com/freetrial?hl=ko] to evaluate how our products perform in real-world scenarios. New customers also get $300 in free credits to run, test, and deploy workloads.
In the Google Cloud console, on the project selector page, select or create a Google Cloud project.
Note: If you don't plan to keep the resources that you create in this procedure, create a project instead of selecting an existing project. After you finish these steps, you can delete the project, removing all resources associated with the project.
Go to project selector [https://console.cloud.google.com/projectselector2/home/dashboard?hl=ko]
Enable the BigQuery API.
Enable the API [https://console.cloud.google.com/flows/enableapi?apiid=bigquery&hl=ko]
새 프로젝트에서는 BigQuery API가 자동으로 사용 설정됩니다.
(선택사항) 프로젝트에 대한 결제를 사용 설정 [https://cloud.google.com/billing/docs/how-to/modify-project?hl=ko]합니다. 결제를 사용 설정하거나 신용카드를 제공하지 않는 경우 이 문서의 단계가 계속 작동합니다. BigQuery에서는 단계를 수행하기 위한 샌드박스를 제공합니다. 자세한 내용은 BigQuery 샌드박스 사용 설정 [https://cloud.google.com/bigquery/docs/sandbox?hl=ko]을 참조하세요.
참고: 프로젝트에 결제 계정이 있고 BigQuery 샌드박스를 사용하려는 경우 프로젝트의 결제를 중지 [https://cloud.google.com/billing/docs/how-to/modify-project?hl=ko#disable_billing_for_a_project]하세요.
Google Cloud 콘솔에서 BigQuery 열기
Google Cloud 콘솔로 이동합니다.
Google Cloud 콘솔로 이동 [https://console.cloud.google.com/?hl=ko]
Google Cloud 콘솔 툴바에서 menu 탐색 메뉴를 클릭합니다.
솔루션 > 모든 제품을 클릭합니다.
애널리틱스 섹션에서 BigQuery를 클릭합니다.
브라우저에 다음 URL을 입력하면 BigQuery 페이지를 직접 열 수도 있습니다.
https://console.cloud.google.com/bigquery
가장 최근에 액세스한 프로젝트에서 BigQuery가 열립니다.
탐색 메뉴에서 BigQuery를 최상위 제품으로 추가하거나 고정하면 간편하게 탐색할 수 있습니다.
Google Cloud 콘솔 탐색 메뉴에서 BigQuery 위에 포인터를 가져갑니다.
push_pin 고정을 클릭합니다.

작업 탐색기 [https://cloud.google.com/bigquery/docs/admin-jobs-explorer?hl=ko]: 조직 전반의 작업 활동을 모니터링하는 데 도움이 됩니다.

추천 [https://cloud.google.com/bigquery/docs/recommendations-intro?hl=ko]: 조직 또는 프로젝트의 추천 목록을 표시합니다.

SQL 변환 [https://cloud.google.com/bigquery/docs/batch-sql-translator?hl=ko#submit_a_translation_job]: Teradata SQL 쿼리를 BigQuery에서 작동하도록 변환할 수 있습니다.
추가 리소스
파트너 센터 [https://cloud.google.com/bigquery/docs/bigquery-ready-overview?hl=ko#partner_center]: 파트너의 도구와 서비스를 제공하여 워크플로를 가속화합니다.
설정(프리뷰 [https://cloud.google.com/products?hl=ko#product-launch-stages]): 다음 탭이 포함됩니다.
내 설정: BigQuery Studio에서 세션을 시작할 때 적용되는 기본 설정을 지정할 수 있습니다. 일부 설정은 프로젝트 또는 조직에서 상속되지만 내 설정 페이지에서 재정의할 수 있습니다.
구성 설정: BigQuery 관리자가 선택한 프로젝트 또는 조직 내 사용자의 BigQuery Studio 환경을 맞춤설정할 수 있습니다. 결과 저장 > CSV(Google Drive), 다음에서 열기 > Looker Studio 및 내보내기 > Sheets로 탐색 같은 사용자 인터페이스 요소를 표시하거나 숨길 수 있습니다. 이러한 설정은 BigQuery Studio에서 숨겨져 있더라도 기본 데이터 및 도구에 대한 액세스를 제한하지 않습니다. 조직 수준에서 변경하려면 bigquery.config.get 권한이 필요합니다.
출시 노트 [https://cloud.google.com/bigquery/docs/release-notes?hl=ko]: BigQuery의 최신 제품 업데이트 및 공지사항이 포함되어 있습니다.
BigQuery 탐색 메뉴가 표시되는 방식을 제어할 수 있습니다.
아이콘만 표시되도록 탐색 메뉴를 접으려면 BigQuery 탐색 메뉴 전환을 클릭합니다.
접을 때 메뉴를 일시적으로 펼치려면 메뉴 위에 포인터를 놓습니다.
라벨이 계속 표시되도록 메뉴를 확장하려면 BigQuery 탐색 메뉴 전환을 클릭합니다.
탐색기 창
탐색 메뉴에서 Studio를 선택하면 탐색기 창이 표시됩니다. 이 창에는 현재 Google Cloud 프로젝트와 모든 별표표시 프로젝트가 나열됩니다.
프로젝트 및 데이터 세트의 리소스를 보려면 다음을 수행합니다.
프로젝트에 대한 액세스 권한이 있는 데이터 세트를 보려면 프로젝트를 펼칩니다.
데이터 세트의 테이블, 뷰, 함수를 보려면 데이터 세트를 펼칩니다.
검색창을 사용하여 현재 프로젝트와 별표표시된 프로젝트 내에서 이름(데이터 세트, 테이블, 뷰 이름)별로 또는 라벨별로 [https://cloud.google.com/bigquery/docs/filtering-labels?hl=ko] 리소스를 검색할 수도 있습니다. 검색창에서는 검색어와 직접 일치하거나 일치하는 항목이 포함된 리소스를 찾습니다. 일치하는 리소스 수준의 모든 리소스가 표시되지 않을 수도 있습니다. 모든 리소스를 보려면 더보기를 클릭합니다.
탐색기 창이 표시되는 방법을 제어할 수 있습니다.
탐색기 창을 접으려면 first_page 아이콘을 클릭합니다.
탐색기 창을 펼치려면 last_page 아이콘을 클릭합니다.
세부정보 창
세부정보 창에 BigQuery 리소스에 대한 정보가 표시됩니다. 탐색기 창에서 데이터 세트, 테이블, 뷰 또는 기타 리소스를 선택하면 새 탭이 표시됩니다. 이러한 탭에서 리소스에 대한 정보를 보거나, 테이블 및 뷰를 만들고, 테이블 스키마를 수정하고, 데이터를 내보내고, 기타 작업을 수행할 수 있습니다.
탭을 편집기 가장자리로 드래그하여 새 열에서 열어 탭을 비교하거나 탭을 현재 열 또는 인접한 열의 다른 위치로 드래그할 수 있습니다. 이 기능은 프리뷰 [https://cloud.google.com/products?hl=ko#product-launch-stages] 버전으로 제공됩니다.
쿼리 편집기에서 대화형 쿼리를 실행 [https://cloud.google.com/bigquery/docs/running-queries?hl=ko#queries]하고 쿼리 실행 후 열리는 쿼리 결과 창에서 결과를 탐색할 수 있습니다.
탭을 탐색할 때 포커스 탭에 해당하는 리소스가 탐색기 창에서 선택됩니다. 작업공간 URL을 사용하여 BigQuery를 열면 작업공간 쿼리 편집기 탭이 열리고 탐색기 창에서 해당 리소스가 선택됩니다.
페이지 상단의 검색창을 사용하여 Google Cloud에서 리소스(프로젝트, 데이터 세트, 테이블), 문서, 제품(예: Compute Engine 및 Cloud Storage)을 검색할 수 있습니다. 다른 제품의 리소스에 액세스하려면 BigQuery 권한 [https://cloud.google.com/bigquery/docs/access-control?hl=ko#bq-permissions]과 비슷한 권한이 필요할 수 있습니다.
데이터 추가 대화상자
데이터 추가 대화상자에서 검색 및 필터링 기능을 사용하여 작업할 데이터 소스를 찾을 수 있습니다. 데이터 소스를 선택한 후에는 데이터 소스에서 사용할 수 있는 기능에 따라 다음 작업을 할 수 있습니다.
외부 데이터(제휴)를 통한 BigQuery 테이블 설정: BigQuery로 수집하지 않고도 BigQuery에서 외부 데이터에 액세스할 수 있습니다. 외부 데이터에 액세스하는 테이블을 만들거나 [https://cloud.google.com/bigquery/docs/external-data-sources?hl=ko] 외부 소스에 대한 연결을 만들 [https://cloud.google.com/bigquery/docs/connections-api-intro?hl=ko] 수 있습니다.
BigQuery로 데이터 로드: Data Transfer Service [https://cloud.google.com/bigquery/docs/dts-introduction?hl=ko]를 설정하거나 파트너 기능 [https://cloud.google.com/bigquery/docs/load-data-third-party?hl=ko]을 사용하여 BigQuery에 데이터를 로드할 수 있습니다. 규모에 맞춘 최적의 데이터 처리를 위해서는 BigQuery로 데이터를 로드하는 것이 좋습니다.
BigQuery로 변경 데이터 캡처: 변경사항을 캡처하고 적용하여 데이터 소스의 데이터를 BigQuery로 복제합니다. datastream [https://cloud.google.com/datastream/docs/overview?hl=ko] 또는 파트너 솔루션 [https://cloud.google.com/bigquery/docs/load-data-third-party?hl=ko]과 같은 애플리케이션을 사용하여 데이터 소스에서 데이터를 수집할 수 있습니다.
BigQuery에 데이터 스트리밍: 짧은 지연 시간으로 BigQuery로 데이터를 수집합니다. dataflow [https://cloud.google.com/dataflow/docs/guides/write-to-bigquery?hl=ko], Pub/Sub [https://cloud.google.com/pubsub/docs/overview?hl=ko], 파트너 솔루션 [https://cloud.google.com/bigquery/docs/load-data-third-party?hl=ko]과 같은 애플리케이션을 사용하여 데이터 소스에서 데이터를 수집할 수 있습니다.
BigQuery에 데이터를 로드하는 방법에 대한 자세한 내용은 데이터 로드 소개 [https://cloud.google.com/bigquery/docs/loading-data?hl=ko]를 참조하세요.
또한 다음을 수행할 수 있습니다.
공개 데이터 세트에 액세스하고 쿼리 [https://cloud.google.com/bigquery/docs/quickstarts/query-public-dataset-console?hl=ko]합니다.
Sharing(Analytics Hub) 페이지로 이동합니다. 자세한 내용은 BigQuery Sharing 소개 [https://cloud.google.com/bigquery/docs/analytics-hub-introduction?hl=ko]를 참조하세요.
이름으로 프로젝트에 별표표시 [https://cloud.google.com/bigquery/docs/bigquery-web-ui?hl=ko#starring_adding_a_project]합니다.
프로젝트 작업
BigQuery에서 수행하는 모든 작업은Google Cloud 프로젝트 내에서 수행됩니다.Google Cloud 콘솔 툴바에서 프로젝트 이름을 볼 수 있습니다.
BigQuery 사용에 따라 발생하는 모든 비용은 프로젝트에 연결된 결제 계정으로 청구됩니다. 자세한 내용은 BigQuery 가격 책정 [https://cloud.google.com/bigquery/pricing?hl=ko]을 참조하세요.
프로젝트로 전환
Google Cloud 콘솔에서 Google Cloud 프로젝트로 전환하려면 다음 단계를 따르세요.
Google Cloud 콘솔 툴바에서 프로젝트 이름을 클릭합니다.
프로젝트 대화상자에서 전환할 프로젝트의 이름을 클릭합니다.
선택한 프로젝트가 활성 프로젝트가 됩니다.
프로젝트 시작
다음 방법으로 탐색기 창에서 Google Cloud 프로젝트를 별표표시할 수 있습니다.
데이터 세트 또는 테이블에 대한 액세스 권한이 있지만 해당 데이터 세트가 포함된 프로젝트에 대한 액세스 권한이 없으면 다음 단계를 따르세요.
Google Cloud 콘솔 툴바에서 프로젝트 이름을 클릭합니다.
프로젝트 대화상자에서 별표표시할 프로젝트를 검색합니다.
프로젝트 이름 위로 포인터를 가져간 다음 star_border 별표 추가를 클릭합니다.
프로젝트에 대한 뷰어(roles/viewer) IAM 역할이 있으면 다음을 수행하세요.
방법 1
별표표시할 프로젝트로 전환 [https://cloud.google.com/bigquery/docs/bigquery-web-ui?hl=ko#switching_projects]합니다.
탐색기 창에서 별표표시할 프로젝트의 이름 위에 포인터를 가져간 다음 star_border 별표표시된 리소스에 추가를 클릭합니다.
방법 2
탐색기 창에서 add데이터 추가를 클릭합니다.
데이터 추가 대화상자에서 이름으로 프로젝트에 별표표시를 클릭합니다.
대화상자가 열리면 별표표시할 프로젝트의 이름을 입력하고 별표표시를 클릭합니다.
프로젝트 삭제
탐색기 창에서 프로젝트를 삭제하려면 프로젝트 이름을 클릭한 다음 star 별표표시된 리소스에서 삭제를 클릭합니다.
리소스 표시
탐색기 창에는 현재 프로젝트나 별표표시한 프로젝트에서 둘러볼 수 있는 데이터 세트가 나열됩니다.
프로젝트의 데이터 세트를 표시하려면 해당 프로젝트에 대해 arrow_right 전환 노드를 클릭합니다.
데이터 세트의 테이블, 뷰, 함수를 표시하려면 데이터 세트를 클릭합니다.
리소스를 다른 방식으로 열면 탐색기 창에도 리소스가 표시됩니다. 예를 들어 테이블을 만든 후 테이블로 이동을 클릭하면 탐색기 창에서 데이터 세트 아래에 해당 테이블이 강조표시됩니다.
프로젝트나 데이터 세트의 리소스에 대한 정보를 표시하려면 탐색기 창에서 리소스 이름을 클릭하고 세부정보 창을 확인합니다. 사용 가능한 작업 세트는 리소스에 따라 다릅니다. 예를 들어 다음 이미지에서는 테이블의 탭을 보여줍니다.
리소스 별표표시

프로젝트 별표표시에 대한 자세한 내용은 프로젝트 별표표시 [https://cloud.google.com/bigquery/docs/bigquery-web-ui?hl=ko#starring_adding_a_project]를 참조하세요.

세부정보 창에서 리소스를 선택하거나 add_box 새 쿼리 작성을 클릭할 때마다 새 탭이 열립니다. 탭이 두 개 이상 열려 있는 경우 탭을 두 개의 창으로 분할하여 나란히 볼 수 있습니다.
탭을 창 두 개로 분할하려면 다음 단계를 수행합니다.
탭 이름 옆에 있는 arrow_drop_down 메뉴 열기를 클릭합니다.
다음 옵션 중 하나를 선택합니다.
선택한 탭을 왼쪽 창에 배치하려면 탭을 왼쪽으로 분할을 선택합니다.
선택한 탭을 오른쪽 창에 배치하려면 탭을 오른쪽으로 분할을 선택합니다.
참고: 탭이 하나만 열리면 이 메뉴 옵션을 사용할 수 없습니다.
탭 분할을 해제하려면 열린 탭 중 하나에서 arrow_drop_down 메뉴 열기를 선택한 다음 탭을 왼쪽 창으로 이동 또는 탭을 오른쪽 창으로 이동을 선택합니다.
테이블을 쿼리할 때 탭을 분할하려면 다음 단계를 수행합니다.
탐색기 메뉴에서 쿼리할 테이블을 클릭합니다.
쿼리를 클릭한 다음 새 탭 또는 분할 탭을 클릭합니다.
쿼리할 필드 이름을 클릭합니다.
다음 이미지는 탭 두 개가 열린 세부정보 창을 보여줍니다. 탭 하나에는 SQL 쿼리가 있고 다른 탭에는 테이블에 대한 세부정보가 표시됩니다.
탭 이동
창 하나에서 다른 창으로 탭을 이동하려면 다음 단계를 수행합니다.
탭 이름 옆에 있는 arrow_drop_down 메뉴 열기를 클릭합니다.
탭을 오른쪽 창으로 이동 또는 탭을 왼쪽 창으로 이동(사용 가능한 옵션 선택)을 선택합니다.
탭 닫기
탭 하나를 제외한 모든 탭을 닫으려면 다음 단계를 수행합니다.
탭 이름 옆에 있는 arrow_drop_down 메뉴 열기를 클릭합니다.
cancel 다른 탭 닫기를 선택합니다.
개인 및 프로젝트 기록 보기
세부정보 창의 바닥글에서 작업 기록을 볼 수 있습니다.
자체 작업의 세부정보를 보려면 개인 기록을 클릭합니다.
프로젝트의 최근 작업의 세부정보를 보려면 프로젝트 기록을 클릭합니다.
작업의 세부정보를 보거나 편집기에서 쿼리를 열려면 다음을 수행합니다.
작업 또는 쿼리에 대한 작업 열에서 more_vert 작업을 클릭합니다.
작업 세부정보 표시 또는 편집기에서 쿼리 열기를 선택합니다.
작업 기록에는 지난 6개월 동안 제출한 모든 로드, 내보내기, 복사, 쿼리 작업이 포함됩니다(최대 1,000개 항목). 작업 한도인 1,000개는 모든 작업 유형에서 누적 계산됩니다.
