리소스: AccessProposal
파일에 대한 대기 중인 액세스 제안의 액세스 제안 리소스
JSON 표현
{
  "fileId": string,
  "proposalId": string,
  "requesterEmailAddress": string,
  "recipientEmailAddress": string,
  "rolesAndViews": [
    {
      object (RoleAndView [https://developers.google.com/workspace/drive/api/reference/rest/v3/accessproposals?hl=ko#RoleAndView])
    }
  ],
  "requestMessage": string,
  "createTime": string
}
필드
fileId
string
액세스 제안이 있는 파일 ID입니다.
proposalId
string
액세스 제안의 ID입니다.
requesterEmailAddress
string
요청한 사용자의 이메일 주소
recipientEmailAddress
string
수락 시 권한을 받게 되는 사용자의 이메일 주소입니다.
rolesAndViews[]
object (RoleAndView [https://developers.google.com/workspace/drive/api/reference/rest/v3/accessproposals?hl=ko#RoleAndView])
액세스 제안서의 역할 및 뷰를 위한 래퍼입니다.
requestMessage
string
요청자가 제안서에 추가한 메시지
createTime
string (Timestamp [https://protobuf.dev/reference/protobuf/google.protobuf/#timestamp] format)
생성 시간
생성된 출력은 항상 Z-정규화되고 소수점 이하 0, 3, 6 또는 9자리인 RFC 3339를 사용합니다. 'Z' 이외의 오프셋도 허용됩니다. 예를 들면 "2014-10-02T15:01:23Z", "2014-10-02T15:01:23.045123456Z" 또는 "2014-10-02T15:01:23+05:30"입니다.
RoleAndView
액세스 제안서의 역할 및 뷰를 위한 래퍼입니다.
JSON 표현
{
  "role": string,
  "view": string
}
필드
role
string
요청자가 제안한 역할입니다. 향후 새로운 값이 추가될 수 있지만 현재는 다음 값을 사용할 수 있습니다. * writer * commenter * reader
view
string
이 액세스 제안서의 뷰를 나타냅니다. 뷰에 속한 제안서에 대해서만 채워집니다. 유일하게 지원되는 값은 published입니다.
메서드
get [https://developers.google.com/workspace/drive/api/reference/rest/v3/accessproposals/get?hl=ko]
ID로 AccessProposal을 검색합니다.
list [https://developers.google.com/workspace/drive/api/reference/rest/v3/accessproposals/list?hl=ko]
파일에 AccessProposals를 나열합니다.
resolve [https://developers.google.com/workspace/drive/api/reference/rest/v3/accessproposals/resolve?hl=ko]
액세스 제안서를 승인하거나 거부하는 데 사용됩니다.
