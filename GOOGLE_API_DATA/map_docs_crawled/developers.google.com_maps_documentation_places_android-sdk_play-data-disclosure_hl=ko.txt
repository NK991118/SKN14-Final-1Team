Google Play의 데이터 공개 요건 준비

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

2021년 5월, Google Play는 앱의 데이터 수집, 공유, 보안 관행과 관련하여 개발자가 제공하는 공개 정보인 새로운 데이터 보안 섹션을 발표 [https://android-developers.googleblog.com/2021/05/new-safety-section-in-google-play-will.html]했습니다.

이 페이지는 Android용 Places SDK 사용과 관련하여 이러한 데이터 공개 요건을 충족하는 데 도움이 되는 정보를 제공합니다. 이 페이지에서는 앱 개발자로서 관리할 수 있는 관련 구성 또는 호출을 포함하여 Android용 Places SDK에서 최종 사용자 데이터를 처리하는 방식에 대해 확인할 수 있습니다.

Google은 최대한 투명한 방식으로 개발자를 지원하려고 노력하고 있습니다. 하지만 앱의 최종 사용자 데이터 수집, 공유 및 보안 관행과 관련하여 Google Play의 데이터 보안 섹션 양식에 어떻게 답변할지 결정할 책임은 전적으로 앱 개발자에게 있습니다.

이 페이지의 정보를 사용하는 방법

이 페이지에는 Android용 Places SDK의 최신 버전에서 수집된 최종 사용자 데이터만 표시됩니다.

데이터 공개 정보를 작성하려는 경우 Android의 데이터 유형 가이드 [https://developer.android.com/guide/topics/data/collect-share?hl=ko]를 참고하면 수집되는 데이터를 가장 잘 설명하는 데이터 유형을 판단하는 데 도움이 됩니다. 데이터 공개 정보에는 특정 앱이 수집된 데이터를 공유하고 사용하는 방식에 대한 설명도 포함되어야 합니다.

중요: 앱의 데이터 공개에 정확한 정보가 반영되도록 다음 사항을 권장합니다.

앱에서 항상 최신 SDK 버전을 사용합니다. 이 페이지에는 최신 버전의 Android용 Places SDK에서 수집된 데이터만 표시됩니다.
앱의 SDK 버전을 업데이트할 때마다 이 페이지를 검토하고, 데이터 공개가 정확하며 최신 정보인지 확인하세요.

Android용 Places SDK의 데이터 암호화, 데이터 공유 개요

| 데이터 암호화 | 이 페이지에 나온 수집되는 최종 사용자 데이터의 경우 Android용 Places SDK는 HTTPS를 사용해 전송 중 데이터를 암호화합니다. |
| --- | --- |
| 데이터 공유 | 이 페이지에 명시된 수집되는 최종 사용자 데이터의 경우 Android용 Places SDK는 Google 서비스 제공을 지원하는 경우를 제외하고 이 데이터를 서드 파티에 전송하지 않습니다. |
| 데이터 삭제 | 이 페이지에 나열된 수집된 최종 사용자 데이터의 경우 Android용 Places SDK는 해당 데이터를 Google 계정 정보와 연결하지 않습니다. 데이터는 Google 개인정보처리방침에 따라 자동으로 삭제되거나 익명처리됩니다. |

자동으로 수집되는 데이터

| 데이터 | 기본적으로 Android용 Places SDK는 다음을 수집합니다. |
| --- | --- |
| 요청 메타데이터 예를 들면 다음과 같습니다. 기기 메타데이터: OS 버전, 이름, 모델, 브랜드, 폼 팩터 Android용 Places SDK 빌드 및 버전 정보 반환된 결과의 수 | 요청 메타데이터를 수집하여 Google 서비스를 관리하고 개선합니다. |
| 스택 트레이스 비정상 종료 측정항목 | SDK 안정성과 Google 서비스를 개선하기 위해 SDK 코드 내의 비정상 종료 중 스택 트레이스를 수집하고 이상 종료와 같은 비정상 종료 측정항목을 수집합니다. |
| IP 주소 | IP 주소를 수집하여 SDK 사용량을 파악하고 Google 서비스를 개선합니다. |
| Google Maps Platform용 식별자 | 앱에 특화된 Google Maps Platform SDK 식별자를 생성하고 수집하여 일일 활성 SDK 사용자를 측정하고 Google 서비스를 개선합니다. |

| 위도/경도 좌표 Wi-Fi 액세스 포인트 | 앱에서 기능을 사용하여 기기의 현재 보고된 위치에서 장소를 검색하는 경우 위도/경도 좌표와 주변 Wi-Fi 액세스 포인트를 수집하여 주변 장소를 찾습니다. |
| 위젯 상호작용 이벤트 | 앱에서 Place Autocomplete 위젯과 같은 구성요소를 사용하는 경우 Google 서비스를 개선하기 위해 상호작용 데이터를 수집합니다. |
| 메타데이터 요청 예를 들면 다음과 같습니다. Places SDK 버전 및 API 사용 Places API의 입력 매개변수 반환된 결과 수 | 요청 메타데이터를 수집하여 Google 서비스를 관리하고 개선합니다. |
