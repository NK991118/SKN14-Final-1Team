차량 삭제

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

이 문서에서는 차량을 삭제하는 방법을 설명합니다. Fleet Engine을 설정했다고 가정합니다. Fleet Engine 설정 [https://developers.google.com/maps/documentation/mobility/fleet-engine/essentials/set-up-fleet?hl=ko]을 참고하세요.

차량 삭제 기본사항

다음과 같은 상황에서 시스템이 Fleet Engine을 사용하여 차량을 삭제할 수 있습니다.

Fleet Engine API를 테스트하는 동안 정리 작업을 실행합니다.
더 이상 필요하지 않은 차량을 즉시 삭제합니다.

차량을 삭제하려면 gRPC 또는 REST를 사용하여 요청을 전송합니다.

DeleteVehicle() 메서드: gRPC [https://developers.google.com/maps/documentation/mobility/fleet-engine/reference/trips/rpc/maps.fleetengine.v1?hl=ko#maps.fleetengine.v1.VehicleService] 또는 REST [https://developers.google.com/maps/documentation/mobility/fleet-engine/reference/trips/rest/v1/providers.vehicles/delete?hl=ko]
DeleteVehicleRequest 메시지: gRPC [https://developers.google.com/maps/documentation/mobility/fleet-engine/reference/trips/rpc/maps.fleetengine.v1?hl=ko#deletevehiclerequest]만 해당

Fleet Engine: 서비스 계정 역할 [https://developers.google.com/maps/documentation/mobility/fleet-engine/essentials/set-up-fleet/service-accounts?hl=ko]에 설명된 대로 프로젝트의 서비스 계정에 적절한 사용자 인증 정보를 사용합니다.
참고: Fleet Engine은 일정 기간 동안 비활성 상태인 차량을 자동으로 삭제합니다. 차량 재사용 [https://developers.google.com/maps/documentation/mobility/fleet-engine/essentials/vehicles?hl=ko#vehicle_re-use]을 참고하세요.

삭제 작업이 성공하면 API가 빈 응답을 반환합니다.

오류 처리

차량을 삭제할 때 FAILED_PRECONDITION 오류가 발생할 수 있습니다. 이 경우 차량을 참조하는 여정이 있습니다.
삭제를 진행하려면 다음 단계를 따르세요.

SearchTrips를 호출하여 차량을 참조하는 여정을 찾습니다.
DeleteTrip을 호출하여 발견된 각 여행을 삭제합니다.

경고: 발견된 여정이 더 이상 필요하지 않은지 확인하세요. 삭제한 여행은 복구할 수 없습니다.
