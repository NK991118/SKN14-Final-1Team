Apple App Store 개인 정보 보호 세부정보

Google은 고객이 최종 사용자를 위한 훌륭한 iOS 앱을 만들 수 있도록 최선을 다해 돕고 있습니다.
iOS 14부터 Apple은 앱 개인정보 보호 세부정보 [https://developer.apple.com/app-store/app-privacy-details/] 페이지와 개인 정보 보호 매니페스트 문서 [https://developer.apple.com/documentation/bundleresources/privacy_manifest_files]에 앱 개인 정보 보호 요구사항을 게시해 왔습니다.

참고: Google Maps Platform SDK의 개인 정보 보호 매니페스트는 항상 수집되고 기본적으로 수집되는 데이터를 공개합니다. 앱의 실제 SDK 사용량에 따라 개인 정보 보호 영양 성분 라벨이 정확한지 확인하는 것은 개발자의 책임입니다.

이제 Apple에서 설문지에 관한 자세한 내용을 발표했습니다.  Google은 앱 개발자로서 제어할 수 있는 관련 설정 또는 구성을 포함하여 Google Maps Platform API 및 SDK가 최종 사용자의 데이터를 처리하는지 여부와 방법에 대한 정보를 공유할 것입니다.

iOS용 Google Maps Platform SDK는 iOS 앱에서 서드 파티 SDK로 간주됩니다. 개인 정보 보호 매니페스트는 개발자가 서드 파티 SDK의 데이터 사용을 비롯해 개인 정보 보호 관련 데이터에 액세스하는 이유를 설명하는 데 도움이 됩니다.

Google은 최대한 투명한 방식으로 개발자를 지원하려고 노력하고 있습니다. 하지만 Apple의 개인 정보 보호 요구사항을 충족하는 방법과 최종 사용자에게 Apple에서 설명하는 권한 또는 프롬프트를 보여줄 것인지 여부를 결정할 책임은 전적으로 앱 개발자에게 있습니다.

Google에서 제공하는 리소스는 Apple이 서드 파티 SDK 소유자에게 제공하는 최신 정보를 기반으로 하며, 새로운 정보가 나오면 새로운 기능 섹션이 업데이트됩니다.

새로운 소식
업데이트: 2024년 9월 18일
이제 iOS용 Google Maps Platform SDK 설치 안내에 최신 버전의 SDK에 포함된 개인 정보 보호 매니페스트 파일을 검사하는 방법이 포함되어 있습니다. 이러한 개인 정보 보호 매니페스트의 콘텐츠에 관해 Apple로부터 의견을 받으면 관련 Issue Tracker 문제 [https://issuetracker.google.com/issues/323059684?hl=ko]에 의견을 보고하세요. 각 SDK의 안내와 매니페스트는 다음 링크에서 확인할 수 있습니다.

iOS용 Maps SDK [https://developers.google.com/maps/documentation/ios-sdk/config?hl=ko#add-apple-privacy-manifest-file]
iOS용 Places SDK [https://developers.google.com/maps/documentation/places/ios-sdk/config?hl=ko#add-apple-privacy-manifest-file]
iOS용 Navigation SDK [https://developers.google.com/maps/documentation/navigation/ios-sdk/xcode-setup?hl=ko#add-apple-privacy-manifest-file]
iOS용 Driver SDK [https://developers.google.com/maps/documentation/mobility/driver-sdk/on-demand/ios/minimum-requirements?hl=ko#add-apple-privacy-manifest-file]
iOS용 Consumer SDK [https://developers.google.com/maps/documentation/mobility/journey-sharing/on-demand/ios/setup?hl=ko#inspect-apple]
