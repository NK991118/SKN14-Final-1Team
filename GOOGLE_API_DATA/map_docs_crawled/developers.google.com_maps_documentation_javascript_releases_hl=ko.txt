Maps JavaScript API 출시 노트

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

이 출시 노트를 구독 [https://developers.google.com/static/maps/documentation/javascript/feeds/releases.xml?hl=ko]하세요.
[https://developers.google.com/static/maps/documentation/javascript/feeds/releases.xml?hl=ko]

Maps JavaScript API팀은 새로운 기능, 버그 수정, 성능 개선사항을 반영하여 API를 정기적으로 업데이트합니다. Maps JavaScript API 부트스트랩 요청의 v 매개변수에 버전을 지정하여 애플리케이션 내에서 로드할 API 버전을 나타낼 수 있습니다. 버전 관리 [https://developers.google.com/maps/documentation/javascript/versions?hl=ko]에 대해 자세히 알아보세요.

이 변경 로그에는 날짜 및 버전 번호별 출시와 관련 변경사항이 나열되어 있습니다.

새 지도 API 버전에 관한 업데이트를 받으려면 google-maps-js-api-v3-notify [http://groups.google.com/group/google-maps-js-api-v3-notify?hl=ko] 그룹을 구독하세요.

2021년 8월 26일부터 beta 채널을 시작으로 Maps JavaScript API는 여러 기능의 SLO(서비스 수준 목표)를 계산하기 위해 원격 분석 정보를 보고하는 추가 호출을 할 수 있습니다. 이러한 호출이 브라우저 수준에서 차단되지 않도록 하려면 콘텐츠 보안 정책을 업데이트해야 할 수도 있습니다. 자세한 내용은 FAQ [https://developers.google.com/maps/faq?hl=ko#uptime_slo]를 참고하세요.

Maps JavaScript API 버전 3.27이 이제 실험용 버전으로 제공됩니다. (API 버전 관리 [https://developers.google.com/maps/documentation/javascript/versions?hl=ko] 가이드를 참고하세요.)

변경사항:

MapOptions 객체의 새로운 gestureHandling 옵션을 사용하면 모바일 기기에서 지도와 상호작용할 때 사용자의 환경을 최적화할 수 있습니다. 사용 가능한 값은 다음과 같습니다.

greedy: 사용자가 화면을 스와이프(드래그)하면 지도가 항상 위 또는 아래, 왼쪽 또는 오른쪽으로 이동합니다. 즉, 한 손가락 스와이프와 두 손가락 스와이프 모두 지도를 이동시킵니다.
cooperative: 사용자는 한 손가락으로 스와이프하여 페이지를 스크롤하고 두 손가락으로 지도를 이동해야 합니다. 사용자가 한 손가락으로 지도를 스와이프하면 지도에 오버레이가 표시되고 사용자에게 두 손가락을 사용하여 지도를 이동하라는 메시지가 표시됩니다. 휴대기기에서 위의 샘플을 확인하여 협력 모드가 실제로 작동하는 것을 확인하세요.
none: 지도를 이동하거나 확대/축소할 수 없습니다.
auto (기본값): 페이지의 스크롤 가능 여부에 따라 동작이 cooperative 또는 greedy입니다.

자세한 내용과 예시는 개발자 가이드 [https://developers.google.com/maps/documentation/javascript/interaction?hl=ko]를 참고하세요.

전체 화면 컨트롤은 모바일 기기에서 기본적으로 표시되므로 사용자가 지도를 쉽게 확대할 수 있습니다. 지도가 전체 화면 모드인 경우 사용자는 한 손가락 또는 두 손가락을 사용하여 지도를 이동할 수 있습니다. 참고: iOS는 전체 화면 기능을 지원하지 않습니다. 따라서 iOS 기기에는 전체 화면 컨트롤이 표시되지 않습니다.

경로 서비스 응답의 경유지 변경

2016년 8월 29일

Directions 서비스 응답의 via_waypoints [https://developers.google.com/maps/documentation/javascript/reference?hl=ko#DirectionsLeg] 필드에는 원래 요청에 지정되지 않은 경유지 배열이 포함됩니다. via_waypoints 필드는 드래그 가능한 방향 응답에 계속 표시되지만 대체 경로 응답에서는 지원 중단됩니다. 버전 3.27이 대체 경로에서 via_waypoints를 지원하는 마지막 API입니다.

지도 및 스트리트 뷰 파노라마의 여러 컨트롤의 기본 위치와 모양이 변경되었습니다. 이제 지도가 로그인 모드 [https://developers.google.com/maps/documentation/javascript/signedin?hl=ko]를 사용하는지 여부와 관계없이 사용자 환경이 일관되며 Google 지도 웹사이트와도 더 일관됩니다. 이전 제어 세트를 한동안 계속 사용하려면 v3.22에서 google.maps.controlStyle = 'azteca'를 설정하면 됩니다.
스트리트 뷰의 새로운 전체 화면 컨트롤을 사용하면 사용자가 전체 화면 모드에서 스트리트 뷰 파노라마를 열 수 있습니다.

지원 중단됨:

개요 지도 컨트롤을 더 이상 사용할 수 없습니다.
지도에서 이동 컨트롤을 더 이상 사용할 수 없습니다. 뷰를 화면 이동하려면 사용자가 지도를 클릭하고 드래그하거나 스와이프합니다. (스트리트 뷰의 화면 이동 컨트롤은 계속 사용할 수 있습니다.)
확대/축소 컨트롤은 한 가지 스타일로만 제공되므로 google.maps.ZoomControlStyle를 더 이상 사용할 수 없습니다.

개발자가 더 이상 MarkerImage의 크기를 지정할 필요가 없습니다. 제공되지 않으면 API에서 감지합니다. 이와 관련하여 MarkerImage의 크기, 앵커, 원점 인수는 모두 선택사항입니다.

OverlayView 'changed' 메서드가 인터페이스에서 삭제되었습니다. 이러한 메서드는 이전에 실제로 사용되지 않았으므로 개발자의 코드에 영향을 미치지 않습니다.

partialmatch 옵션이 Geocoder Request 객체에서 삭제되었습니다. 개발자가 계속 전달해도 쿼리에는 영향을 미치지 않습니다.
