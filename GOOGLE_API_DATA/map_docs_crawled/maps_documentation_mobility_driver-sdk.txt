운전자 환경
Navigation SDK와 Driver SDK를 모두 사용하여 배송 모델에 맞게 운전 환경을 조정하세요.
[https://developers.google.com/maps/documentation/mobility/driver-sdk/navigation?hl=ko]
Navigation SDK [https://developers.google.com/maps/documentation/mobility/driver-sdk/navigation?hl=ko]
Navigation SDK를 사용하여 앱에 세부 경로 안내 내비게이션 환경을 만들어 운전자를 실시간으로 안내합니다.
개요 [https://developers.google.com/maps/documentation/mobility/driver-sdk/navigation?hl=ko]
[https://developers.google.com/maps/documentation/mobility/driver-sdk/on-demand?hl=ko]
드라이버 SDK [https://developers.google.com/maps/documentation/mobility/driver-sdk/on-demand?hl=ko]
Driver SDK를 사용하면 운전자 위치와 경로 진행 상황을 Fleet Engine을 통해 시각화할 수 있습니다.

드라이버 환경을 통합하는 방법
이 단계에서는 Fleet Engine과 호환되도록 드라이버 환경을 드라이버 앱에 통합하기 위한 대략적인 워크플로를 설명합니다.

내비게이션 SDK는 세부 경로 안내, 맞춤 경로, 맞춤설정된 지도 요소와 같은 운전자에게 맞춤설정된 내비게이션 기능을 제공하는 라이브러리입니다.
local_shipping [https://developers.google.com/maps/documentation/mobility/driver-sdk/on-demand?hl=ko]
드라이버 SDK 설정 [https://developers.google.com/maps/documentation/mobility/driver-sdk/on-demand?hl=ko]
Driver SDK는 Fleet Engine에 실시간 위치 신호를 전송합니다. 이는 Fleet Engine에서 위치 및 라우팅 기능을 사용 설정하는 데 필요한 부분입니다. Driver SDK는 Navigation SDK의 래퍼입니다.
table [https://developers.google.com/maps/documentation/mobility/fleet-engine?hl=ko]
Fleet Engine을 통한 여정 관리 [https://developers.google.com/maps/documentation/mobility/fleet-engine?hl=ko]
Fleet Engine은 Driver SDK와 자체 백엔드 서비스 간의 상호작용을 처리합니다. 백엔드 서비스는 REST 또는 gRPC를 호출하여 Fleet Engine과 통신할 수 있습니다.
