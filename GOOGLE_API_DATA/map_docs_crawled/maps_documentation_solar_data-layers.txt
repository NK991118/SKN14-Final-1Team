데이터 영역 요청
유럽 경제 지역 (EEA) 개발자
dataLayers [https://developers.google.com/maps/documentation/solar/reference/rest/v1/dataLayers?hl=ko] 엔드포인트는 지정된 위치를 둘러싼 지역에 대한 자세한 태양광 정보를 제공합니다. 엔드포인트는 다음을 비롯한 다운로드 가능한 TIFF 파일 17개를 반환합니다.
디지털 표면 모델 (DSM)
RGB 합성 레이어 (항공 이미지)
분석 경계를 식별하는 마스크 레이어
연간 태양 플럭스 또는 특정 표면의 연간 수율
월별 태양 플럭스 또는 특정 표면의 월별 수율
시간별 그늘 (24시간)
Solar API에서 플럭스를 정의하는 방법에 관한 자세한 내용은 Solar API 개념 [https://developers.google.com/maps/documentation/solar/concepts?hl=ko]을 참고하세요.
데이터 영역 요청 정보
다음 예시는 dataLayers 메서드에 대한 REST 요청의 URL을 보여줍니다.
https://solar.googleapis.com/v1/dataLayers:get?parameters
다음을 지정하는 요청 URL 매개변수 [https://developers.google.com/maps/documentation/solar/reference/rest/v1/dataLayers/get?hl=ko#query-parameters]를 포함합니다.
위치의 위도 및 경도 좌표
위치를 둘러싼 지역의 반경
반환할 데이터의 하위 집합 (DSM, RGB, 마스크, 연간 플럭스 또는 월별 플럭스)
결과에 허용되는 최소 품질
반환할 데이터의 최소 스케일(미터/픽셀)
데이터 영역 요청 예
다음 예에서는 위도 = 37.4450, 경도 = -122.1390 좌표에 있는 위치의 반경 100미터 내에 있는 모든 건물 통계 정보를 요청합니다.
API는 다음 형식으로 URL을 반환합니다.
https://solar.googleapis.com/v1/solar/geoTiff:get?id=HASHED_ID
이러한 URL을 사용하여 요청된 데이터가 포함된 GeoTIFF 파일에 액세스할 수 있습니다.

응답 데이터 액세스
응답 URL을 통해 데이터에 액세스하려면 추가 인증이 필요합니다. 인증 키를 사용하는 경우 URL에 API 키를 추가해야 합니다. OAuth 인증을 사용하는 경우 OAuth 헤더를 추가해야 합니다.
RGB 레이어를 제외한 모든 TIFF 파일은 이미지 뷰어 애플리케이션에서 빈 이미지로 표시됩니다. 다운로드한 TIFF 파일을 보려면 QGIS와 같은 매핑 애플리케이션 소프트웨어로 가져오세요.
이 요청 및 응답의 전체 사양은 참조 문서 [https://developers.google.com/maps/documentation/solar/reference/rest/v1/dataLayers/get?hl=ko]에 있습니다.
참고: 응답 URL은 초기 요청 후 1시간 동안만 활성화됩니다. 이 기간이 지난 URL에 액세스하려면 dataLayers 엔드포인트에 요청을 다시 보내야 합니다.
