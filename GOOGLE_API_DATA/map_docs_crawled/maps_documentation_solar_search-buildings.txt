건물 검색
Solar API는 장소 데이터를 사용하여 건물 유형을 확인하고 위성 이미지를 기반으로 건물을 식별합니다. 이 페이지에서는 buildingInsights [https://developers.google.com/maps/documentation/solar/reference/rest/v1/buildingInsights?hl=ko] 요청에서 반환되는 건물의 태양광 정보를 확인하는 방법을 설명합니다.
건물 유형
Solar API는 premise [https://developers.google.com/maps/documentation/places/web-service/place-types?hl=ko] 유형의 건물을 지원합니다. 이 유형은 이름이 지정된 위치, 일반적으로 공통된 이름을 가진 건물 또는 여러 건물을 나타냅니다.
subpremise [https://developers.google.com/maps/documentation/places/web-service/place-types?hl=ko#table-b] 또는 화합물(예: premise [https://developers.google.com/maps/documentation/places/web-service/place-types?hl=ko]로 등록되지 않은 단일 속성에 있는 분리된 건물)과 같은 다른 유형은 지원되지 않습니다.
지붕 건축
Solar API는 위성 이미지를 기반으로 최적의 옥상 태양광 패널 위치를 식별합니다. 주거용 건물, 특히 경사형 또는 박공형 지붕은 정확도가 높게 식별되지만 평평한 지붕이 있는 건물 (예: 아파트 또는 상업용 건물)은 정확도가 다를 수 있습니다.
buildingInsights [https://developers.google.com/maps/documentation/solar/building-insights?hl=ko#example_response_object] 응답에 solarPanelConfigs 필드가 포함되지 않은 경우 건물이 올바르게 처리되었지만 지붕에 패널을 설치할 수 없습니다. 지붕이 너무 작아 패널을 설치할 수 없거나 패널이 상당한 에너지를 생성할 수 없을 정도로 그늘이 많은 경우 이러한 문제가 발생할 수 있습니다.
팁: 수동 사용자 선택 또는 맞춤 프로그래매틱 선택을 비롯한 다른 선택 접근 방식을 기반으로 Solar API dataLayers [https://developers.google.com/maps/documentation/solar/reference/rest/v1/dataLayers?hl=ko] 엔드포인트를 사용하여 자체 태양광 잠재력 계산 및 패널 배치를 만들 수 있습니다.
주소에서 검색
Geocoding API [https://developers.google.com/maps/documentation/geocoding/requests-geocoding?hl=ko#Types] 또는 Place Autocomplete API [https://developers.google.com/maps/documentation/places/web-service/autocomplete?hl=ko]를 사용하여 주소를 위도 및 경도 좌표로 변환할 수 있습니다.
place_id을 사용하여 일치하는 장소를 참조합니다.
Solar API buildingInsights [https://developers.google.com/maps/documentation/solar/reference/rest/v1/buildingInsights?hl=ko] 요청에 장소의 위도 및 경도 좌표를 사용합니다.
Solar API에서 일치하는 건물을 찾으면 buildingInsights 응답의 place_id가 Geocoding API 또는 Place Autocomplete API의 place_id와 일치하는지 확인합니다.
결과를 유형이 premise인 장소로 제한하려면 Geocoding API 또는 Places API 응답에 장소 유형 필터를 추가하세요.
{
"name": "buildings/ChIJh0CMPQW7j4ARLrRiVvmg6Vs",
"center": {
"latitude": 37.4449739,
"longitude": -122.13914659999998
},...
}
참고: 장소 ID가 일치하지 않는 경우가 있습니다. 즉, 검색된 (lat, lng)이 동일한 주소나 건물로 확인되지 않습니다. 지오코딩 API [https://developers.google.com/maps/documentation/geocoding/overview?hl=ko] 응답의 세부사항이 ROOFTOP [https://developers.google.com/maps/documentation/geocoding/requests-geocoding?hl=ko#results]인 경우에도 place_id의 유형이 premise이 아닐 수 있습니다.
오픈소스 샘플 앱 [https://solar-potential-296769475687.us-central1.run.app/]을 사용하거나 buildingInsights [https://developers.google.com/maps/documentation/solar/building-insights?hl=ko] 요청을 하여 Solar API가 적용되는 건물의 place_id를 확인할 수 있습니다.
지도에서 위치 선택
사용자가 Maps JavaScript API [https://developers.google.com/maps/documentation/javascript/overview?hl=ko]를 사용하여 지도에서 건물을 선택할 수 있도록 지원하여 보완적인 환경을 제공할 수도 있습니다. 마우스 이벤트 [https://developers.google.com/maps/documentation/javascript/examples/event-click-latlng?hl=ko]에서 건물의 위도와 경도를 추출한 다음 buildingInsights [https://developers.google.com/maps/documentation/solar/building-insights?hl=ko] 요청에서 해당 좌표를 전달합니다.
