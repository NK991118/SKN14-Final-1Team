이 기능을 사용하려면 차세대 Places API인 Places API (New)를 사용 설정해야 합니다. 자세한 내용은 API 사용 설정 [https://developers.google.com/maps/documentation/places/ios-sdk/cloud-setup?hl=ko#enabling-apis]을 참고하세요.

Places SDK for iOS (New)는 장소의 이름과 주소, 위도/경도 좌표로 지정된 지리적 위치, 장소 유형 (예: 나이트클럽, 반려동물 가게, 박물관) 등 장소에 관한 풍부한 정보를 앱에 제공합니다. 특정 장소의 이 정보에 액세스하려면 장소를 고유하게 식별하는 안정적인 식별자인 장소 ID를 사용하면 됩니다.

장소 세부정보 가져오기

GMSPlace [https://developers.google.com/maps/documentation/places/ios-sdk/reference/objc/Classes/GMSPlace?hl=ko] 클래스에는 장소 데이터 필드 (신규) [https://developers.google.com/maps/documentation/places/ios-sdk/data-fields?hl=ko]에 표시된 모든 데이터 필드를 비롯한 특정 장소에 관한 정보가 포함됩니다. GMSPlacesClient
fetchPlaceWithRequest: [https://developers.google.com/maps/documentation/places/ios-sdk/reference/objc/Classes/GMSPlacesClient?hl=ko#-fetchplacewithrequest:callback:]을 호출하여 GMSPlace [https://developers.google.com/maps/documentation/places/ios-sdk/reference/objc/Classes/GMSPlace?hl=ko] 객체를 가져오고 GMSFetchPlaceRequest [https://developers.google.com/maps/documentation/places/ios-sdk/reference/objc/Classes/GMSFetchPlaceRequest?hl=ko] 객체와 GMSPlaceResultCallback [https://developers.google.com/maps/documentation/places/ios-sdk/reference/objc/Type-Definitions?hl=ko#/c:GMSPlacesClient.h@T@GMSPlaceResultCallback] 유형의 콜백 메서드를 전달합니다.

GMSFetchPlaceRequest 객체는 다음을 지정합니다.

(필수사항) Google 지역 정보 데이터베이스 및 Google 지도에 있는 장소의 고유 식별자인 장소 ID [https://developers.google.com/maps/documentation/places/ios-sdk/place-id?hl=ko]입니다.
(필수) GMSPlace 객체에서 반환할 필드의 목록입니다. GMSPlaceProperty [https://developers.google.com/maps/documentation/places/ios-sdk/reference/objc/Constants?hl=ko]에 의해 정의되며 필드 마스크라고도 합니다.
필드 목록에 필드를 하나 이상 지정하지 않거나 필드 목록을 생략하면 호출에서 오류가 반환됩니다.
(선택사항) 응답의 형식을 지정하는 데 사용되는 지역 코드입니다.
(선택사항) Autocomplete (New) 세션을 종료하는 데 사용되는 세션 토큰입니다.

isOpenWithRequest 응답

isOpenWithRequest는 비즈니스가 영업 중인지, 폐업했는지 또는 상태를 알 수 없는지 여부를 나타내는 status라는 불리언 값을 포함하는 GMSPlaceIsOpenResponse 객체를 반환합니다.

| 언어 | 열려 있는 경우의 값 | 닫힌 경우의 값 | 상태를 알 수 없는 경우의 값 |
| --- | --- | --- | --- |
| Places Swift | true | false | nil |
| Swift | .open | .closed | .unknown |
| Objective-C | GMSPlaceOpenStatusOpen | GMSPlaceOpenStatusClosed | GMSPlaceOpenStatusUnknown |

isOpenWithRequest 결제

GMSPlacePropertyUTCOffsetMinutes 및 GMSPlacePropertyBusinessStatus 필드는 Basic Data SKU [https://developers.google.com/maps/documentation/places/ios-sdk/usage-and-billing?hl=ko#basic-data-ess-sku]에 따라 요금이 청구됩니다. 나머지 영업시간은 Place Details Enterprise SKU [https://developers.google.com/maps/documentation/places/ios-sdk/usage-and-billing?hl=ko#place-details-ent-sku]에 따라 요금이 청구됩니다.
이전 요청에서 GMSPlace 객체에 이러한 필드가 이미 있는 경우 다시 청구되지 않습니다.
경고 GMSPlace 객체에 이러한 필드가 없으면 기본 데이터 SKU [https://developers.google.com/maps/documentation/places/ios-sdk/usage-and-billing?hl=ko#basic-data-ess-sku]에 대해 다시 청구됩니다. 나머지 영업시간은 Place Details Enterprise SKU [https://developers.google.com/maps/documentation/places/ios-sdk/usage-and-billing?hl=ko#place-details-ent-sku]에 따라 요금이 청구됩니다. 프로젝트에서 isOpenWithRequest를 사용하는 경우 필요한 장소 세부정보를 가져오는지 확인하는 것이 좋습니다.

장소 ID

iOS용 Places SDK에서 사용되는 장소 ID [https://developers.google.com/maps/documentation/places/ios-sdk/place-id?hl=ko]는 Places API, Android용 Places SDK 및 기타 Google API에서 사용되는 식별자와 동일합니다. 각 장소 ID는 하나의 장소만 나타낼 수 있지만 하나의 장소에 두 개 이상의 장소 ID가 있을 수 있습니다.

장소에 새 장소 ID가 부여되는 경우가 있습니다.
예를 들어, 사업체를 새 위치로 이전하는 경우에 이러한 상황이 발생할 수 있습니다.

장소 ID를 지정하여 장소를 요청하면 장소가 여전히 존재하는 경우 항상 동일한 장소가 응답으로 반환됩니다. 하지만 응답에 요청의 장소 ID와 다른 장소 ID가 포함될 수 있습니다.

선택적 매개변수

GMSFetchPlaceRequest 객체를 사용하여 선택적 매개변수를 지정합니다.

regionCode
대답의 형식을 지정하는 데 사용되는 지역 코드입니다. 2자리 CLDR 코드 [https://www.unicode.org/cldr/charts/latest/supplemental/territory_language_information.html] 값으로 지정됩니다. 이 매개변수는 검색 결과에 편향 효과를 줄 수도 있습니다. 기본값은 없습니다.
응답의 주소 필드의 국가 이름이 지역 코드와 일치하면 국가 코드가 주소에서 생략됩니다.
대부분의 CLDR 코드는 ISO 3166-1 코드와 동일하지만 일부 주목할 만한 예외가 있습니다. 예를 들어 영국의 ccTLD는 'uk' (.co.uk)이지만 ISO 3166-1 코드는 'gb' (엄밀히 말해 '그레이트브리튼 북아일랜드 연합왕국'의 법인)입니다.
이 매개변수는 관련 법규에 따라 결과에 영향을 미칠 수 있습니다.

sessionToken
세션 토큰은 자동 완성(신규) 호출을 '세션'으로 추적하는 사용자 생성 문자열입니다. 자동 완성 (신규)은 세션 토큰을 사용하여 사용자 자동 완성 검색의 쿼리 및 장소 선택 단계를 결제 목적의 개별 세션으로 그룹화합니다. 세션 토큰은 Autocomplete (신규) 호출 다음에 오는 Place Details (신규) 호출에 전달됩니다. 자세한 내용은 세션 토큰 [https://developers.google.com/maps/documentation/places/ios-sdk/place-session-tokens?hl=ko]을 참고하세요.

앱에 특성 표시

앱에서 GMSPlacesClient [https://developers.google.com/maps/documentation/places/ios-sdk/reference/objc/Classes/GMSPlacesClient?hl=ko]에서 가져온 정보(예: 사진, 리뷰)를 표시하는 경우, 앱은 필수 저작자 표시도 표시해야 합니다.

예를 들어 GMSPlacesClient 객체의 reviews 속성에는 최대 5개의 GMSPlaceReview [https://developers.google.com/maps/documentation/places/ios-sdk/reference/objc/Classes/GMSPlaceReview?hl=ko] 객체 배열이 포함됩니다. 각 GMSPlaceReview 객체에는 출처 및 작성자 출처가 포함될 수 있습니다.
앱에 리뷰를 표시하는 경우 저작자 표시 또는 작성자 저작자 표시도 함께 표시해야 합니다.

자세한 내용은 저작자 표시 [https://developers.google.com/maps/documentation/places/ios-sdk/policies?hl=ko#other_attribution_requirements] 문서를 참고하세요.
