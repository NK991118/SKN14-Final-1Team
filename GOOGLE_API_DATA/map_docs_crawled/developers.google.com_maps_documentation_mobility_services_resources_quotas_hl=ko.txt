Title: 할당량 관리

할당량 관리

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

Fleet Engine API의 사용량을 관리하려면 모든 요청에 '분당 요청 수' 할당량 한도를 설정하세요.

다음 단계에 따라 Fleet Engine API의 할당량 한도를 보거나 변경하세요.

Cloud 콘솔에서 Google Maps Platform 할당량 페이지 [https://console.cloud.google.com/project/_/google/maps-apis/quotas?hl=ko]를 엽니다.
API 드롭다운을 클릭하고 'Local Rides and Deliveries API'를 선택합니다.
할당량 한도를 보려면 각 요청 유형의 카드를 엽니다.
요청 유형의 할당량 한도를 변경하려면 해당 한도의 수정 아이콘을 클릭합니다. 대화상자가 나타납니다. 할당량 한도 필드에 선호하는 '분당 요청 수' 할당량 한도를 입력하고 (Google에서 지정한 할당량 한도가 있다면 해당 한도까지) 저장을 선택합니다.
제공된 한도를 초과하여 프로젝트에 할당량이 더 필요한 경우 지원팀 [https://console.cloud.google.com/google/maps-apis/support/createcase?hl=ko]에 문의하세요.

고정 비율 제한

고정 비율 제한은 단일 리소스에서 작동하는 일부 API 메서드에 적용됩니다.
예를 들면 providers/project_id/vehicles/vehicle_id입니다. Google은 모든 고객에게 예외 없이 단일 리소스에 고정 비율 제한을 적용합니다. 예를 들어 프로젝트 내에서 동일한 vehicle_id에 대해 초당 3회 이상 maps.fleetengine.v1.VehicleService.UpdateVehicle를 호출하면 HTTP 429 또는 gRPC RESOURCE_EXHAUSTED 오류가 발생할 수 있습니다.

영향을 받는 속도 제한 요청은 Google Cloud 콘솔에서 할당량 오류로 집계되지 않습니다.

오류 처리 [https://cloud.google.com/apis/design/errors?hl=ko#handling_errors]에 대해 자세히 알아보세요.
