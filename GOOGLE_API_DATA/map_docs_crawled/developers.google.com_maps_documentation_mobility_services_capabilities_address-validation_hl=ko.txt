Title: 주소 캡처 및 유효성 검사

주소 캡처 및 유효성 검사

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

주소 캡처 및 유효성 검사 기능은 소비자의 부정확한 주소 입력, 배송 불가능한 주소, 배송 지점 위치 조정 필요 등 배송 주소와 관련된 여러 일반적인 문제를 해결하는 데 도움이 되는 모빌리티 서비스 툴킷입니다.

다음과 같은 서비스가 포함됩니다.

Places API [https://developers.google.com/maps/documentation/places/web-service?hl=ko]:
최종 사용자의 주소 입력을 간소화하고 라우팅 목적으로 사용할 수 있는 관심 장소에 관한 정보를 제공합니다.
Address Validation API [https://developers.google.com/maps/documentation/address-validation?hl=ko]:
주소가 실제 배송 가능한 장소를 나타내는지 확인하고 추가 주소 수준 메타데이터를 제공합니다.
Geocoding API [https://developers.google.com/maps/documentation/geocoding?hl=ko]: 지정된 주소의 위도와 경도를 제공합니다.

이 문서에서는 이러한 서비스를 사용하는 방법을 설명합니다.

주소 수집 및 확인 정보

정확하고 상세한 주소 정보는 배송 작업에 매우 중요합니다. 이 정보는 주소 정보에 의존하는 모든 다운스트림 활동의 품질에 영향을 미치기 때문입니다.
사용자 대상 애플리케이션을 통해 주소를 가져오든 API를 사용하여 업스트림 파트너로부터 주소를 가져오든 Google 모빌리티 서비스는 이 단계를 지원할 수 있습니다.

사용자 대상 애플리케이션을 통해 주소를 가져오든 API를 사용하여 업스트림 파트너로부터 주소를 가져오든 Google 모빌리티 서비스를 사용하면 다음과 같이 주소를 캡처할 수 있습니다.

소비자 환경 개선: 사용자가 올바른 주소를 입력하고 상품이 올바른 장소로 배송되도록 지원하는 도구를 제공합니다.
운전자 생산성 향상: 운전자가 배송 주소를 검색하는 시간을 줄이고 배송 시도를 반복할 필요성을 줄입니다.

주소 캡처

Places API에는 Place Autocomplete [https://developers.google.com/maps/documentation/places/web-service/autocomplete?hl=ko] 서비스가 포함되어 있습니다. 이 서비스는 사용자가 주소를 입력할 때 비즈니스, 주소, 관심 장소와 같은 장소를 반환하여 텍스트 기반 지리 검색을 위한 자동 완성 기능을 제공합니다.

온라인 소비자 또는 운송업체 사이트, 매장 내 직원 포털과 같이 배송 주소를 획득하는 사용자 대상 애플리케이션의 경우 Place Autocomplete를 사용하여 고객의 주소 입력 및 선택을 간소화하세요.

예를 들어 사용자가 택배 수거를 요청하고 수거 장소를 입력한다고 가정해 보겠습니다. Place Autocomplete를 사용하면 몇 글자만 입력하고 일치하는 목록에서 선택하여 주소를 제공할 수 있습니다.

Places API는 검색 결과를 제한하는 방법도 제공하므로 개발자는 추천 주소가 비즈니스에서 서비스를 제공하는 위치와 일치하는지 확인할 수 있습니다.

주소 확인

직접 사용자 입력 또는 API를 통해 주소를 캡처한 후 고객이 올바른 주소를 제공했는지 확인합니다. 즉, 주소 구성요소 누락이나 오타와 같은 부정확한 부분이 없는 주소여야 합니다. 주소 유효성 검사는 잘못된 주소를 포착하여 배송 실패를 줄이고 주소 세부정보를 더 잘 파악하여 배송 예측 가능성을 개선합니다.

Address Validation API [https://developers.google.com/maps/documentation/address-validation/overview?hl=ko]는 주소 구성요소를 검증하고, 우편 주소를 정규화하고, 가장 잘 알려진 위치를 찾습니다. 주소가 실제 장소를 가리키는지 파악하는 데 도움이 됩니다. 주소가 실제 장소를 참조하지 않는 경우 잘못되었을 수 있는 구성요소를 식별하여 사용자가 수정할 수 있도록 합니다.

Address Validation API에서 주소가 잘못되었거나 배송 불가능한 위치를 참조한다고 판단하는 경우 사용자에게 주소를 수정하라는 메시지를 표시할 수 있습니다. API 호출을 사용하여 주소를 가져온 경우 주소를 거부할 수 있습니다.

주소 위치 확인 허용

사용자가 주소를 입력하거나 선택하고 개발자가 주소를 검증한 후에는 Google Maps Platform을 사용하여 지도와 마커를 제공하여 주소의 위치를 나타냅니다.

Geocoding API [https://developers.google.com/maps/documentation/geocoding?hl=ko]를 사용하여 주소의 지리적 좌표를 확인합니다. 위치 외에도 Geocoding API는 주소 구성요소를 반환하며, 이를 사용하여 필요한 수정에 필요한 누락된 주소 부분을 가져올 수 있습니다. 또한 API에서 가져온 좌표의 정확도를 나타내는 위치 유형도 제공합니다.

지오코딩된 위치는 최적의 순서로 운전자에게 배송을 할당하는 등 다른 용도로도 사용할 수 있습니다. 자세한 내용은 경로 최적화 [https://developers.google.com/maps/documentation/mobility/services/capabilities/route-optimization?hl=ko]를 참고하세요.
참고: Address Validation API는 입력된 주소의 지오코드도 반환합니다. Geocoding API와 Address Validation API를 비교하는 자세한 내용은 주소와 지오코드 비교 [https://developers.google.com/maps/documentation/address-validation/requests-address?hl=ko]를 참고하세요.
주소 캡처를 위해 Google 모빌리티 서비스 사용

이 섹션에서는 주소 캡처 및 유효성 검사를 구현하는 방법에 관한 기본 정보를 제공합니다. 이 단계에서는 Google API 콘솔에서 프로젝트를 설정하고 올바른 서비스 계정과 결제가 사용 설정되어 있다고 가정합니다. Google 모빌리티 서비스와 관련된 자세한 내용은 프로젝트 설정에 관한 Fleet Engine 문서 [https://developers.google.com/maps/documentation/mobility/fleet-engine/essentials/set-up-fleet?hl=ko]를 참고하세요.

주소 입력 기능 사용 설정

Google API 콘솔의 API 및 서비스 섹션에서 Places API [https://developers.google.com/maps/documentation/places/web-service/cloud-setup?hl=ko]를 사용 설정합니다. Places API에는 Place Autocomplete 서비스가 포함됩니다.
선택적 매개변수 [https://developers.google.com/maps/documentation/places/web-service/autocomplete?hl=ko#optional-parameters] 섹션과 같이 사용할 수 있는 추가 기능은 Place Autocomplete [https://developers.google.com/maps/documentation/places/web-service/autocomplete?hl=ko] 문서를 참고하세요. 예를 들면 다음과 같습니다.
components 매개변수는 출력을 특정 국가로 제한합니다.
types 매개변수는 결과를 특정 위치 유형(예: 주소) 및 허용되는 사용자 입력 언어로 제한합니다.

주소 양식의 배포를 간소화하기 위해 Google Maps Platform은 Google API 콘솔 내에서 빠른 빌더 [https://console.cloud.google.com/google/maps-apis/build/address-selection?hl=ko]를 제공합니다.

배송지 주소 확인

Google API 콘솔의 API 및 서비스 섹션에서 Address Validation API [https://developers.google.com/maps/documentation/address-validation/cloud-setup?hl=ko]를 사용 설정합니다.
Address Validation API [https://developers.google.com/maps/documentation/address-validation/overview?hl=ko] 문서를 검토하여 주소 품질을 평가하고 사용자에게 추천을 표시하는 데 사용할 수 있는 다양한 데이터를 파악하세요.

수령 또는 배송 위치 확인 허용

모바일 및 웹용으로 선호하는 플랫폼에서 사용자를 위한 대화형 지도를 빌드하세요. 자세한 내용은 Maps JavaScript API [https://developers.google.com/maps/documentation/javascript/overview?hl=ko] 문서를 참고하고 여기에서 다른 플랫폼 안내로 이동하세요.
Address Validation API [https://developers.google.com/maps/documentation/address-validation/overview?hl=ko]에서 주소 지오코드를 가져오거나 Place Autocomplete의 장소 ID를 Geocoding API [https://developers.google.com/maps/documentation/javascript/geocoding?hl=ko]에 전달하여 위치에 마커를 추가합니다.
사용자가 배송 좌표를 조정할 수 있도록 마커를 드래그 가능하게 만드세요. 마커 동작(Android [https://developers.google.com/maps/documentation/android-sdk?hl=ko#marker], iOS [https://developers.google.com/maps/documentation/ios-sdk?hl=ko#marker], 웹 [https://developers.google.com/maps/documentation/javascript/markers?hl=ko#draggable]) 관련 문서를 참고하세요.
조정된 마커 위치의 지오코드를 가져옵니다. 업데이트된 지오코드를 리버스 지오코딩 API [https://developers.google.com/maps/documentation/geocoding/requests-reverse-geocoding?hl=ko]에 전달하여 해당 주소를 가져올 수도 있습니다. 그런 다음 주소 구성요소와 새 위치의 정확도를 평가할 수 있습니다.

참고: 애플리케이션 내에서 API 콘텐츠를 사용하는 방법에 관한 자세한 내용은 Google 모빌리티 서비스 계약의 약관을 참고하세요.
배송 주소에 관한 추가 컨텍스트 가져오기

주소가 결정되면 배송 주소에 관한 자세한 컨텍스트 데이터를 확보하는 것이 필수적인 다음 단계입니다. 이 정보를 통해 다음 작업을 할 수 있습니다.

운전자 생산성 향상: 운전자가 배송을 성공적이고 효율적으로 완료하는 데 필요한 모든 정보를 갖추도록 합니다. 다른 Google 모빌리티 서비스 기능과 결합하면 운전자가 적시에 적절한 장소에 도착할 수 있습니다.
비용 절감: 운전자가 올바른 위치로 효율적으로 이동하고 반복 배송을 줄이면 인건비와 차량 연료비도 절감할 수 있습니다.
운영 확장: 모든 주소 캡처 기능은 Google의 전 세계 지리정보 데이터베이스를 사용하므로 비즈니스를 확장하는 모든 지역에서 세부적인 커버리지를 제공합니다.

Google 모빌리티 서비스를 사용하면 주소와 위치에 관한 다양한 컨텍스트 데이터를 얻을 수 있습니다. 올바른 주소를 확인한 후 아래에 설명된 서비스를 호출하여 관련 정보를 가져올 수 있습니다. 이 정보는 경로 최적화 시스템의 입력으로 사용하거나 운전자 애플리케이션 내에서 운전자에게 추가 컨텍스트를 제공하는 데 사용할 수 있습니다.

Geocoding API는 입력된 주소를 Google 지도 데이터베이스의 위치와 일치시키고 위도/경도를 제공합니다.
Address Validation API는 다양한 검증 신호를 제공할 뿐만 아니라 입력 주소의 위도/경도도 제공합니다.
또한 위치에 대한 주거용 및 상업용 플래그와 미국 주소에 대한 USPS의 다양한 데이터를 제공합니다.
Place Details API는 비즈니스 영업시간을 비롯한 관심 장소에 관한 정보를 제공합니다.
