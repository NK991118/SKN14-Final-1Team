유럽 경제 지역 (EEA) 개발자
다음 엔드포인트 중 하나에 POST 요청을 실행하여 주소 유효성 검사 요청을 보냅니다.
validateAddress [https://developers.google.com/maps/documentation/address-validation/reference/rest/v1/TopLevel/validateAddress?hl=ko] (REST)
ValidateAddress [https://developers.google.com/maps/documentation/address-validation/reference/rpc/google.maps.addressvalidation.v1?hl=ko#google.maps.addressvalidation.v1.AddressValidation.ValidateAddress] (gRPC)
참고: 이 문서에서는 예시로 REST를 사용하지만 gRPC [https://www.grpc.io/]를 사용하는 경우에도 접근 방식이 유사합니다.
또한 요청은 유효성을 검사할 주소를 정의하는 요청에 JSON 요청 본문 [https://developers.google.com/maps/documentation/address-validation/reference/rest/v1/TopLevel/validateAddress?hl=ko#request-body]을 전달해야 합니다. 다음 요청 본문에는 하나의 필드와 세 개의 하위 필드가 포함되어 있습니다.
curl -X POST -d '{
  "address": {
    "regionCode": "US",
    "locality": "Mountain View",
    "addressLines": ["1600 Amphitheatre Pkwy"]
  }
}' \
-H 'Content-Type: application/json' \
"https://addressvalidation.googleapis.com/v1:validateAddress?key=API_KEY"
locality를 생략하고 addressLines 배열의 요소로 지정할 수도 있습니다.
{
  "address": {
    "regionCode": "US",
    "addressLines": ["1600 Amphitheatre Pkwy", "Mountain View, CA, 94043"]
  },
}
API 탐색기를 사용하면 API와 API 옵션을 숙지할 수 있도록 라이브 요청을 할 수 있습니다.
사용해 보기 [https://developers.google.com/maps/documentation/address-validation/requests-validate-address?hl=ko#try_it]
주소 요청 필드
요청 본문은 단일 필드로 구성됩니다.
address: PostalAddress [https://developers.google.com/maps/documentation/address-validation/reference/rest/v1/TopLevel/validateAddress?hl=ko#postaladdress] 유형의 필드로, addressLines에 항목이 하나 이상 포함되어야 합니다.
address 필드를 Address Validation API에 보낼 때는 addressLines 필드 하나만 제공하면 됩니다. 하지만 가장 간단한 주소 요청에는 다음 하위 필드가 포함되어야 합니다.
regionCode - 권장 선택사항 필드이지만 생략하면 Address Validation API가 주소에서 지역을 추론합니다. 하지만 최상의 결과를 얻으려면 regionCode를 알고 있는 경우 포함하세요. 지원되는 리전 목록은 지원되는 리전 [https://developers.google.com/maps/documentation/address-validation/coverage?hl=ko]을 참고하세요.
addressLines - 필수 주소의 하위 구조를 설명하는 주소가 포함된 배열입니다.
참조 가이드에서 PostalAddress [https://developers.google.com/maps/documentation/address-validation/reference/rest/v1/TopLevel/validateAddress?hl=ko#postaladdress]를 확인하세요.
USPS CASS™ 옵션
미국 우편 서비스 (USPS®)1 [https://developers.google.com/maps/documentation/address-validation/requests-validate-address?hl=ko#fn1]는 주소 유효성 검사 제공업체를 지원하고 인증하기 위해 우편번호 정확도 지원 시스템 (CASS™) [https://postalpro.usps.com/certifications/cass]을 유지합니다.
참고: Address Validation API는 CASS-Certified™ 서비스로 확인되었습니다. 즉, 서비스가 누락된 주소 정보를 입력하고, 표준화하고, 가장 최신이고 정확한 주소로 업데이트할 수 있음이 확인된 것입니다.
미국 및 푸에르토리코 지역의 경우 요청 본문에서 enableUspsCass을 true로 설정하여 CASS 처리를 사용 설정할 수 있습니다. CASS를 사용할 때 최상의 결과를 얻으려면 도로명 및 건물 번호와 함께 도시, 주, 우편번호가 포함된 주소를 입력하세요.
{
  "address": {
    "regionCode": "US",
    "locality": "Mountain View",
    "administrativeArea": "CA",
    "postalCode": "94043",
    "addressLines": ["1600 Amphitheatre Pkwy"]
  },
  "enableUspsCass": true
}
참고: CASS 처리가 필요하지 않거나 미국 및 푸에르토리코 지역 외부의 주소를 검증하는 경우 이 기능을 사용 설정하지 않는 것이 좋습니다.
