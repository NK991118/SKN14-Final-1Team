Title: 프리미엄 플랜 인증 개요

프리미엄 플랜 인증 개요

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

중요: Google Maps Platform 프리미엄 플랜은 이제 가입이 불가능하며 신규 고객에게 제공되지 않습니다.

신규 고객이 Google Maps Platform API 및 SDK를 사용하려면 API 키를 사용해야 합니다. 프리미엄 플랜에서 사용한 만큼만 지불하는
Google Maps Platform 가격 모델로 이전한 고객은
계속해서 기존 API 키 또는 클라이언트 ID를 사용하여
요청을 인증할 수 있습니다.

Google Maps Platform은 API 키를 사용하여 요청을 인증하는 방법으로 표준화될 예정입니다. 클라이언트 ID 사용 종료일은 현재 확정되지 않았지만 새로운 구현 시 클라이언트 ID 대신 API 키를 사용하여 빌드하시기 바랍니다. 자세한 내용은 사용 가능한 인증 방법 [https://developers.google.com/maps/premium/authentication/overview?hl=ko#available-authentication-methods]을 참고하세요.

주의: 클라이언트 ID와 API 키가 모두 포함된 요청은 예기치 않은 API 동작이나 의도하지 않은 결제 동작을 초래할 수 있습니다. 이를 방지하려면 요청에서 두 매개변수 중 하나만 사용해야 합니다.

클라이언트 ID 관리

클라이언트 ID 프로젝트 찾기

Google Maps Platform 프리미엄 플랜에서 기존에 제공하던 기능에 계속 액세스하려는 고객은 기존 프리미엄 애셋과 연결된 프로젝트를 계속 사용해야 합니다. Google Maps Platform의 사용한 만큼만 지불하는 가격 모델로 이전한 고객도 마찬가지입니다.

프리미엄 플랜 라이선스를 처음 구매하면 gme-[company] & proj-[projectnumber] ([type]) 형식의 프리미엄 애셋 이름이 제공됩니다. 애셋 이름과 기존 프로젝트 소유자는 환영 메일에 기재되어 있습니다.
중요: 사용자가 Cloud 콘솔에서 모든 기능에 액세스하고 클라이언트 ID를 관리하려면 기존 프리미엄 플랜 프로젝트의 소유자이거나 편집자여야 합니다.
자세한 내용은 Google Cloud IAM(Identity and Access Management) 문서인
역할 이해 [https://cloud.google.com/iam/docs/understanding-roles?hl=ko]를
참고하세요.
다음 URL을 사용하여 Google Cloud 콘솔에 로그인합니다.
https://console.cloud.google.com/project/projectnumber

projectnumber를 프로젝트 번호로 바꿉니다.

Cloud 콘솔에서 클라이언트 ID 관리

주의:
이
섹션 의 경우
클라이언트 ID를 사용하는 기존 애플리케이션의 유지 관리를 위한 참고 용도로만 제공됩니다.
새 애플리케이션의 경우 항상 API 키를 사용하세요.

Google Cloud 콘솔에서 클라이언트 ID를 관리하는 위치

프리미엄 플랜 클라이언트 ID 관리 기능은
Cloud 콘솔 [https://console.cloud.google.com/project/_/google/maps-apis/credentials?utm_source=Docs_Credentials&hl=ko]의
Google Maps Platform 사용자 인증 정보 페이지 하단에 있는
클라이언트 ID 섹션에서 사용할 수 있습니다.

URL 승인 및 클라이언트 ID 서명 비밀번호 관리를 포함한 기타 클라이언트 ID 관리 작업은 별도의
클라이언트 ID 페이지 [https://console.cloud.google.com/project/_/google/maps-apis/clientid?hl=ko]에서
클라이언트 ID 섹션의 오른쪽 끝에 있는
edit 수정 아이콘을 클릭하여 액세스할 수 있습니다.

클라이언트 ID 일시중지

Google Maps Platform 사용자 인증 정보 [https://console.cloud.google.com/project/_/google/maps-apis/credentials?utm_source=Docs_Credentials&hl=ko] 페이지에서 클라이언트 ID 섹션의 오른쪽 끝에 있는 pause 일시중지 아이콘을 클릭하여 클라이언트 ID를 일시중지하거나 일시중지를 해제할 수 있습니다.
주의: 클라이언트 ID를 일시중지하면 일시중지된 클라이언트 ID에 대한 모든 Google Maps Platform API 호출이 거부됩니다. 클라이언트 ID를 일시중지했다면 최대 30일 이내에 일시중지를 해제해야 합니다. 이 기간에 일시중지를 해제하지 않으면 클라이언트 ID가 만료되며 30일이 더 지나면 (총 60일 후) Cloud 콘솔에 더 이상 표시되지 않습니다.
클라이언트 ID URL 승인

승인된 URL은 모두 클라이언트 ID 페이지 [https://console.cloud.google.com/project/_/google/maps-apis/clientid?hl=ko]의 gme-[company]의 클라이언트 ID에 대해 승인된 URL 표에서 확인할 수 있습니다. 자세한 내용은 승인된 URL 관리 [https://developers.google.com/maps/premium/authentication/client-id/url-authorization?hl=ko#registering-authorized-urls]를 참고하세요.

클라이언트 ID 서명 비밀번호 가져오기

클라이언트 ID의 URL 서명 비밀번호 암호화 키는 클라이언트 ID 페이지 [https://console.cloud.google.com/project/_/google/maps-apis/clientid?hl=ko]의 클라이언트 ID 오른쪽에 있습니다. 자세한 내용은 요청 서명 [https://developers.google.com/maps/premium/authentication/client-id/request-signing?hl=ko#signing-requests]을 참고하세요.
