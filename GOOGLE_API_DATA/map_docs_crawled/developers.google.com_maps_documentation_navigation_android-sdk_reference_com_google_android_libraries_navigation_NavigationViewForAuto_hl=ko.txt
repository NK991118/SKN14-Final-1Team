public  final   class
NavigationViewForAuto
extends FrameLayout

A View which displays a map for navigation in Android Auto.
Users of this class must forward all the life cycle methods from the Activity [https://developer.android.com/reference/android/app/Activity.html] or Fragment [https://developer.android.com/reference/android/app/Fragment.html] containing this view to the corresponding
ones in this class. In particular, you must forward on the following methods:

onCreate(Bundle) [https://developers.google.com/maps/documentation/navigation/android-sdk/reference/com/google/android/libraries/navigation/NavigationViewForAuto#onCreate(android.os.Bundle)]
onStart() [https://developers.google.com/maps/documentation/navigation/android-sdk/reference/com/google/android/libraries/navigation/NavigationViewForAuto#onStart()]
onResume() [https://developers.google.com/maps/documentation/navigation/android-sdk/reference/com/google/android/libraries/navigation/NavigationViewForAuto#onResume()]
onPause() [https://developers.google.com/maps/documentation/navigation/android-sdk/reference/com/google/android/libraries/navigation/NavigationViewForAuto#onPause()]
onStop() [https://developers.google.com/maps/documentation/navigation/android-sdk/reference/com/google/android/libraries/navigation/NavigationViewForAuto#onStop()]
onDestroy() [https://developers.google.com/maps/documentation/navigation/android-sdk/reference/com/google/android/libraries/navigation/NavigationViewForAuto#onDestroy()]
onConfigurationChanged(Configuration) [https://developers.google.com/maps/documentation/navigation/android-sdk/reference/com/google/android/libraries/navigation/NavigationViewForAuto#onConfigurationChanged(android.content.res.Configuration)]
onSaveInstanceState(Bundle) [https://developers.google.com/maps/documentation/navigation/android-sdk/reference/com/google/android/libraries/navigation/NavigationViewForAuto#onSaveInstanceState(android.os.Bundle)]
onTrimMemory(int) [https://developers.google.com/maps/documentation/navigation/android-sdk/reference/com/google/android/libraries/navigation/NavigationViewForAuto#onTrimMemory(int)]

If the NavigationViewForAuto is added after its parent Activity [https://developer.android.com/reference/android/app/Activity.html] or Fragment [https://developer.android.com/reference/android/app/Fragment.html] has been through its creation lifecycle, you must call onCreate(Bundle) [https://developers.google.com/maps/documentation/navigation/android-sdk/reference/com/google/android/libraries/navigation/NavigationViewForAuto#onCreate(android.os.Bundle)],
onStart() [https://developers.google.com/maps/documentation/navigation/android-sdk/reference/com/google/android/libraries/navigation/NavigationViewForAuto#onStart()] and onResume() [https://developers.google.com/maps/documentation/navigation/android-sdk/reference/com/google/android/libraries/navigation/NavigationViewForAuto#onResume()] manually to properly initialize it. Similarly, if a
NavigationViewForAuto is detached from its parent Activity [https://developer.android.com/reference/android/app/Activity.html] or Fragment [https://developer.android.com/reference/android/app/Fragment.html], you must manually call onPause() [https://developers.google.com/maps/documentation/navigation/android-sdk/reference/com/google/android/libraries/navigation/NavigationViewForAuto#onPause()], onStop() [https://developers.google.com/maps/documentation/navigation/android-sdk/reference/com/google/android/libraries/navigation/NavigationViewForAuto#onStop()] and onDestroy() [https://developers.google.com/maps/documentation/navigation/android-sdk/reference/com/google/android/libraries/navigation/NavigationViewForAuto#onDestroy()] to prevent leaks.
Note: Adding children to this view is not supported.
All calls to NavigationViewForAuto should be performed on the UI thread.
