이 페이지에는 Address Validation API 클라이언트 라이브러리를 사용하여 다음 서비스를 호출하는 방법의 예가 나와 있습니다.

validateAddress [https://developers.google.com/maps/documentation/address-validation/reference/rest/v1/TopLevel/validateAddress?hl=ko]
provideValidationFeedback [https://developers.google.com/maps/documentation/address-validation/reference/rest/v1/TopLevel/provideValidationFeedback?hl=ko]

클라이언트 라이브러리 설치

설치 안내는 Address Validation API 클라이언트 라이브러리 [https://developers.google.com/maps/documentation/address-validation/client_libraries?hl=ko#installing_the_client_library]를 참고하세요.

인증

클라이언트 라이브러리를 사용할 때는 애플리케이션 기본 사용자 인증 정보(ADC) [https://cloud.google.com/docs/authentication/application-default-credentials?hl=ko]를 사용하여 인증합니다. ADC 설정은 애플리케이션 기본 사용자 인증 정보에 대한 사용자 인증 정보 제공 [https://cloud.google.com/docs/authentication%0Aprovide-credentials-adc?hl=ko]을 참고하세요. 클라이언트 라이브러리에서 ADC를 사용하는 방법은 클라이언트 라이브러리를 사용하여 인증 [https://cloud.google.com/docs/authentication/client-libraries?hl=ko]을 참고하세요.

API 키를 사용하여 클라이언트 라이브러리에 인증할 수도 있습니다. 예를 들면 다음과 같습니다.
using Google.Maps.AddressValidation.V1;
using Google.Api.Gax.Grpc;
using Grpc.Core;
...
ValidateAddressSettings = defaultSettings.ValidateAddressSettings.MergedWith(apiHeader),
ProvideValidationFeedbackSettings = defaultSettings.ProvideValidationFeedbackSettings.MergedWith(apiHeader)
AddressValidationClientBuilder builder = new AddressValidationClientBuilder
Settings = settings,
ChannelCredentials = new SslCredentials()
AddressValidationClient client = await builder.BuildAsync();

애플리케이션에서 API 키를 사용하는 경우 저장 및 전송 중에 안전하게 보호되도록 해야 합니다. API 키를 공개적으로 노출하면 계정에 예상치 않은 비용이 부과될 수 있습니다.

이 페이지의 예시에서는 애플리케이션 기본 사용자 인증 정보를 사용합니다.

예

validateAddress

다음은 .NET 클라이언트 라이브러리를 사용하여 validateAddress [https://developers.google.com/maps/documentation/address-validation/reference/rest/v1/TopLevel/validateAddress?hl=ko]를 호출하는 방법의 예입니다.
using Google.Maps.AddressValidation.V1;
using Google.Type;
...
private static async Task CallAddressValidation()
AddressValidationClient client = await AddressValidationClient.CreateAsync();
Address = new PostalAddress
RegionCode = "US",
LanguageCode = "en",
PostalCode = "94043",
AdministrativeArea = "CA",
Locality = "Mountain View",
AddressLines = { "1600 Amphitheatre Parkway" }
ValidateAddressResponse response = await client.ValidateAddressAsync(request);
RegionCode, Locality, AddressLines과 같은 구성요소화된 주소 필드가 있는 PostalAddress 객체를 정의합니다. 예시 요청을 빌드할 때 이 PostalAddress는 ValidateAddressRequest를 만드는 데 사용됩니다. 그런 다음 ValidateAddressAsync 메서드를 호출하여 요청을 하고 ValidationGranularity 및 FormattedAddress과 같은 응답의 세부정보가 출력됩니다.

provideValidationFeedback

다음은 .NET 클라이언트 라이브러리를 사용하여 provideValidationFeedback [https://developers.google.com/maps/documentation/address-validation/reference/rest/v1/TopLevel/provideValidationFeedback?hl=ko]를 호출하는 방법의 예입니다.
using Google.Maps.AddressValidation.V1;
...
private static async Task ProvideValidationFeedback()
AddressValidationClient client = await AddressValidationClient.CreateAsync();

Conclusion = ProvideValidationFeedbackRequest.Types.ValidationConclusion.ValidatedVersionUsed,
ResponseId = "Response_ID"
