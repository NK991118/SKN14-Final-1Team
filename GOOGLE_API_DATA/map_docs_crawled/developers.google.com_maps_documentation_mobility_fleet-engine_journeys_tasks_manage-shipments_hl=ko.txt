Title: 배송 관리

배송 관리

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

이 문서에서는 Fleet Engine으로 배송을 관리할 수 있는 여러 관련 활동을 설명합니다.

배송 경로 다시 지정

배송 작업이 생성되면 계획된 위치를 변경할 수 없습니다. 배송을 재경로 지정하려면 결과를 설정하지 않고 배송 작업을 닫은 다음 업데이트된 계획된 위치로 새 작업을 만들고 닫힌 작업에 사용된 것과 동일한 추적 ID를 포함합니다. 새 작업을 만든 후 동일한 차량에 작업을 할당합니다. 자세한 내용은 작업 종료 [https://developers.google.com/maps/documentation/mobility/fleet-engine/journeys/tasks/finalize-tasks?hl=ko#close-task] 및 배송 차량 작업 업데이트 [https://developers.google.com/maps/documentation/mobility/fleet-engine/journeys/tasks/update-tasks?hl=ko]를 참고하세요.

배송 상태 및 기타 메타 정보 저장

배송 작업이 완료되면 작업 상태와 결과가 작업에 기록됩니다. 하지만 배송과 관련된 다른 메타 정보를 업데이트해야 할 수도 있습니다. Fleet Engine 서비스 외부에서 참조할 수 있는 기타 메타 정보를 저장하려면 외부 테이블에서 작업과 연결된 tracking_id을 키로 사용하세요.
