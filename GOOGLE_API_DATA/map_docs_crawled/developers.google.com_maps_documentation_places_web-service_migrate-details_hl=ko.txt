소개

Places API는 장소 세부정보 (기존) [https://developers.google.com/maps/documentation/places/web-service/details?hl=ko]를 지원합니다.
Places API (기존)를 잘 알고 있다면 Place Details (신규) [https://developers.google.com/maps/documentation/places/web-service/place-details?hl=ko]에서 다음과 같이 변경됩니다.

필드 마스킹이 필요합니다. 응답에서 반환할 필드를 지정해야 합니다. 반환된 필드의 기본 목록은 없습니다. 이 목록을 생략하면 메서드에서 오류를 반환합니다. 자세한 내용은 FieldMask [https://developers.google.com/maps/documentation/places/web-service/place-details?hl=ko#fieldmask]를 참고하세요.
Place Details (New)는 인증 메커니즘으로 API 키 [https://developers.google.com/maps/documentation/places/web-service/get-api-key?hl=ko]와 OAuth [https://developers.google.com/maps/documentation/places/web-service/oauth-token?hl=ko] 토큰을 모두 지원합니다.
장소 세부정보 (신규)에서는 JSON만 응답 형식으로 지원됩니다.
Places API (New)의 JSON 응답 형식이 기존 API의 형식과 다릅니다. 자세한 내용은 Places API 응답 이전 [https://developers.google.com/maps/documentation/places/web-service/migrate-response?hl=ko]을 참고하세요.

다음 표에는 장소 세부정보 (신규)에서 이름이 바뀌거나 수정된 장소 세부정보 (기존)의 매개변수 또는 더 이상 지원되지 않는 매개변수가 나열되어 있습니다.

| 현재 매개변수 | 새 매개변수 | 참고 |
| --- | --- | --- |
| place_id | places/PLACE_ID | places/PLACE_ID 문자열을 장소 리소스 이름이라고도 합니다. 장소 세부정보 (신규), 주변 지역 검색 (신규), 텍스트 검색 (신규) 요청의 응답에서 이 문자열은 응답의 name 필드에 포함됩니다. 독립형 장소 ID는 응답의 id 필드에 포함됩니다. |
| language | languageCode |  |
| region | regionCode |  |

요청 예시

다음 예시 GET 요청은 장소 세부정보 (기존)를 사용합니다. 이 예에서는 장소 ID에 관한 정보가 포함된 JSON 응답을 요청하고 모든 매개변수를 URL 매개변수로 전달합니다. 이 예시에서는 fields도 사용하므로 응답에는 장소의 표시 이름과 형식화된 주소만 포함됩니다.

curl -L -X GET \
'https://maps.googleapis.com/maps/api/place/details/json?place_id=ChIJj61dQgK6j4AR4GeTYWZsKWw&fields=name%2Cformatted_address&key=API_KEY'

Place Details (New)를 사용하면 GET 요청을 수행하고 요청의 일부로 URL 매개변수와 헤더에 모든 매개변수를 전달합니다. 이 예시에서는 필드 마스크도 사용하므로 응답에는 장소의 표시 이름과 형식화된 주소만 포함됩니다.

curl -X GET -H 'Content-Type: application/json' \
-H "X-Goog-Api-Key: API_KEY" \
-H "X-Goog-FieldMask: displayName,formattedAddress" \
https://places.googleapis.com/v1/places/ChIJj61dQgK6j4AR4GeTYWZsKWw

places/PLACE_ID 문자열을 장소 리소스 이름이라고도 합니다. Place Details (New), Nearby Search (New), Text Search (New) 요청의 응답에서 이 문자열은 응답의 name 필드에 포함됩니다. 독립형 장소 ID는 응답의 id 필드에 포함됩니다.
참고: 장소 세부정보 (이전) [https://developers.google.com/maps/documentation/places/web-service/details?hl=ko]에서 응답의 name 필드에 인간이 읽을 수 있는 장소 이름이 포함되었습니다. 새 API에서는 이 필드를 displayName이라고 합니다.
