A

활동
운전자 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#driver]가 할당 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#assignment]의 일부로 수행하는 작업(주로 작업 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#task])의 모음입니다(예: 패키지 수령 및 배송). 예약된 작업 [https://developers.google.com/maps/documentation/mobility/fleet-engine/essentials/tasks-intro?hl=ko] 문서의 작업 유형 [https://developers.google.com/maps/documentation/mobility/fleet-engine/essentials/tasks-intro?hl=ko#task-types]도 참고하세요.
할당
비즈니스와 고객 간 거래의 일환으로 운전자 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#driver]가 수락하고 완료하는 작업 항목입니다. 할당에는 트럭의 모든 패키지를 배송하는 것과 같은 여러 활동 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#activity]이 포함될 수 있습니다.
ADC
애플리케이션 기본 사용자 인증 정보 Google Cloud 클라이언트 라이브러리가 Google Cloud 서비스 및 API를 사용하여 자동으로 인증하는 데 사용하는 메서드입니다.
이를 통해 코드 변경 없이 개발 및 프로덕션 환경에서 동일한 사용자 인증 정보를 애플리케이션이 사용할 수 있으므로 프로세스가 간소화됩니다. 자세한 내용은 애플리케이션 기본 사용자 인증 정보의 작동 방식 [https://cloud.google.com/docs/authentication/application-default-credentials?hl=ko]을 참고하세요.
실제
계획 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#planned]된 것이 아니라 어떤 일이 발생했음을 나타내는 수정자입니다. 경로 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#route], 정류장 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop], 위치, 도착 시간과 같은 다양한 유형의 데이터에 적용됩니다.
인증 (: id="authentication")
요청을 실행하는 항목의 ID를 확인합니다. 인증의 경우 Fleet Engine은 신뢰도가 높은 환경에는 애플리케이션 기본 사용자 인증 정보 (ADC)를 사용하고 신뢰도가 낮은 환경에는 JSON 웹 토큰 (JWT)을 사용합니다.
승인 (: id="authorization")
인증된 항목이 액세스할 수 있는 리소스를 지정합니다. 승인을 위해 Fleet Engine은 Google Cloud IAM 역할이 있는 서비스 계정과 요청하는 데이터를 볼 수 있거나 변경할 수 있는 권한이 엔티티에 있는지 확인하는 JWT 클레임을 사용합니다.

B

기본 지도
지리 정보 시스템 (GIS)의 다른 레이어에 대한 참조 정보를 제공하는 배경 지도입니다. 일반적으로 베이스맵에는 주요 도로, 해안선, 행정구역과 같은 지리적 특징이 포함됩니다.

D

delivery
예약된 중지 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop]로의 예약된 상품 배송 또한 드라이버 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#driver]가 패키지로 무엇을 하고 있는지 나타내는 작업 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#task] 유형도 있습니다.
배송 작업
Fleet Engine에서 기록한 예약된 작업 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#task] 유형으로, 상품을 배송하거나 소비자의 작업을 완료하는 데 사용됩니다. 예약된 작업 [https://developers.google.com/maps/documentation/mobility/fleet-engine/essentials/tasks-intro?hl=ko] 문서의 작업 유형 [https://developers.google.com/maps/documentation/mobility/fleet-engine/essentials/tasks-intro?hl=ko#task-types]도 참고하세요.
destination
드라이버 활동 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#driver_activity]의 의도된 위치입니다. 배송 작업 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#delivery_task]의 예약된 중지 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop] 또는 주문형 이동 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#on_demand_trip]의 경유지 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#waypoint]일 수 있습니다.
드라이버
차량에 탑승한 동안 및 과제 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#assignment]를 수행하는 동안 사용하는 기기의 앱을 통해 차량과 연결된 사람
운전자 활동
활동 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#activity]을 참고하세요.
중도 이탈
운전자 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#driver]가 사람이나 물품을 태운 후 내려주는 정류장 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop]입니다.

F

Fleet Engine
Fleet Engine은 API 및 SDK와 상호 운용하여 매핑, 라우팅, 위치 관리를 개선하는 백엔드 서비스입니다. Fleet Engine 서비스란 무엇인가요? [https://developers.google.com/maps/documentation/mobility/fleet-engine/essentials?hl=ko]를 참고하세요.
Fleet 요금제
차량 모음의 정류장 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop] 순서가 지정된 목록으로 구성된 행렬입니다.
Fleet Engine 서비스
Fleet Engine [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#fleet_engine]을 참고하세요.

G

상품
할당 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#assignment]의 일부로 이동된 상품(예: 패키지 또는 음식)
gRPC
Google에서 처음 만든 크로스 플랫폼 오픈소스 리모트 프로시져 콜 프레임워크입니다. gRPC [https://wikipedia.org/wiki/GRPC]를 참고하세요.

H

I

여행 일정
차량의 현재 할당 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#assignment] 컬렉션입니다.
예약된 서비스의 경우 여정에는 매니페스트 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#manifest]와 배송 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#delivery] 순서가 포함됩니다.
주문형 이동 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#on_demand_trip]의 경우 여정에는 특정 기간에 할당된 이동 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#trip] 수가 포함되며, 일반적으로 이동은 하나입니다.
Cloud Fleet Routing [https://cloud.google.com/optimization/docs/overview?hl=ko]의 경우 여정을 투어 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#tour]라고 합니다.

J

여정
출발지 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#origin]에서 특정 할당 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#assignment]이 종료되는 지점까지의 차량 경로 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#route]입니다. 예를 들어 한 사람이 이용하는 차량 공유 이동이나 특정 정류장 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop]에서 배송되는 택배의 배송 여정이 있습니다. 여정은 운전기사 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#driver]의 할당 중 공유하려는 부분(예: 상품 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#goods] 배송 상태를 확인하는 경우)이기도 합니다.
여정 공유
주문형 이동 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#on_demand_trip]의 진행 상황을 소비자와 공유하거나 예약된 작업 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#scheduled_task]을 차량 관리자와 공유하는 기능
소비자 SDK의 여정 공유, Maps JavaScript API의 이동 및 주문 진행 상황, 배송 추적, 차량 추적 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#journey]이 포함됩니다. [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#journey]
JWT
JSON 웹 토큰입니다. 두 당사자 간에 전송될 클레임을 나타내는 콤팩트하고 URL 안전한 수단입니다. 기본적으로 시스템 간에 정보를 안전하게 전송하는 독립형 방식으로, 웹 애플리케이션의 인증 및 승인에 일반적으로 사용됩니다. JWT는 OAuth 2.0 및 OpenID에서 자주 사용됩니다.

L

다리(신체 부위)
경로 세그먼트 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#route_segment]를 참고하세요.
위치 정보 제공자
배송 추적과 차량 추적 모두에 JavaScript 라이브러리에서만 사용되며 추적된 객체의 위치 정보를 제공하는 인터페이스입니다.
이 서비스를 통해 소비자와 차량 운영자는 지도에서 작업 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#task]의 진행 상황을 확인할 수 있습니다.

M

매니페스트
배송 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#delivery] 순서와 관계없이 단일 차량으로 배송할 모든 상품의 목록입니다.

N

O

온디맨드 이동
운전자 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#driver]가 사람이나 음식을 픽업하고 드롭오프하도록 소비자가 요청한 여정 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#journey]입니다.
주문형 이동이란 무엇인가요? [https://developers.google.com/maps/documentation/mobility/fleet-engine/journeys/trips?hl=ko]를 참고하세요.
최적화된 경로
거리나 시간과 같은 기준에 따라 최적화된 정류장 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop] 간의 가장 효율적인 경로
origin
여정 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#journey]의 첫 번째 경유지 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#waypoint]입니다.

P

승객
차량 공유 활동 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#rideshare_activity]이 할당된 차량의 탑승자입니다.
픽업 작업
상품 수령을 나타내기 위해 Fleet Engine에서 기록한 작업 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#task] 유형입니다. 상응하는 배송 작업 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#delivery_task]이 있어야 합니다.
예약된 작업 [https://developers.google.com/maps/documentation/mobility/fleet-engine/journeys/tasks?hl=ko] 문서의 작업 유형 [https://developers.google.com/maps/documentation/mobility/fleet-engine/essentials/tasks-intro?hl=ko#task-types]도 참고하세요.
계획됨
[https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#actual]실제로 발생한 것이 아니라 발생할 예정임을 나타내는 수정자입니다. [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#actual] 경로 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#route], 정류장 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop], 위치, 도착 시간과 같은 다양한 유형의 데이터에 적용됩니다.
다중선
연속된 점 사이에 선 세그먼트가 그려지는 점 목록입니다. 지도에서 여러 지점을 연결하여 경로 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#route] 등을 만드는 선을 의미합니다.
제공업체 ID
Google의 고객인 귀하로부터 요청이 전송되었음을 식별하는 Fleet Engine 엔드포인트 요청에 사용되는 ID입니다(Google Cloud 프로젝트 ID와 동일).

R

차량 공유 활동
하나 이상의 소비자가 요청된 위치에서 승차하거나 하차하는 활동 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#activity]입니다.
경로
시작, 종료, 중간 경유지 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#waypoint] (선택사항)를 연결하는 일련의 연결된 경로 세그먼트 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#route_segment]입니다.
경로에는 경로 세그먼트 간 시간과 거리가 포함됩니다.
Routes API [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#routes_api]도 참고하세요.
Route Optimization API
Route Optimization API는 차량에 작업과 경로를 할당하여 운송 목표에 대해 제공하는 목표와 제약 조건에 따라 최적화합니다. Route Optimization API [https://developers.google.com/maps/documentation/route-optimization?hl=ko]를 참고하세요.
경로
단일 차량의 정류장 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop]의 순서가 지정된 목록입니다.
배송 경로 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#shipment_route]도 참고하세요.
경로 세그먼트
인접한 두 정류장 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop] 사이의 경로 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#route]의 특정 부분으로, 폴리라인 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#polyline], 시간, 두 정류장 간 거리가 포함됩니다. Routes API [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#routes_api]에서는 구간 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#leg]이라고도 합니다.
Routes API
중간 경유지 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#waypoint]를 포함하여 출발지 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#origin]에서 목적지 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#destination] 위치까지의 최적 경로를 찾는 데 도움이 되는 API입니다.
예를 들어 Routes API는 출발지 및 목적지 위치의 행렬에 대한 도착예정시간과 거리를 계산합니다. Routes API 문서 [https://developers.google.com/maps/documentation/routes?hl=ko]를 참고하세요.
라우팅
사람 또는 차량의 출발지 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#origin]와 목적지 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#destination] 사이의 경로를 찾는 프로세스입니다.

S

중지 예약됨
Fleet Engine에서 기록한 배송 외 작업 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#task]으로, 드롭박스에서 수집하는 등 특정 위치에서 정차 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop]가 필요합니다.
Fleet Engine 작업 유형 [https://developers.google.com/maps/documentation/mobility/services/resources/1?hl=ko]을 참고하세요.
예약된 작업
운전기사 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#driver]를 위해 계획된 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#planned] 작업 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#task]의 연속입니다(예: 택배 수령, 현장 서비스를 제공하기 위해 특정 위치에 정차). 예약된 작업 [https://developers.google.com/maps/documentation/mobility/fleet-engine/essentials/tasks-intro?hl=ko]도 참고하세요.
배송
예약된 서비스에 의해 소비자에게 배송된 상품입니다.
상품을 수령하고 배송하는 것과 관련된 과제 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#assignment] 유형이기도 합니다.
배송 경로
단일 차량의 정류장 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop]의 순서가 지정된 목록입니다. 경로 경로 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#route_path]도 참고하세요.
도로 옆 경유지
Routes API [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#routes_api]에서 차량이 멈춰야 하는 도로 측면을 나타내는 경유지 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#waypoint] 속성으로 사용됩니다.
중지
차량이 정차하는 경로 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#route]상의 위치입니다. 예약된 서비스에서 각 정류장 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop]에는 하나 이상의 작업 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#task]이 있을 수 있습니다.

T

태스크
작업은 차량의 운전자 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#driver]가 활동 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#activity] 내에서 취하는 단일 예약 및 로깅된 작업을 나타냅니다. 각 작업은 정류장 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop]과 연결됩니다. 예를 들어 패키지 수령 (수령 작업), 주유 또는 필수 휴식 (이용 불가능 작업), 패키지 배송(배송 작업)이 있습니다. 차량 공유의 경우 태스크에는 승객 승하차가 포함됩니다. 작업에는 상태가 있으며 열거나 닫을 수 있고, 닫힌 작업은 성공하거나 실패할 수 있습니다. 작업은 예약된 서비스가 소비자의 거래를 식별하고 종료하는 데 사용하는 핵심 메커니즘입니다. 모든 유형의 작업 목록은 Task.Types [https://developers.google.com/maps/documentation/mobility/services/resources/5?hl=ko]를 참고하세요. 예약된 작업 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#scheduled_tasks]도 참고하세요.
여행
승차와 하차를 포함하는 할당 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#assignment] 유형입니다. 예를 들어 승객을 태우고 내려주는 것이 여정입니다.

U

이용 불가 작업
운전자가 휴식을 취하거나 차량에 연료를 보급하는 등 차량이 서비스를 이용할 수 없음을 나타내는 작업 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#task]입니다. 예약된 작업 [https://developers.google.com/maps/documentation/mobility/fleet-engine/essentials/tasks-intro?hl=ko] 문서의 작업 유형 [https://developers.google.com/maps/documentation/mobility/fleet-engine/essentials/tasks-intro?hl=ko#task-types]도 참고하세요.

V

차량 경로 문제 (VRP)
각 위치를 차량이 방문하도록 지정된 차량 및 위치 집합에 대한 최적의 경로 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#route]를 찾는 문제입니다. 차량 경로 문제 [https://en.wikipedia.org/wiki/Vehicle_routing_problem]를 참고하세요.
차량 경유지 웨이포인트
[https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#destination] [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#origin]출발지와 목적지 사이의 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#origin]경로 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#route]를 따라 계획된 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#planned]
정류장 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop]을 지정하는 경유지 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#waypoint] 유형입니다.
Routes API [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#routes]의 waypoint [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#waypoint] 객체에 있는 waypoint [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#waypoint] 속성도 있습니다.
중지 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#stop]도 참고하세요.
경유지를 통해
정차 지점이 아닌 마일스톤으로 표시된 경로 [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#waypoint] 유형입니다. Routes API [https://developers.google.com/maps/documentation/mobility/services/resources/glossary?hl=ko#routes]에서 가져옵니다.
