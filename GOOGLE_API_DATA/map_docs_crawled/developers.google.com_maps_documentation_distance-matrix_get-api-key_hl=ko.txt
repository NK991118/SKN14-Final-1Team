Distance Matrix API와 함께 API 키 사용

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

Google Maps Platform 제품은 API 호출을 적절한 사용자 인증 정보를 제공하는 API 호출로 제한하여 무단 사용으로부터 보호합니다. 이러한 사용자 인증 정보는 Google 결제 계정을 프로젝트, 특정 API나 SDK와 연결하는 고유한 영숫자 문자열인 API 키 형태입니다.
이 가이드에서는 Google Maps Platform에서 API 키를 생성, 제한 및 사용하는 방법을 설명합니다.

시작하기 전에
Distance Matrix API를 사용하려면 결제 계정이 있고 Distance Matrix API가 사용 설정된 프로젝트가 필요합니다. 자세한 내용은 Cloud 콘솔에서 설정하기 [https://developers.google.com/maps/documentation/distance-matrix/cloud-setup?hl=ko]를 참고하세요.

API 키 만들기

API 키는 사용 및 결제에 관한 프로젝트와 관련된 요청을 인증하는 고유 식별자입니다. 프로젝트에 연결된 API 키가 하나 이상 있어야 합니다.
API 키를 만들려면 다음 단계를 따르세요.

[TAB_GROUP_PLACEHOLDER_0]

API 키 제한

API 키를 애플리케이션에 필요한 API에서만 사용하도록 제한하는 것이 좋습니다.  API 키를 제한하면 불필요한 요청으로부터 애플리케이션이 보호되어 애플리케이션의 보안이 강화됩니다. 제한되지 않은 API 키를 악용하여 발생한 비용은 개발자가 부담해야 합니다.
자세한 내용은 API 보안 권장사항 [https://developers.google.com/maps/api-security-best-practices?hl=ko#restrict_apikey]을 참고하세요.

Cloud 콘솔에서 API 키를 제한하는 경우 애플리케이션 제한이 API 제한 아래에서 사용 설정된 모든 API를 재정의합니다.
앱 및 앱을 사용할 수 있는 플랫폼마다 API 키를 따로 만들어 권장사항을 따르세요.

API 키를 제한하는 방법은 다음과 같습니다.

[TAB_GROUP_PLACEHOLDER_1]

요청에 API 키 추가
모든 Distance Matrix API 요청에 API 키를 포함해야 합니다. 다음 예에서 YOUR_API_KEY를 API 키로 바꾸세요.

https://maps.googleapis.com/maps/api/distancematrix/json?origins=Seattle&destinations=San+Francisco&key=YOUR_API_KEY

HTTPS는 API 키를 사용하는 요청에 필요합니다.

참고: 웹 서비스를 사용할 때는 매개변수 내의 모든 특수문자를 URL로 인코딩해야 합니다.
URL은 유효하려면 올바르게 인코딩되어야 하며 모든 웹 서비스의 경우 2,048자(영문 기준)로 제한됩니다.
URL을 구성할 때 이 제한사항에 유의하세요. 브라우저, 프록시, 서버에 따라 URL 문자 수 제한이 다를 수도 있습니다.
