장소 사진으로 마이그레이션 (신규)

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

유럽 경제 지역 (EEA) 개발자

청구서 수신 주소가 유럽 경제 지역에 있는 경우 2025년 7월 8일부터
Google Maps Platform EEA 서비스 약관 [https://cloud.google.com/terms/maps-platform/eea?hl=ko]
이 서비스 이용에 적용됩니다. 기능은 리전에 따라 다릅니다.
자세히 알아보기 [https://developers.google.com/maps/comms/eea/faq?hl=ko]

이 기능을 사용하려면 차세대 Places API인 Places API (New)를 사용 설정해야 합니다. 자세한 내용은 API 사용 설정 [https://developers.google.com/maps/documentation/places/android-sdk/legacy/cloud-setup?hl=ko#enabling-apis]을 참고하세요.

Android용 Places SDK는 장소 사진 (레거시) [https://developers.google.com/maps/documentation/places/android-sdk/photos?hl=ko]을 지원합니다.
Place Photo (기존)를 잘 알고 있다면 Place Photo (신규) [https://developers.google.com/maps/documentation/places/android-sdk/place-photos?hl=ko]에서 다음과 같이 변경됩니다.

새로운 가격 책정 모델을 사용합니다. 모든 API의 가격 정보는 Android용 Places SDK(신규) [https://developers.google.com/maps/documentation/places/android-sdk/usage-and-billing?hl=ko#pricing-new]를 참고하세요.
장소 사진 (레거시)은 비트맵 이미지를 반환합니다.
장소 사진 (신규)은 이미지의 URI를 반환합니다.
장소 사진 (기존)은 최대 사진 크기가 1600x1600픽셀이었습니다. 장소 사진 (신규)은 최대 4800x4800픽셀 크기를 지원합니다.
요청하려면 새 PlacesClient.fetchResolvedPhotoUri() [https://developers.google.com/maps/documentation/places/android-sdk/reference/com/google/android/libraries/places/api/net/PlacesClient?hl=ko#fetchResolvedPhotoUri(com.google.android.libraries.places.api.net.FetchResolvedPhotoUriRequest)] 메서드를 호출합니다.
Places API를 사용 설정한 경우에만 PlacesClient.fetchPhoto() [https://developers.google.com/maps/documentation/places/android-sdk/reference/com/google/android/libraries/places/api/net/PlacesClient?hl=ko#fetchPhoto(com.google.android.libraries.places.api.net.FetchPhotoRequest)] 메서드를 호출할 수 있습니다. Places API (New)만 사용 설정한 경우에는 이 메서드를 사용할 수 없습니다.

자세한 내용은 API 키 사용 [https://developers.google.com/maps/documentation/places/android-sdk/get-api-key?hl=ko]을 참고하세요.
새 FetchResolvedPhotoUriRequest [https://developers.google.com/maps/documentation/places/android-sdk/reference/com/google/android/libraries/places/api/net/FetchResolvedPhotoUriRequest.Builder?hl=ko] 클래스의 객체를 요청에 전달합니다.
대답의 각 사진은 PhotoMetadata [https://developers.google.com/maps/documentation/places/android-sdk/reference/com/google/android/libraries/places/api/model/PhotoMetadata?hl=ko] 객체로 표현됩니다. Android용 Places SDK (New)의 경우 PhotoMetadata 객체에는 AuthorAttributions [https://developers.google.com/maps/documentation/places/android-sdk/reference/com/google/android/libraries/places/api/model/AuthorAttributions?hl=ko] 유형의 객체를 반환하는 새로운 PhotoMetadata.getAuthorAttributions() [https://developers.google.com/maps/documentation/places/android-sdk/reference/com/google/android/libraries/places/api/model/PhotoMetadata?hl=ko#public-abstract-authorattributions-getauthorattributions] 메서드가 포함되어 있습니다.

반환된 PhotoMetadata 객체에 저작자 표시 또는 작성자 저작자 표시가 포함된 경우 이미지를 표시할 때마다 애플리케이션에 이러한 저작자 표시를 포함해야 합니다. 저작자 표시 [https://developers.google.com/maps/documentation/places/android-sdk/policies?hl=ko#other_attribution_requirements]에 관한 문서를 참고하세요.
PhotoMetadata에서 새로운 저작자 표시를 액세스하려면 Places.initializeWithNewPlacesApiEnabled() [https://developers.google.com/maps/documentation/places/android-sdk/reference/com/google/android/libraries/places/api/Places?hl=ko#initializeWithNewPlacesApiEnabled(android.content.Context,%20java.lang.String)] 메서드를 호출하여 앱을 초기화해야 합니다. Places API 서비스 선택에 대한 자세한 내용은 Google Cloud 프로젝트 설정 [https://developers.google.com/maps/documentation/places/android-sdk/legacy/cloud-setup?hl=ko]을 참고하세요.
