AggregatedMetrics

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

JSON 표현 [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/AggregatedMetrics?hl=ko#SCHEMA_REPRESENTATION]

ShipmentRoute [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentRoute?hl=ko] (모든 Transition [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentRoute?hl=ko#Transition] 요소에 대한 OptimizeToursResponse [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/OptimizeToursResponse?hl=ko] 또는 모든 Visit [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentRoute?hl=ko#Visit] (모든 ShipmentRoute [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentRoute?hl=ko] 요소에 대한 OptimizeToursResponse [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/OptimizeToursResponse?hl=ko])) 요소에 대한 집계된 측정항목입니다.

| JSON 표현 |
| --- |
| {   "performedShipmentCount": integer,   "travelDuration": string,   "waitDuration": string,   "delayDuration": string,   "breakDuration": string,   "visitDuration": string,   "totalDuration": string,   "travelDistanceMeters": number,   "maxLoads": {     string: {       object (VehicleLoad)     },     ...   },   "performedMandatoryShipmentCount": integer,   "performedShipmentPenaltyCostSum": number } |

| 필드 |
| --- |
| performedShipmentCount | integer 처리된 배송 건수입니다. 수령 및 배송 쌍은 한 번만 집계됩니다. |
| travelDuration | string (Duration format) 경로 또는 솔루션의 총 이동 시간입니다. 소수점 아래가 최대 9자리까지이고 's'로 끝나는 초 단위 기간입니다. 예를 들면 "3.5s"입니다. |
| waitDuration | string (Duration format) 경로 또는 솔루션의 총 대기 시간입니다. 소수점 아래가 최대 9자리까지이고 's'로 끝나는 초 단위 기간입니다. 예를 들면 "3.5s"입니다. |
| delayDuration | string (Duration format) 경로 또는 솔루션의 총 지연 시간입니다. 소수점 아래가 최대 9자리까지이고 's'로 끝나는 초 단위 기간입니다. 예를 들면 "3.5s"입니다. |
| breakDuration | string (Duration format) 경로 또는 솔루션의 총 휴식 시간입니다. 소수점 아래가 최대 9자리까지이고 's'로 끝나는 초 단위 기간입니다. 예를 들면 "3.5s"입니다. |
| visitDuration | string (Duration format) 경로 또는 솔루션의 총 방문 시간입니다. 소수점 아래가 최대 9자리까지이고 's'로 끝나는 초 단위 기간입니다. 예를 들면 "3.5s"입니다. |
| totalDuration | string (Duration format) 총 시간은 위의 모든 시간의 합계와 같아야 합니다. 경로의 경우 다음과도 일치합니다. [ShipmentRoute.vehicle_end_time][google.maps.routeoptimization.v1.ShipmentRoute.vehicle_end_time] - [ShipmentRoute.vehicle_start_time][google.maps.routeoptimization.v1.ShipmentRoute.vehicle_start_time] 소수점 아래가 최대 9자리까지이고 's'로 끝나는 초 단위 기간입니다. 예를 들면 "3.5s"입니다. |
| travelDistanceMeters | number 경로 또는 솔루션의 총 이동 거리입니다. |
| maxLoads | map (key: string, value: object (VehicleLoad)) 이 경로 (해결 방법)의 각 수량에 대해 전체 경로 (해결 방법)에서 달성된 최대 부하로, 모든 Transition.vehicle_loads (해결 방법)의 최대값으로 계산됩니다. ShipmentRoute.metrics.max_loads. |
| performedMandatoryShipmentCount | integer 수행된 필수 배송 건수입니다. 실험용: 이 필드의 동작 또는 존재 여부는 향후 변경될 수 있습니다. |
| performedShipmentPenaltyCostSum | number 완료된 배송의 Shipment.penalty_cost의 합계입니다. 실험용: 이 필드의 동작 또는 존재 여부는 향후 변경될 수 있습니다. |
