IAM 설정 및 서비스 계정 역할

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

IAM을 올바르게 구성하는 것은 Fleet Engine 시스템의 보안 및 ID 관리의 필수 부분입니다. IAM 역할을 사용하여 운전자, 소비자, 차량 관리자의 요구사항을 충족하도록 다양한 작업 및 데이터에 대한 액세스를 맞춤설정합니다.

서비스 계정 및 IAM 역할이란 무엇인가요?

Google Cloud 콘솔에서 서비스 계정을 설정하여 Fleet Engine의 데이터에 대한 액세스를 인증하고 승인합니다. Fleet Engine에는 서비스 계정에 할당하여 해당 계정이 액세스할 수 있는 데이터를 결정하는 사전 정의된 IAM 역할이 있습니다. 자세한 내용은 Google Cloud 문서의 서비스 계정 개요 [https://cloud.google.com/iam/docs/service-account-overview?hl=ko]를 참고하세요.

Fleet Engine은 IAM 역할과 정책을 사용하여 Fleet Engine API 메서드와 리소스에 대한 승인을 관리합니다. 자세한 내용은 Google Cloud 문서의 역할 개요 [https://cloud.google.com/iam/docs/roles-overview?hl=ko]를 참고하세요. 다음 섹션에 설명된 Fleet Engine 서비스 계정 역할만 사용하세요.

IAM 역할 부여에 대한 일반적인 내용은 Google Cloud 콘솔을 사용하여 IAM 역할 부여 [https://cloud.google.com/iam/docs/grant-role-console?hl=ko]를 참고하세요.

Fleet Engine 서비스 계정 역할

Fleet Engine 설치에 선택한 모빌리티 서비스에 따라 포함되는 역할과 권한이 결정됩니다.
참고: Driver 및 Consumer SDK는 Fleet Engine에 정의된 특정 권한이 있는 표준 역할을 기반으로 빌드됩니다. 여기에 표시된 표준 역할 세트를 사용하고 맞춤 역할을 개발하지 마세요.
다음 역할은 Fleet Engine 역할과 함께 권한이 작동하는 방식을 보여줍니다.

ondemandAdmin 및 deliveryAdmin 역할은 Fleet Engine에서 모든 작업을 수행할 수 있습니다. 백엔드 서버와 Fleet Engine 간의 통신과 같은 신뢰할 수 있는 환경에서만 이러한 역할을 사용하세요.
driverSdkUser 및 consumerSdkUser 역할은 할당된 여정의 세부정보를 가져오고 차량 위치를 업데이트하거나 수신하는 것만 허용됩니다. 이러한 유형의 역할은 일반적으로 신뢰도가 낮은 환경의 클라이언트(예: 드라이버, 소비자 또는 모니터링 앱)에서 사용됩니다.

Fleet Engine에서 IAM 역할 및 서비스 계정을 사용하는 방법

Fleet Engine에서 인증 및 승인을 위해 서비스 계정을 사용하려면 다음 일반 단계를 따르세요.

필요한 각 역할에 대해 Google Cloud 콘솔에서 서비스 계정을 만듭니다. 운전자, 소비자, 차량 모니터링, 차량 관리 애플리케이션 및 웹사이트(Fleet Engine 데이터에 액세스해야 하는 모든 소프트웨어)를 인증하려면 서비스 계정이 필요합니다. 동일한 권한이 필요한 소프트웨어는 동일한 서비스 계정을 사용할 수 있습니다.
각 서비스 계정에 Fleet Engine IAM 정책 역할 할당 Fleet Engine에서 데이터에 액세스하거나 데이터를 업데이트할 수 있는 적절한 권한을 제공하는 Fleet Engine 전용 IAM 정책 역할을 선택합니다.
앱과 소프트웨어에서 적절한 서비스 계정을 사용하여 Fleet Engine에 대한 연결을 인증하고 할당된 역할에서 부여된 리소스에 대한 액세스를 승인합니다.

서비스 계정 역할이 Fleet Engine 보안에 어떻게 적용되는지 자세히 알아보려면 보안 개요 [https://developers.google.com/maps/documentation/mobility/fleet-engine/essentials/set-up-fleet/security-design?hl=ko]를 참고하세요. 서비스 계정 역할에 대한 전체 설명은 Google Cloud 문서의 IAM 역할 이해 [https://cloud.google.com/iam/docs/understanding-roles?hl=ko]를 참고하세요.
