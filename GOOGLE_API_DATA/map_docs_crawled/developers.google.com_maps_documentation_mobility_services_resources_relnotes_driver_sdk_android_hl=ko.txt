공지사항: API 수준 23, 24, 25 지원 중단

모바일 OS 버전 지원 정책 [https://developers.google.com/maps/software-support?hl=ko#mobile-os-version-support-policy]에 따라 향후 출시될 Android용 Driver SDK의 주요 버전에서 Android 6 및 7 (API 수준 23, 24, 25)에 대한 지원이 중단됩니다.

2025년 4분기부터 출시되는 Android용 Driver SDK 버전은 최소 Android 8 (API 수준 26)을 지원합니다. 이전 SDK 버전은 Android 6과 7을 계속 지원합니다.

종속 항목에서 버전 번호를 지정하지 않으면 IDE에서 최신 SDK 버전을 로드하고 앱의 새 빌드에서 Android 6과 7을 지원하지 않습니다.

새 앱 버전의 최소 지원 OS를 올리는 시점을 제어하려면 애플리케이션의 빌드 종속 항목에 Android용 Driver SDK 버전을 지정하세요.

v4.3.0 (2022년 8월 24일)

내부 개선사항

공지: Android 5 지원 중단 (2022년 7월 21일)

최신 SDK 버전 (v4.2.0)의 경우 API 수준 21과 22 모두에서 Android 5로 실행되는 앱에 대한 지원이 1년 추가로 제공됩니다.

이 기능의 작동 방식은 다음과 같습니다.

Android 앱에서 실행되는 Navigation SDK 및 Driver SDK는 2023년 6월 30일까지 최소 Android 5 (API 수준 21)를 지원합니다.
2023년 6월 30일 이후에는 Android API 수준 23 이상만 지원됩니다.
즉, 이 날짜 이후에는 모든 SDK 버전에서 Android API 수준 21과 22가 지원되지 않습니다. 즉, SDK 버전 (4.x 포함)에서 Android 21 또는 22와 관련된 버그가 수정되지 않으며 SDK가 올바르게 작동한다고 보장하지 않습니다.

이 공지는 2021년 6월 21일의 Android 21 및 22 지원 중단 공지와 2021년 10월 18일의 Android 23, 24, 25 지원 중단 공지를 대체합니다.

VehicleStops API 변경사항

VehicleStop의 getTasks 함수를 getTaskInfoList [https://developers.google.com/maps/documentation/transportation-logistics/last-mile-fleet-solution/reference/driver_4/android/reference/com/google/android/libraries/mapsplatform/transportation/driver/api/base/data/VehicleStop?hl=ko#public-abstract-immutablelisttaskinfo-gettaskinfolist] 함수로 대체했습니다.
VehicleStop.Builder의 setTasks 함수를 setTaskInfoList [https://developers.google.com/maps/documentation/transportation-logistics/last-mile-fleet-solution/reference/driver_4/android/reference/com/google/android/libraries/mapsplatform/transportation/driver/api/base/data/VehicleStop.Builder?hl=ko#public-abstract-vehiclestop.builder-settaskinfolist-listtaskinfo-taskinfolist] 함수로 대체했습니다.

비정상 종료 모니터링

SDK 안정성을 개선하는 데 도움이 되는 비정상 종료 모니터링 및 보고가 추가되었습니다.
이 기능은 기본적으로 사용 설정되어 있지만 SDK를 초기화하기 전에 setAbnormalTerminationReportingEnabled()를 호출하고 false 값을 전달하면 사용 중지할 수 있습니다.

DriverApi에 setAbnormalTerminationReportingEnabled [https://developers.google.com/maps/documentation/transportation-logistics/last-mile-fleet-solution/reference/driver_4/android/reference/com/google/android/libraries/mapsplatform/transportation/driver/api/base/DriverApi?hl=ko#public-static-void-setabnormalterminationreportingenabled-boolean-enabled] 함수를 추가했습니다.

비동기 getRemainingVehicleStops

DeliveryVehicleReporter [https://developers.google.com/maps/documentation/transportation-logistics/last-mile-fleet-solution/reference/driver_4/android/reference/com/google/android/libraries/mapsplatform/transportation/driver/api/delivery/vehiclereporter/DeliveryVehicleReporter?hl=ko]의 getRemainingVehicleStops [https://developers.google.com/maps/documentation/transportation-logistics/last-mile-fleet-solution/reference/driver_4/android/reference/com/google/android/libraries/mapsplatform/transportation/driver/api/delivery/vehiclereporter/DeliveryVehicleReporter?hl=ko#public-futurelistvehiclestop-getremainingvehiclestops] 함수가 List<VehicleStop> 대신 Future<List<VehicleStop>>을 반환하도록 변경했습니다.

지원 중단 공지 (2021년 6월 21일)

이 공지사항에서는 위에 나열된 날짜를 기준으로 Android용 Driver SDK와 Android용 Navigation SDK의 지원 중단에 대해 설명합니다. 이 공지는 영향을 받는 고객에게 필수 서비스 공지 (MSA)로도 전송되었습니다.

Android용 Nav/Driver SDK가 Android용 Navigation SDK v3.x 및 Android용 Driver SDK v3.x (바이너리 분할)로 변경됩니다.

고객 통합을 간소화하고 유용한 기능을 제공하는 기능을 가속화하기 위해 Android용 Navigation SDK와 Android용 Driver SDK가 두 개의 서로 다른 바이너리로 분할됩니다.

Android용 Navigation SDK: 운전자에게 내비게이션/경로 기능을 제공합니다.
Android용 Driver SDK를 사용하면 이동 또는 작업 진행 상황 및 할당에 Fleet Engine 상태 저장 웹 서비스를 활용할 수 있습니다.

Android용 Driver SDK에는 핵심 라우팅 기능을 위해 Android용 Navigation SDK에 대한 필수 종속 항목이 있습니다.

변경사항은 무엇인가요?

v3의 주요 변경사항은 다음과 같습니다.

바이너리가 하나가 아닌 두 개입니다.
Android용 Driver SDK 클래스의 패키지 정의가 더 명확해졌습니다.
Android용 Navigation SDK에는 패키징의 일부로 Android용 Maps SDK가 포함되어 있습니다 (Android용 Navigation SDK v2 기준).

이를 통해 지도 및 탐색 기능이 동일한 지도에 공존할 수 있으며 (예: NavigationView에 폴리라인 그리기) 두 개의 지도 대신 하나의 지도가 사용되므로 메모리 사용량이 줄어듭니다.
Android용 Navigation SDK 3.0에는 Android용 Navigation SDK와 완전히 동일한 기능이 없지만 v3.2에서 대부분의 문제가 해결됩니다. 자세한 내용은 무엇을 언제 할 수 있나요? [https://developers.google.com/maps/documentation/mobility/services/resources/relnotes_driver_sdk_android?hl=ko#whatcanido]를 참고하세요.

Navigator#startGuidance(resumeIntent)

완전성과 명확성을 위해 v1에서 v2로 전환할 때 여러 메서드가 삭제되었습니다.  이러한 변경사항 대부분은 Android용 Navigation SDK와 Android용 Maps SDK 간의 중복 클래스를 통합한 것입니다. 이러한 변경사항을 자세히 알아보려면 이전 가이드 [https://developers.google.com/maps/documentation/navigation-sdk-android/v2/migration?hl=ko]를 참고하세요.

Android용 Driver SDK 및 Android용 Navigation SDK v1.x 지원 중단

Nav/Driver SDK v1은 2018년에 출시되었으며 v3 버전이 곧 출시될 예정이므로 여러 주요 버전을 지원하는 지속 불가능성을 방지하기 위해 v1을 지원 중단할 때가 되었습니다.

이 추가 버전을 지원하지 않아 절약되는 시간을 통해 최신 주요 버전을 더 잘 지원하고 고객에게 중요한 기능을 더 많이 빌드할 수 있습니다.

따라서 Nav/Driver SDK v1.x는 현재 지원 중단되었으며 2022년 6월 21일 이후에는 더 이상 유지관리되지 않습니다.

잠시 시간을 내어 SDK 버전의 지원 중단 작동 방식 [https://developers.google.com/maps/deprecations?hl=ko#deprecation_in_sdk_versions]을 확인하세요.

Navigation 및 Driver SDK Android 21 및 22 지원 종료
주의: 이 공지는 취소되었습니다. 2022년 7월 21일 [https://developers.google.com/maps/documentation/mobility/services/resources/relnotes_driver_sdk_android?hl=ko#announcement_july_21_2022]에 업데이트된 공지를 참고하세요.

취소된 공지 읽기
예정된 Android용 Navigation SDK v4.0 및 Android용 Driver SDK는 최소 [Android 6.0 (API 수준 23)](https://developer.android.com/studio/releases/platforms#6.0){:
.external}을 지원합니다.  현재 지원되는 최소 버전은 Android 5.0 (API 수준 21)입니다.  2021년 4분기에 v4.0이 출시될 것으로 예상됩니다. 이는 다음을 의미합니다.

v4.x 이전 버전으로 빌드된 앱 버전은 Android 21 및 22 기기에서 계속 작동합니다. 현재 SDK를 통합한 드라이버 앱은 Android 21 이상을 실행하는 기기에서 계속 작동합니다.  이는 이전과 동일합니다.
Android용 Driver SDK 및 Android용 Navigation SDK v4.x (2021년 4분기 출시 예정, 변경될 수 있음) 이상으로 빌드된 앱 버전은 Android 21 및 22를 실행하는 기기에 설치할 수 없습니다. v4.x는 최소 Android 6.0 (API 수준 23)에서 실행되므로 향후 출시될 v4.x로 빌드된 앱 버전은 Android 23 이상이 설치된 기기를 사용하는 운전자만 수신할 수 있습니다. 다시 말해 SDK 관련 변경사항과 개발자가 적용한 앱 변경사항이 모두 Android 21 및 22를 실행하는 기기를 사용하는 운전자에게는 적용되지 않습니다.

업그레이드할 Android용 Navigation SDK v3 및 Android용 Driver SDK v3 버전은 다음 표에 출시 예정일과 함께 설명되어 있습니다.
