2025년 7월 8일부터 프로젝트가 EEA 결제 계정에 연결된 경우 새로운 EEA용 Google Maps Platform 서비스 약관(EEA 약관)이 사용자를 위한 애플리케이션 및 환경을 빌드하는 방식에 영향을 미칩니다.

세부정보는 Google Maps Platform을 통합하는 기존 및 신규 애플리케이션에 영향을 미칠 수 있으므로 공지사항 [https://developers.google.com/maps/comms/eea/faq?hl=ko]을 읽고 이해하시기 바랍니다.

이 문서에서는 Places API 및 관련 서비스의 사용 및 Google Maps Platform 기능 변경사항을 설명합니다.

이 문서에 설명된 정보는 2025년 7월 8일 현재 Google Maps Platform 서비스 사용에 적용되는 EEA 약관 [https://cloud.google.com/terms/maps-platform/eea?hl=ko]을 반영하며, 이는 시간이 지남에 따라 변경될 수 있습니다. 귀하에게 적용되는 최신 구속력 있는 약관은 Google과의 Google Maps Platform 계약을 참고하세요.

영향을 받은 서비스
모든 Places API (신규 및 기존) 메서드가 영향을 받습니다. Places API (신규 및 기존)의 영향을 받는 서비스에는 장소 세부정보, 텍스트 검색, 주변 검색, 자동 완성, 장소 자동 완성, 검색어 자동 완성, 장소 찾기, 현재 장소, 장소 사진과 같은 기능이 포함됩니다.

Places API (신규 및 기존)에는 Places JavaScript API (신규 및 기존), Android용 Places SDK (신규 및 기존), iOS용 Places SDK(신규 및 기존)를 통한 Places API 서비스 사용 가능 여부도 포함됩니다.

이 변경사항은 장소 UI 키트에는 적용되지 않습니다.

EEA 고객을 위한 Places API 조정

이 조정은 (a) 2025년 7월 8일 이후에 생성되었으며 EEA 결제 주소가 있는 계정에 연결된 프로젝트 또는 (b) 2025년 7월 8일 이후에 더 이상 수정되지 않은 상태 [https://developers.google.com/maps/comms/eea/faq?hl=ko#material-modification]가 아닌 프로젝트에 적용됩니다.

다음 표에는 각 Google Maps Platform 서비스에 대해 EEA 서비스별 약관 [https://cloud.google.com/terms/maps-platform/eea/maps-service-terms?hl=ko]에 명시된 조정이 요약되어 있습니다.

| Places API 기능 조정 | 서비스 약관 변경사항 |
| --- | --- |
| 없음 | 지도와 함께 사용 금지 고객은 위도, 경도, place_id 외에는 어떤 지도와도 Places API의 Google 지도 콘텐츠를 사용해서는 안 됩니다. 허용되는 사용 고객은 위도, 경도, place_id 외에도 Places API EEA 허용 사용에 따라 Places API의 Google 지도 콘텐츠만 사용할 수 있습니다. ***명확히 하기 위해 위의 제한사항(지도와 함께 사용 불가 및 허용되는 사용)은 장소 UI 키트에 적용되지 않습니다.*** |

'지도 포함' 및 '지도 없음'의 예
EEA 약관 변경사항은 이 서비스의 Google 지도 콘텐츠를 '모든 지도와 함께' 사용하는 방식에 영향을 미칩니다. '지도와 함께'는 Google 지도를 포함한 지도 옆에 또는 지도와 시각적으로 연결된 방식으로 Google 지도 콘텐츠를 표시하거나 Google 지도 콘텐츠를 지도에 연결하거나 지도와 Google 지도 콘텐츠를 연결하는 것을 의미합니다(연결되는 특정 지도가 관련 Google 지도 콘텐츠의 소스인 경우 제외).

명확히 하기 위해 서비스에서 제공하는 Google 지도 콘텐츠 소스 링크 (예: Places API의 googleMapsLinks [https://developers.google.com/maps/documentation/places/web-service/place-details?hl=ko#required-parameters])에 계속 링크할 수 있습니다.

다음 예는 계약 약관을 준수하는 데 도움이 되도록 '지도 포함'과 '지도 없음'이 어떤 모습일 수 있는지 보여주는 그림입니다. 예시는 포괄적이지 않습니다. 궁극적으로 계약 약관을 준수할 책임은 개발자에게 있습니다.

대체 통합

다음 통합 가이드라인과 예는 Google 지도 콘텐츠를 사용하지 않는 요구사항과 관련하여 Places API를 시각적으로 구현하는 데 도움이 됩니다. Places API의 Google 지도 콘텐츠를 사용할 때는 Places API EEA 허용 사용 [https://cloud.google.com/terms/maps-platform/eea-places-api-permitted-uses?hl=ko]도 준수해야 합니다.

지도와 함께 Places API의 위도, 경도, 장소 ID를 계속 사용할 수 있습니다. 여기에는 다른 Google 지도 콘텐츠가 어떤 지도와도 사용되지 않는 한 위도, 경도 또는 장소 ID만 사용하여 지도에 연결하는 것도 포함됩니다.

|  |  |  |
| --- | --- | --- |
| 위도, 경도 또는 장소 ID를 사용하여 지도에 연결하는 예시(여전히 허용됨) | 지도에 표시되는 위도 및 경도의 예시로, 여전히 허용됨 |  |

Google 지도 콘텐츠가 어떤 지도와도 함께 사용되지 않는 허용되는 상황에서는 Places API 서비스를 계속 사용할 수 있습니다.

|  |  |
| --- | --- |
| 장소 ID를 사용하여 지도에 연결하는 예 : 지도 옆에 장소 UI 키트가 표시됨 |  |

그 밖의 대부분의 경우 장소 UI 키트는 모든 지도에서 사용할 수 있습니다.

장소 UI 키트

Google Maps Platform에는 사용자가 지도 유무와 관계없이 장소 콘텐츠를 시각화할 수 있도록 하는 통합을 빌드할 수 있는 장소 UI 키트가 포함되어 있습니다.
Places API 허용되는 사용 [https://cloud.google.com/terms/maps-platform/eea-places-api-permitted-uses?hl=ko] 제한사항은 Places UI Kit에 적용되지 않습니다.

예를 들어 장소 UI 키트를 사용하면 다음 이미지와 같이 지도에 장소 세부정보를 표시하거나 지도 옆에 검색 결과의 장소 목록을 렌더링할 수 있습니다.

참고: 장소 세부정보, 장소 자동 완성, 장소 검색을 위한 Places UI Kit 요소는 2025년 7월 중순에 정식 버전으로 제공될 예정입니다.

도로 안전

이제 EEA 고객은 적절한 경우 삽입된 차량 시스템에서 Google Maps Platform 서비스를 사용할 수 있습니다. 또한 서드 파티 지도의 실시간 내비게이션에 적합한 경우 Google Maps Platform 서비스를 사용할 수 있습니다(단, 실시간 내비게이션 사용이 Google의 안전 요구사항을 준수하는 경우에 한함).

이러한 새로운 약관에 따라 발생할 수 있는 도로 안전 위험으로 인해 이제 Google에서는 개발자가 안전 요구사항 [https://cloud.google.com/terms/maps-platform/eea-safety-requirements?hl=ko]을 준수하도록 요구합니다. 예를 들어 도로 불일치의 위험을 제한하는 것이 여기에 해당합니다.

참고: EEA의 삽입형 차량 시스템에서 Google 지도 플랫폼 서비스를 사용하는 것은 각 고객의 재량에 따라 이루어지며, 고객은 관련 법규, 규정, 업계 표준 또는 기타 요구사항을 준수할 책임이 있습니다.
