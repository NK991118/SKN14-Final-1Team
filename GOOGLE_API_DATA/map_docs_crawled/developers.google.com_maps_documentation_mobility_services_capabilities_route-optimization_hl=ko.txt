Title: 중지 시퀀스 최적화

중지 시퀀스 최적화

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

Route Optimization API [https://developers.google.com/maps/documentation/route-optimization?hl=ko]는 구성 가능한 최적화 목표와 제약 조건을 기반으로 하나 이상의 차량에 대한 최적의 경유지 순서를 계산합니다.
Route Optimization API는 운전자가 작업을 완료할 최적의 순서를 찾는 문제를 해결하는 엔터프라이즈급 기능을 제공합니다. Mobility Optimize and Accelerate 고객인 경우 사용 가능한 서비스 목록 [https://developers.google.com/maps/documentation/transportation-logistics-terms/mobility/services?hl=ko]에 경로 최적화 API, 단일 차량 라우팅 [https://developers.google.com/maps/documentation/route-optimization?hl=ko]이 포함됩니다.

모든 모빌리티 고객은 기본적인 정지 순서 최적화를 지원하는 다음 API에도 액세스할 수 있습니다.

Routes API, Compute Routes [https://developers.google.com/maps/documentation/routes?hl=ko]
Routes Preferred API, Compute Routes [https://developers.google.com/maps/documentation/routes_preferred?hl=ko]
Directions API [https://developers.google.com/maps/documentation/directions?hl=ko]

자체 경로 최적화 프로그램을 개발한 경우 Routes Preferred ComputeRouteMatrix [https://developers.google.com/maps/documentation/routes_preferred/compute_route_matrix?hl=ko] 또는 Routes API ComputeRouteMatrix [https://developers.google.com/maps/documentation/routes/compute_route_matrix?hl=ko] 메서드를 사용하여 여러 출발지와 목적지의 경로 거리와 시간을 추정할 수 있습니다. ComputeRouteMatrix를 사용하면 다른 Google Maps Platform 제품에서 사용하는 것과 동일한 트래픽 및 라우팅 알고리즘을 사용할 수 있습니다. Distance Matrix API [https://developers.google.com/maps/documentation/distance-matrix?hl=ko]에 비해 몇 가지 장점이 있습니다.
참고: Routes API는 차세대 Directions API 및 Distance Matrix API입니다. Directions API 및 Distance Matrix API의 새로운 사용은 권장되지 않습니다.
Route Optimization API의 차량 경로 지정 기능 [https://developers.google.com/maps/documentation/route-optimization?hl=ko]은 전체 차량을 대상으로 작동하여 정지 순서 최적화와 할당을 동시에 해결합니다. Fleet Routing은 모빌리티 고객이 사용할 수 있는 서비스 목록 [https://developers.google.com/maps/documentation/transportation-logistics-terms/mobility/services?hl=ko]에 포함되어 있지 않지만 소비량에 따라 요금이 청구되는 방식으로 사용할 수 있습니다. 자세한 내용은 Route Optimization API 사용량 및 결제 [https://developers.google.com/maps/documentation/route-optimization/usage-and-billing?hl=ko] 가이드를 참고하세요.
참고: 실수로 인한 Fleet Routing 사용 요금을 방지하기 위해 모빌리티에 등록된 Cloud 프로젝트는 Fleet Routing 요청을 할 수 없습니다. 모빌리티에 등록된 Cloud 프로젝트에서 Fleet Routing을 사용하려면 액세스를 요청하는 Maps 지원 케이스를 만드세요 [https://console.cloud.google.com/google/maps-apis/support/createcase?hl=ko].
Route Optimization API를 사용해야 하는 이유

Route Optimization API를 사용하면 운전자가 가장 효율적인 경로로 운전할 수 있습니다. 고객의 주문이 최대한 빠르고 저렴하게 도착할 수 있도록 최적의 순서로 정차하고 작업을 완료합니다. 경로 최적화는 다음 목표를 달성하는 데 필요한 노력을 간소화합니다.

운전기사 작업 생산성 향상: 운전기사가 정체로 인한 지연을 방지할 수 있도록 실시간 교통 업데이트를 포함한 효율적인 경로를 제공합니다.
소비자 경험 개선: 소비자가 더 정확한 예상 배송 시간을 알 수 있도록 지정된 시간 내에 정류장이 예약되도록 합니다.

다음 단계

Maps SDK로 경로 시각화

일부 차량 운영자는 운전자나 차량 관리자가 사용자 인터페이스 내에서 직접 경로를 볼 수 있기를 원합니다. Maps SDK를 사용하여 Routes API의 경로를 표시합니다. 지도에서 경로를 맞춤설정하는 방법에 관한 자세한 내용은 Maps SDK for Javascript [https://developers.google.com/maps/documentation/javascript?hl=ko], Maps SDK for Android [https://developers.google.com/maps/documentation/android-sdk?hl=ko], Maps SDK for iOS [https://developers.google.com/maps/documentation/ios-sdk?hl=ko]를 참고하세요.

Fleet Engine으로 경로 추적

경로가 최적화되면 Mobility Accelerate 고객은 Fleet Engine을 사용하여 경로 실행을 추적할 수 있습니다. 자세한 내용은 Fleet Engine [https://developers.google.com/maps/documentation/mobility/fleet-engine?hl=ko] 문서를 참고하세요.
