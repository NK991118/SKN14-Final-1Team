유럽 경제 지역 (EEA) 개발자

청구서 수신 주소가 유럽 경제 지역에 있는 경우 2025년 7월 8일부터 서비스 사용 시 Google Maps Platform EEA 서비스 약관 [https://cloud.google.com/terms/maps-platform/eea?hl=ko]이 적용됩니다. 기능은 리전에 따라 다릅니다.
자세히 알아보기 [https://developers.google.com/maps/comms/eea/faq?hl=ko]

'이륜차'는 오토바이와 같은 이륜 전동차를 의미합니다.
오토바이 이동 모드는 사람이 직접 힘을 가해 이동하는 자전거 이동 모드와 다릅니다.
주의: TWO_WHEELER 이동 모드를 사용하는 요청에는 더 높은 요율이 적용됩니다. Routes API의 결제에 대해 자세히 알아보세요 [https://developers.google.com/maps/documentation/routes/usage-and-billing?hl=ko].
요청의 travelMode 속성을 사용하여 모드를 지정합니다. 이 속성의 가능한 값은 RouteTravelMode [https://developers.google.com/maps/documentation/routes/reference/rest/v2/RouteTravelMode?hl=ko] (REST) 및 RouteTravelMode [https://developers.google.com/maps/documentation/routes/reference/rpc/google.maps.routing.v2?hl=ko#routetravelmode] (gRPC)의 열거형으로 정의됩니다.

참고: 도보, 자전거, 이륜차 경로는 베타 버전이며 명확한 인도, 보행자 전용 도로 또는 자전거 전용 도로가 누락될 수 있습니다. 앱에 표시하는 모든 도보, 자전거, 이륜차 경로에 이 경고를 사용자에게 표시해야 합니다.

오토바이 이동 모드 지정

Routes API를 호출할 때 RouteTravelMode 열거형 값을 지정하여 이동 수단을 지정합니다.

다음 예는 computeRoutes [https://developers.google.com/maps/documentation/routes/reference/rest/v2/TopLevel/computeRoutes?hl=ko] 메서드 (REST) 요청 본문에서 이동 모드를 TWO_WHEELER로 설정하는 방법을 보여줍니다. 이 옵션을 ComputeRoutes [https://developers.google.com/maps/documentation/routes/reference/rpc/google.maps.routing.v2?hl=ko#google.maps.routing.v2.Routes.ComputeRoutes] 메서드 (gRPC)에 지정할 수도 있습니다.

이 예에서는 Plus 코드로 지정된 인도의 두 위치 간의 경로를 계산합니다.

"address": "H8MW+WP Kolkata India"
