2025년 7월 8일부터 프로젝트가 EEA 결제 계정에 연결된 경우 새로운 EEA 전용 Google Maps Platform 서비스 약관(EEA 약관)이 사용자를 위한 애플리케이션 및 환경 구축 방식에 영향을 미칩니다.

세부정보가 Google Maps Platform을 통합하는 기존 및 신규 애플리케이션에 영향을 미칠 수 있으므로 공지사항 [https://developers.google.com/maps/comms/eea/faq?hl=ko]을 읽고 이해하는 것이 좋습니다.

이 문서에서는 지도 타일 API의 사용량 및 Google Maps Platform 기능 변경사항을 설명합니다.

이 문서에 설명된 정보는 2025년 7월 8일 현재 Google Maps Platform 서비스 사용에 적용되는 EEA 약관 [https://cloud.google.com/terms/maps-platform/eea?hl=ko]을 반영하며, 이는 시간이 지남에 따라 변경될 수 있습니다. 귀하에게 적용되는 최신 구속력 있는 약관은 Google과의 Google Maps Platform 계약을 참고하세요.

EEA 고객을 위한 Map Tiles API 조정

이 조정은 (a) 2025년 7월 8일 이후에 생성되고 EEA 결제 주소가 있는 계정에 연결된 프로젝트 또는 (b) 2025년 7월 8일 이후에 더 이상 수정되지 않은 상태 [https://developers.google.com/maps/comms/eea/faq?hl=ko#material-modification]가 아닌 프로젝트에 적용됩니다.

다음 표에는 각 Google Maps Platform 서비스에 대한 조정사항이 요약되어 있으며, 이는 EEA 서비스별 약관 [https://cloud.google.com/terms/maps-platform/eea/maps-service-terms?hl=ko]에 명시되어 있습니다.

| Map Tiles API 기능 조정 | 서비스 약관 변경 |
| --- | --- |
| 2D 타일의 위성 이미지 타일을 사용할 수 없습니다. 포토리얼리스틱 3D 타일을 사용할 수 없습니다. EEA 프로젝트에서 이러한 유형의 타일을 요청하면 Map Tiles API에서 403 HTTP 오류가 발생합니다. 자세한 내용은 Map Tiles API 문서의 오류 처리를 참고하세요. | 지도와 함께 사용 금지. 고객은 지도와 함께 Map Tiles API의 스트리트 뷰 타일을 사용할 수 없습니다. |

'지도가 있는' 및 '지도가 없는'의 예
EEA 약관 변경사항은 이 서비스의 Google 지도 콘텐츠를 '모든 지도와 함께' 사용하는 방식에 영향을 미칩니다. '모든 지도와 함께'는 Google 지도를 포함한 모든 지도에 또는 지도 옆에 Google 지도 콘텐츠를 표시하거나 시각적으로 연결하거나, Google 지도 콘텐츠를 모든 지도에 연결하거나 모든 지도를 Google 지도 콘텐츠에 연결하는 것을 의미합니다(연결되는 특정 지도가 해당 Google 지도 콘텐츠의 소스인 경우는 제외).

명확성을 위해 서비스에서 제공하는 Google 지도 콘텐츠 소스 링크 (예: Places API의 googleMapsLinks [https://developers.google.com/maps/documentation/places/web-service/place-details?hl=ko#required-parameters])로 계속 연결할 수 있습니다.

다음 예는 계약의 약관을 준수하는 데 도움이 되도록 '지도가 있는 경우'와 '지도가 없는 경우'의 모습을 보여줍니다. 이 예시는 포괄적이지 않습니다. 궁극적으로 '계약'의 약관을 준수할 책임은 사용자에게 있습니다.

대체 통합

다음 대안 중 하나를 사용하도록 통합을 변경하는 것이 좋습니다.

Maps JavaScript API

스트리트 뷰 서비스 [https://developers.google.com/maps/documentation/javascript/streetview?hl=ko]
3D 지도 [https://developers.google.com/maps/documentation/javascript/3d/overview?hl=ko]가 포토리얼리스틱 3D 타일을 대체할 수 있습니다.
위성 지도 유형 [https://developers.google.com/maps/documentation/javascript/maptypes?hl=ko]은 2D 타일의 위성 이미지를 대체할 수 있습니다.

Android용 Maps SDK

스트리트 뷰 [https://developers.google.com/maps/documentation/android-sdk/streetview?hl=ko]
3D 지도 [https://developers.google.com/maps/documentation/maps-3d/android-sdk?hl=ko]가 포토리얼리스틱 3D 타일을 대체할 수 있습니다.
위성 지도 [https://developers.google.com/maps/documentation/android-sdk/configure-map?hl=ko#maptype]는 2D 타일의 위성 이미지를 대체할 수 있습니다.

iOS용 Maps SDK

스트리트 뷰 [https://developers.google.com/maps/documentation/ios-sdk/streetview?hl=ko]
3D 지도 [https://developers.google.com/maps/documentation/maps-3d/ios-sdk/overview?hl=ko]가 포토리얼리스틱 3D 타일을 대체할 수 있습니다.
위성 지도 [https://developers.google.com/maps/documentation/ios-sdk/configure-map?hl=ko#map_types]는 2D 타일의 위성 이미지를 대체할 수 있습니다.

도로 안전

이제 EEA 고객은 적합한 경우 삽입된 차량 시스템에서 Google Maps Platform 서비스를 사용할 수 있습니다. 또한 실시간 내비게이션 사용이 Google의 안전 요구사항을 준수하는 경우 적합한 경우 서드 파티 지도를 사용한 실시간 내비게이션에 Google Maps Platform 서비스를 사용할 수 있습니다.

이러한 새로운 약관으로 인해 발생할 수 있는 도로 안전 위험으로 인해 Google에서는 이제 개발자가 안전 요구사항 [https://cloud.google.com/terms/maps-platform/eea-safety-requirements?hl=ko]을 준수하도록 요구합니다. 예를 들어 도로 불일치 위험을 제한하기 위해서입니다.

참고: EEA의 차량에 내장된 시스템에서 Google Maps Platform 서비스를 사용하는 것은 각 고객의 재량에 따라 이루어지며, 고객은 관련 법규, 규정, 업계 표준 또는 기타 요구사항을 준수할 책임이 있습니다.
