TypeScript [https://www.typescriptlang.org/]는 유형이 있는 JavaScript 상위 집합으로, 일반 JavaScript로 컴파일됩니다. 아래 스니펫은 TypeScript를 사용한 Google 지도의 간단한 사용 예를 보여줍니다.
zoom: 8
참고: 이러한 유형은 자동으로 생성됩니다. 이러한 유형 관련 문제를 신고하려면 지원 티켓 [https://issuetracker.google.com/issues/new?component=188853&%3Btemplate=788207&hl=ko]을 여세요.
알파 및 베타 기능

이러한 유형 [https://github.com/DefinitelyTyped/DefinitelyTyped/tree/master/types/google.maps]에는 일반적으로 알파 또는 베타 버전의 속성, 함수 또는 클래스가 없습니다. 이러한 경우 대부분 객체를 적절한 유형으로 캐스팅할 수 있습니다.

다음 오류는 MapOptions의 mapId 베타 속성으로 인해 발생합니다.

error TS2345: Argument of type '{ center: google.maps.LatLng; zoom: number;
'MapOptions'.

위의 오류는 아래 캐스팅으로 수정할 수 있습니다.

@types 패키지 충돌

일부 라이브러리에서 @types/google.maps [https://www.npmjs.com/package/@types/google.maps] 이외의 패키지를 사용할 수 있으며 이로 인해 충돌이 발생할 수 있습니다. 일관되지 않은 유형 문제를 방지하려면 skipLibCheck [https://www.typescriptlang.org/tsconfig#skipLibCheck] 컴파일러 옵션을 사용합니다.
"compilerOptions": {
"skipLibCheck": true

Angular와 같은 일부 프레임워크에서는 @types/google.maps [https://www.npmjs.com/package/@types/google.maps] 및 기타 모든 '@types' 패키지에서 설치된 유형을 포함하도록 typeRoots [https://www.typescriptlang.org/tsconfig#typeRoots] 컴파일러 옵션을 지정해야 할 수 있습니다.
참고: 표시되는 모든 '@types' 패키지는 기본적으로 컴파일에 포함됩니다.
상위 폴더의 node_modules/@types에 있는 패키지는 표시되는 것으로 간주됩니다.{
...
"compilerOptions": {
...
"typeRoots": [
"node_modules/@types",
],
...
