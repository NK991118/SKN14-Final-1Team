자주 묻는 질문(FAQ)

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

유럽 경제 지역 (EEA) 개발자

청구서 수신 주소가 유럽 경제 지역에 있는 경우 2025년 7월 8일부터
Google Maps Platform EEA 서비스 약관 [https://cloud.google.com/terms/maps-platform/eea?hl=ko]이 서비스 이용에 적용됩니다. 자세히 알아보기 [https://developers.google.com/maps/comms/eea/faq?hl=ko]
또한 Solar API의 특정 콘텐츠가 더 이상 반환되지 않습니다.
자세히 알아보기 [https://developers.google.com/maps/comms/eea/solar?hl=ko]

도움말

이전 버전에서 새 Solar API로 이전하려면 어떻게 해야 하나요?

마이그레이션 가이드 [https://developers.google.com/maps/documentation/solar/migration?hl=ko]를 검토하세요.

어디에서 도움을 받을 수 있나요?

Solar API 지원 옵션 [https://developers.google.com/maps/documentation/solar/support?hl=ko]을 확인하세요.

이 오류 코드는 무엇을 의미하나요?

HTTP 상태 코드 [https://developer.mozilla.org/en-US/docs/Web/HTTP/Status]에 대해 자세히 알아보세요.

프로젝트 구성

Cloud 프로젝트에 어떤 권한이 필요한가요?

문서 [https://developers.google.com/maps/roles?hl=ko]를 참고하세요.

하나 이상의 Cloud 프로젝트에서 API 키를 사용할 수 있나요?

아니요. 하나의 Cloud 프로젝트에만 API 키를 사용하세요. Cloud 프로젝트가 여러 개인 경우 각 프로젝트에 자체 API 키가 있어야 합니다. 단일 프로젝트 내에는 여러 API 키가 있을 수 있습니다.

비용

비용을 제한하려면 어떻게 해야 하나요?

전략과 팁은 예산 문서 [https://cloud.google.com/billing/docs/how-to/budgets?hl=ko]를 참고하세요.

태양 모델

데이터는 얼마나 자주 새로고침되나요?

새로운 데이터는 항상 추가되지만 특정 지역에 대한 구체적인 새로고침 빈도는 없습니다. 일부 지역에서는 수년 된 이미지를 사용합니다.
관련 이미지의 날짜를 확인하고 의견을 제출 [https://issuetracker.google.com/issues/new?component=1356349&%3Btemplate=1817492&hl=ko]해 주세요.

태양광 재무 분석의 가정에 대해 자세히 알아보려면 어디를 참고해야 하나요?

Google은 Clean Power Research [https://www.cleanpower.com/]의 정부 태양광 인센티브에 관한 정보와 기타 금융 데이터 및 모델을 사용합니다.

태양광 패널 배치를 평가할 때 API에서 굴뚝과 채광창을 고려하나요?

예, 높이 지도를 사용하면 굴뚝과 채광창이 작은 직사각형으로 식별되고 태양광 패널은 모델에서 제외됩니다.

API가 건물 내부를 볼 수 있나요?

아니요, 지붕만 있습니다.

API가 건물 단지 내 특정 아파트에 대한 평가를 할 수 있나요?

경우에 따라 다릅니다. 이 모델은 Google 지도 데이터를 사용하므로 아파트가 Google 지도에서 전체 건물로 분류되면 해당 건물의 특정 아파트 지붕이 별도로 표시됩니다.

API가 내가 알고 있는 건물의 데이터를 반환하지 않는 경우가 있는 이유는 무엇인가요?

이러한 문제는 당시 매핑이 이루어질 때 존재하지 않았던 새 건물이 있는 경우에 발생할 수 있습니다.

플럭스 태양광 데이터 레이어를 사용하여 그늘을 시각화하려면 어떻게 해야 하나요?

북쪽을 향하는 지붕은 플럭스 (일사량 [https://en.wikipedia.org/wiki/Solar_irradiance])가 적으므로 그늘이 더 많이 생깁니다. 태양광 평가를 수행할 때 이 정보를 활용할 수 있습니다. 플럭스 데이터 레이어를 사용하면 플럭스를 색상으로 시각화할 수 있습니다.
지붕에서 특정 숫자 값의 음영을 찾으려는 경우 자체 플럭스 타일을 만들어 색상을 맞춤설정할 수도 있습니다.

dataLayers service [https://developers.google.com/maps/documentation/solar/reference/rest/v1/dataLayers?hl=ko]에서 hourlyShadeUrls를 어떻게 사용할 수 있나요?

연중 그늘의 일일 스냅샷을 다시 가져와 연중 그늘이 변하는 애니메이션 GIF 또는 동영상을 만들 수 있습니다.

맞춤 계산을 직접 실행할 수 있나요?

예, 플럭스 및 DSM (각 지점의 높이 맵) 태양광 데이터 레이어를 사용하여 더 세부적으로 맞춤설정할 수 있습니다.

API가 URL을 반환하는 이유는 무엇이며, URL을 사용하여 무엇을 할 수 있나요?

URL은 해당 데이터 레이어를 다운로드할 수 있도록 하는 단기 포인터입니다.
