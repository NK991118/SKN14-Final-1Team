Title: 경로 계획하기

경로 계획하기

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

이 제품 또는 기능은 미리보기(GA 이전) 버전입니다.
GA 이전 제품과 기능은 지원이 제한적일 수 있으며, GA 이전 제품과 기능에 대한 변경사항은 다른 GA 이전 버전과 호환되지 않을 수 있습니다. GA 이전 버전 제품 및 서비스에는 Google Maps Platform 서비스별 약관 [https://cloud.google.com/maps-platform/terms/maps-service-terms?hl=ko]이 적용됩니다. 자세한 내용은 출시 단계 설명 [https://developers.google.com/maps/launch-stages?hl=ko]을 참고하세요.

차량 공유 여행의 최적 경로를 결정할 때 가장 빠른 경로가 항상 최선의 선택은 아닙니다. 경로를 계획하는 것이 좋습니다.

Routes Preferred API를 사용하면 ComputeCustomRoutes 메서드를 사용하여 경로 목표를 지정하여 경로를 계획할 수 있습니다.

ComputeCustomRoutes 메서드는 운전자가 선택한 경로와 운전한 경로 간의 차이를 최소화하여 경로 규정 준수에도 도움이 될 수 있습니다.

Routes Preferred API와 Navigation SDK [https://developers.google.com/maps/documentation/navigation-sdk?hl=ko]를 모두 사용하여 경로 계획의 이점을 최대한 활용하세요.
참고: Routes Preferred API와 Navigation SDK에 동일한 클라우드 프로젝트를 사용하지 않아도 됩니다.참고: ComputeRoutes API와 달리 ComputeCustomRoutes는 DRIVE (또는 이에 상응하는 TRAVEL_MODE_UNSPECIFIED) 및 TWO_WHEELER의 두 가지 이동 모드만 지원합니다. 이 메서드는 via 중간 경유지도 지원하지 않습니다.
다른 이동 모드나 via 경유지를 사용하려고 하면 요청이 실패합니다.
via 경유지에 대한 자세한 내용은 Routes Preferred API 경유지 [https://developers.google.com/maps/documentation/routes_preferred/reference/rest/Shared.Types/Waypoint?hl=ko]를 참고하세요.
ComputeCustomRoutes 메서드 사용 설정

코드에서 ComputeCustomRoutes 메서드를 사용하려면 먼저 사용 설정해야 합니다. Routes Preferred API 메서드 사용 설정에 관한 자세한 내용은 시작하기 [https://developers.google.com/maps/documentation/routes_preferred/get_started?hl=ko]를 참고하세요.
중요: Routes Preferred API 메서드의 할당량 한도를 확인하세요. 할당량 한도를 늘려야 하는 경우 Google Cloud Console의 Google Maps Platform 지원 페이지 [https://console.cloud.google.com/google/maps-apis/support?hl=ko]에서 케이스를 제출하거나 Google Cloud 지원 핫라인 [https://developers.google.com/maps/documentation/routes_preferred/support?hl=ko#support_hotline]에 전화하여 지원팀에 문의하세요.
ComputeCustomRoutes 메서드 사용

ComputeCustomRoutes 메서드를 사용하려면 입력에 응답 필드 마스크를 지정해야 합니다. URL 매개변수 $fields 또는 fields를 사용하거나 HTTP/gRPC 헤더 X-Goog-FieldMask를 사용하여 응답 필드 마스크를 제공할 수 있습니다. 자세한 내용은 시스템 매개변수 [https://cloud.google.com/apis/docs/system-parameters?hl=ko]를 참고하세요.

필드 마스크 값은 쉼표로 구분된 필드 경로 목록입니다. 각 필드 경로는 메시지 계층 구조를 나타내는 필드 이름의 점으로 구분된 목록입니다.
필드 이름은 JSON 객체 키 또는 protobuf 필드 태그 이름입니다.
필드 경로는 최상위 응답 메시지 유형에서 시작하며, 선택적으로 하나 이상의 점과 다음 수준의 필드 이름이 뒤따릅니다. 일반적으로 필드 경로는 다음과 같은 방식으로 구성됩니다.

topLevelField[.secondLevelField][.thirdLevelField][...]

특수한 경우의 필드 마스크는 모든 루트 수준 필드를 선택하는 와일드카드 '*'입니다.

자세한 내용은 다음을 참고하세요.

gRPC API 참조 [https://developers.google.com/maps/documentation/routes_preferred/reference/rpc/google.maps.routes.v1?hl=ko]
REST API 참조 [https://developers.google.com/maps/documentation/routes_preferred/reference/rest/v1/TopLevel/computeCustomRoutes?hl=ko]

ComputeCustomRoutes 메서드 사용 예시는 맞춤 경로 계산 (베타) 예시 [https://developers.google.com/maps/documentation/routes_preferred/custom_route_examples?hl=ko]를 참고하세요.
