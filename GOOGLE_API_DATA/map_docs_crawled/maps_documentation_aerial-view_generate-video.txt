새 동영상 생성
renderVideo [https://developers.google.com/maps/documentation/aerial-view/reference/rest/v1/videos/renderVideo?hl=ko] 메서드는 지정된 미국 우편 주소의 항공 뷰 동영상 생성을 요청합니다. 지정된 주소에 동영상이 이미 있으면 이 메서드는 동영상에 관한 메타데이터를 반환합니다.
참고: 이 renderVideo 사용은 Google에서 아직 필요한 항공 뷰 동영상을 생성하지 않았다고 가정합니다. 즉, 새로운 항공 뷰 동영상을 생성해야 합니다.
API 사용 방법
renderVideo [https://developers.google.com/maps/documentation/aerial-view/reference/rest/v1/videos/renderVideo?hl=ko] 엔드포인트에 HTTPS POST 요청을 보내 새 항공 동영상을 생성합니다. 다음 예와 같이 API 키 [https://developers.google.com/maps/documentation/aerial-view/get-api-key?hl=ko]와 미국 우편 주소를 전달합니다. 렌더링 프로세스는 1시간에서 몇 시간까지 걸릴 수 있습니다.
참고: 이 엔드포인트는 무료로 사용할 수 있습니다.
curl -X POST -d '{
  "address": "POSTAL_ADDRESS"
}' \
-H 'Content-Type: application/json' \
"https://aerialview.googleapis.com/v1/videos:renderVideo?key=YOUR_API_KEY"
요청 예시
다음 코드 예에서는 500 W 2nd St, Austin, TX, 78701의 항공 뷰 동영상 생성을 요청합니다.
curl -X POST -d '{
  "address": "500 W 2nd St, Austin, TX 78701"
}' \
-H 'Content-Type: application/json' \
"https://aerialview.googleapis.com/v1/videos:renderVideo?key=YOUR_API_KEY"
응답 예
다음 JSON 코드 샘플은 예시 동영상 요청에 대한 가능한 응답을 보여줍니다.
잘못된 주소 응답
주소가 잘못되었거나 미국 내 주소가 아닙니다.
{
  "error": {
    "code": 400,
    "message": "Address not supported.",
    "status": "INVALID_ARGUMENT"
  }
}
응답 처리
이제 Aerial View API에서 동영상을 렌더링합니다. 대답에는 동영상을 검색하는 데 사용할 수 있는 videoId가 포함됩니다.
{
  "state": "PROCESSING",
  "metadata": {
    "videoId": "x89iJAJi1wzs7DIAFQF7Va"
  }
}
렌더링 프로세스는 1시간에서 몇 시간까지 걸릴 수 있습니다. 동영상이 아직 처리 중인 경우 lookupVideo를 사용하여 동영상 상태가 ACTIVE 상태를 반환할 때까지 폴링할 수 있습니다.
폴링하려면 videoId을 전달하여 lookupVideo에 요청합니다. 렌더링에 몇 시간이 걸릴 수 있으므로 지수 백오프 [https://developers.google.com/maps/documentation/aerial-view/web-api-best-practices?hl=ko#exponential-backoff]를 사용하여 호출 간격을 늘리세요.
활성 응답
renderVideo가 ACTIVE의 state 값을 반환하면 동영상 렌더링이 완료된 것이므로 videoId로 lookupVideo를 호출하여 동영상을 가져올 수 있습니다.
{
  "state": "ACTIVE",
  "metadata": {
    "videoId": "x89iJAJi1wzs7DIAFQF7Va"
  }
}
