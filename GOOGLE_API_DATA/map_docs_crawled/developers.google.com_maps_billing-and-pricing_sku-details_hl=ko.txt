이 페이지에서는 인도의 SKU를 비롯한 Google Maps Platform SKU에 관한 세부정보를 제공합니다.

재고 관리 단위 (SKU)는 제품과 연결된 Google Maps Platform의 개별 항목이며, 제품에는 둘 이상의 SKU가 연결될 수 있습니다. 각 SKU에는 가격 목록 (가격 [https://developers.google.com/maps/billing-and-pricing/pricing?hl=ko], 가격 - 인도 [https://developers.google.com/maps/billing-and-pricing/pricing-india?hl=ko])에서 확인할 수 있는 고정 가격이 있습니다. 서비스를 사용하면 하나 이상의 SKU가 트리거될 수 있으며, 각 SKU는 청구서에 별도의 항목으로 표시됩니다.

SKU 이름

이 섹션에는 이름별로 모든 SKU가 나열되어 있으며 Essentials, Pro, Enterprise로 구분되어 있습니다. 결제 명세서에 표시되는 이름입니다.

굵은 밑줄이 표시된 SKU의 이름이 업데이트되었습니다.
마우스 커서를 가져가 이전 SKU 이름을 확인하고 Esc 키를 눌러 닫습니다.

가격 책정 카테고리에 대한 설명은 결제 개요 가이드의 Essentials, Pro, Enterprise 카테고리 [https://developers.google.com/maps/billing-and-pricing/overview?hl=ko#pricing-categories]를 참고하세요.

Places Data SKU에 대한 정보

세 Places Data SKU는 Basic Data [https://developers.google.com/maps/billing-and-pricing/sku-details?hl=ko#basic-data-legacy-ess-sku], Contact Data [https://developers.google.com/maps/billing-and-pricing/sku-details?hl=ko#contact-data-legacy-ent-sku], Atmosphere Data [https://developers.google.com/maps/billing-and-pricing/sku-details?hl=ko#atmosphere-data-legacy-ent-sku]입니다.

다음에 대해 하나 이상의 데이터 SKU가 트리거됩니다.

Android: 각 fetchPlace() 또는 findCurrentPlace() 요청
iOS: 각 fetchPlaceFromPlaceID: 또는 findPlaceLikelihoodsFromCurrentLocationWithPlaceFields: 호출
웹 서비스: 요청에 지정된 필드에 따라 각 장소 요청

데이터 SKU는 항상 데이터 SKU를 트리거하는 기본 SKU 외에 추가로 요금이 청구됩니다. 예를 들어 Basic Data 필드가 포함된 Place Details 요청에 대해서는 Basic Data SKU 및 Place Details SKU 모두에 따라 요금이 청구됩니다.

웹 서비스의 경우 다음 Places API 요청을 통해 반환할 데이터 필드를 지정할 수 있습니다.

Places Details Pro [https://developers.google.com/maps/billing-and-pricing/sku-details?hl=ko#place-details-legacy-pro-sku]
Find Place [https://developers.google.com/maps/billing-and-pricing/sku-details?hl=ko#find-place-legacy-pro-sku]

웹 서비스의 경우 다음 Places API 호출에서는 반환된 필드를 지정할 수 없습니다. 이 호출에서는 항상 모든 장소 데이터를 반환하고 각 API 요청에 대한 요금 외에 세 개의 Places Data SKU에 대해 요금을 청구합니다.

자동 완성 세션에 대한 정보

자동 완성 세션에는 사용자가 장소를 입력함과 동시에 추천 검색어를 가져오기 위한 Autocomplete 요청 및 사용자가 선택한 장소에 대한 세부정보를 가져오기 위한 최대 1개의 Place Details 요청이 포함됩니다. 동일한 세션 토큰이 Autocomplete 및 이후의 Place Detail 요청에 전달됩니다. 세션은 첫 번째 Autocomplete 요청으로 시작됩니다. 이 요청은 일반적으로 사용자가 입력을 시작할 때 발생합니다. 사용자가 Autocomplete 추천 검색어 중 하나를 선택하면 Place Details 호출이 이루어집니다. 사용자가 추천 검색어를 선택하지 않으면 Place Details 호출이 이루어지지 않습니다.

사용자가 장소를 선택한 후 (즉, Place Details 호출이 이루어진 후) 새 세션 토큰을 사용하여 새 세션을 시작해야 합니다.

자동 완성 세션은 다음과 같은 API에서 생성할 수 있습니다.

Places API Place Autocomplete 서비스 [https://developers.google.com/maps/documentation/places/web-service/autocomplete?hl=ko] 또는 Maps JavaScript API Places Autocomplete 서비스 [https://developers.google.com/maps/documentation/javascript/places-autocomplete?hl=ko]. 세션 토큰을 제공하도록 애플리케이션을 설계해야 합니다. 기존 코드를 변경해야 할 수도 있습니다.
Maps JavaScript API Autocomplete 위젯 [https://developers.google.com/maps/documentation/javascript/places-autocomplete?hl=ko#summary-of-classes] 이 위젯에서는 코드를 변경하지 않아도 세션 기반 결제가 자동으로 사용 설정됩니다.

세션 토큰은 하나의 사용자 세션에 유효하며 두 개 이상의 사용자 세션에 사용해서는 안 됩니다. 세션 토큰을 재사용하면 세션이 유효하지 않은 것으로 간주되어 세션 토큰이 제공되지 않은 것처럼 요청에 대해 요금이 청구됩니다.

세션을 사용하는 Autocomplete 요청 및 Place Details 요청에 대해서는 서로 다른 SKU에 따라 요금이 청구됩니다.
