문제 해결

지오코더에서 ZERO_RESULTS를 반환하는 쿼리가 점점 더 많아지고 있습니다. 어떤 문제인가요?

지오코더에서 잘못된 철자 또는 존재하지 않는 주소와 같이 모호하고 불완전하며 형식이 잘못된 검색어는 ZERO_RESULTS를 생성하는 경향이 있습니다.
주소를 찾을 수 없는 경우 부분 결과 (예: 주소 대신 교외만 반환)보다 ZERO_RESULTS를 사용하는 것이 좋습니다.

애플리케이션에서 사용자 입력 주소를 처리하는 경우 Places API의 Place Autocomplete [https://developers.google.com/maps/documentation/places/web-service/autocomplete?hl=ko] 기능을 사용하면 더 나은 품질의 결과를 얻을 수 있습니다.
Place Autocomplete를 사용하면 사용자가 입력한 내용을 기반으로 결과 집합에서 선택할 수 있으므로 사용자가 비슷한 이름의 결과 중에서 선택하고 주소를 잘못 썼을 경우 쿼리를 조정할 수 있습니다.

애매하거나 불완전한 쿼리 또는 오류가 포함될 수 있는 쿼리를 처리하는 애플리케이션이 있는 경우 Geocoding API에서 제공하는 전방 지오코더 대신 Places API의 장소 자동 완성 [https://developers.google.com/maps/documentation/places/web-service/autocomplete?hl=ko] 기능을 사용하는 것이 좋습니다. 자세한 내용은 주소 지오코딩 권장사항 [https://developers.google.com/maps/documentation/geocoding/best-practices?hl=ko] 및 Google Maps API 블로그 게시물의 주소 지오코딩 [https://mapsplatform.google.com/resources/blog/address-geocoding-in-google-maps-apis/?hl=ko]을 참고하세요.

빠른 응답이 필요하지만 내 애플리케이션에 사용하기에는 지오코더가 너무 느립니다. 더 빠른 응답을 위해 사용할 수 있는 도구는 무엇인가요?

사용자 입력에 응답하므로 지연 시간에 민감한 애플리케이션의 경우 주소 지오코딩 대신 Places API의 장소 자동 완성 [https://developers.google.com/maps/documentation/places/web-service/autocomplete?hl=ko] 기능 (JavaScript [https://developers.google.com/maps/documentation/javascript/places-autocomplete?hl=ko], Android [https://developers.google.com/maps/documentation/places/android-sdk/autocomplete?hl=ko] 또는 iOS [https://developers.google.com/maps/documentation/places/ios-sdk/autocomplete?hl=ko]에서도 사용 가능)을 사용하는 것이 좋습니다.
장소 자동완성은 대화형 방식으로 사용이 최적화되기 때문에, 지연 시간이 매우 짧습니다.

Geocoding API의 주소 지오코딩은 온라인 양식에 입력된 배송지 주소와 같이 완전하고 명확하며 형식이 잘 지정된 주소와 함께 사용하도록 최적화되어 있으므로 Place Autocomplete보다 지연 시간이 더 깁니다. 전방향 지오코더는 적용 범위가 넓고 결과 품질이 우수하지만 지연 시간이 다소 깁니다.

Directions API (기존) 및 Distance Matrix API (기존)의 지연 시간을 줄이려면 어떻게 해야 하나요?

주소 대신 장소 ID [https://developers.google.com/maps/documentation/places/web-service/place-id?hl=ko]를 사용하여 중간 지점, 출발지, 도착지를 지정합니다. 장소 ID는 Places API의 장소 자동 완성 [https://developers.google.com/maps/documentation/places/web-service/autocomplete?hl=ko] 기능 또는 Maps JavaScript API의 장소 라이브러리 [https://developers.google.com/maps/documentation/javascript/places-autocomplete?hl=ko]에서 가져오는 것이 가장 좋습니다. Place Autocomplete 비용을 줄이는 데 사용할 수 있는 placeIdOnly [https://developers.google.com/maps/documentation/javascript/places-autocomplete?hl=ko#placeidonly] 옵션에 관한 정보도 참고하세요.

Directions API (기존) 또는 Distance Matrix API (기존)에 장소 ID나 latlng이 아닌 주소 문자열로 쿼리하면 이러한 API는 Geocoding API와 동일한 백엔드를 사용하여 길을 계산하기 전에 해당 주소를 장소 ID로 변환합니다. Place Autocomplete는 주소 지오코딩보다 빠릅니다. 사용자 입력에 응답하는 등 지연 시간에 민감한 상황에서 Directions API (기존) 또는 Distance Matrix API (기존)를 사용하는 애플리케이션의 경우 장소 자동 완성을 사용하여 해당 주소에 해당하는 장소 ID를 가져오고 장소 ID를 Directions API (기존) 또는 Distance Matrix API (기존)에 전달하는 것이 좋습니다.
이 접근 방식은 지연 시간을 크게 줄입니다. 경로와 함께 Place Autocomplete를 사용하는 방법의 예 [https://developers.google.com/maps/documentation/javascript/examples/places-autocomplete-directions?hl=ko]는 문서를 참고하세요.

지오코더의 버그를 신고하려면 어떻게 해야 하나요?

전방향 지오코더 서비스에 관한 버그 신고 [https://issuetracker.google.com/savedsearches/558483?hl=ko] 또는 기능 요청 [https://issuetracker.google.com/issues/new?component=188871&template=788905&hl=ko]이 있는 경우 공개 Issue Tracker [https://issuetracker.google.com/issues/new?component=188871&template=788906&hl=ko]를 사용하여 알려주세요.

내 사용 사례에 맞는 좋은 결과가 지오코더에서 생성되지 않습니다. 어떤 도움을 받을 수 있나요?

공개 문제 추적기 [https://issuetracker.google.com/savedsearches/558483?hl=ko]를 사용하여 몇 가지 구체적인 검색어를 공유해 주세요. 그러면 결과 품질에 문제를 일으키는 버그나 시스템 문제가 있는지 또는 개발자가 더 나은 결과를 얻는 데 도움이 되도록 권장사항 가이드 [https://developers.google.com/maps/documentation/geocoding/best-practices?hl=ko]를 변경할 수 있는지 조사할 수 있습니다.

구성요소 필터를 사용하여 지오코더로 최상의 결과를 얻으려면 어떻게 해야 하나요?

지오코더에서 구성요소 필터링 [https://developers.google.com/maps/documentation/geocoding/overview?hl=ko#component-filtering]은 postal_code 및 country 제한만 적용합니다. 다음 예는 필요한 결과에 따라 구성요소 필터링을 가장 잘 사용하는 방법을 보여줍니다. 샘플 요청 URL에서 `YOUR_API_KEY` 를 실제 API 키로 대체해야 합니다.

우편번호를 특정 국가로 제한하려면 ISO 3166-2 국가 코드 [https://en.wikipedia.org/wiki/ISO_3166-2]를 사용하여 국가 제한을 지정해야 합니다.
예를 들어 components=country:CH|postal_code:8000는 '8000 Zürich, Switzerland'를 반환합니다.
https://maps.googleapis.com/maps/api/geocode/json?components=country:CH%7Cpostal_code:8000&key=YOUR_API_KEY

검색어를 특정 국가로 제한하려면 ISO 3166-2 국가 코드 [https://en.wikipedia.org/wiki/ISO_3166-2]를 사용하여 국가 제한을 지정해야 합니다.  국가 이름 및 기타 약어는 국가 코드와 동일한 결과를 가져오지 않을 수 있습니다.
예:

components=country:FRA|locality:gallus의 지오코딩은 독일 프랑크푸르트 암 마인의 Gallus 하위 지역을 반환합니다. 이는 프랑스의 3자리 국가 코드 (FRA)가 지원되지 않으므로 모든 국가의 결과가 허용되고 프랑스보다 프랑크푸르트의 결과가 더 적합하기 때문입니다.
https://maps.googleapis.com/maps/api/geocode/json?components=country:FRA%7Clocality:gallus&key=YOUR_API_KEY

Geocoding API는 모호한 쿼리에 대해 반드시 여러 결과를 반환하지는 않습니다. 이러한 검색어는 Place Autocomplete로 더 효과적으로 답변할 수 있습니다.
예:

components=country:US|locality:madrid의 지오코딩은 'Madrid, NM 87010, USA'를 반환하지만 미국의 다른 도시 중 'Madrid'라는 이름을 가진 도시는 없습니다.
https://maps.googleapis.com/maps/api/geocode/json?components=country:US%7Clocality:madrid&key=YOUR_API_KEY

input=madrid&components=country:us&types=(regions)에 Place Autocomplete를 사용하고 location 및 radius를 사용하여 표시 영역을 지정하여 위치 결과에 편향 [https://developers.google.com/maps/documentation/places/web-service/autocomplete?hl=ko#location_biasing]을 적용하면 미국에 있는 '마드리드'라는 이름의 여러 도시가 포함된 응답이 생성됩니다.
https://maps.googleapis.com/maps/api/place/autocomplete/json?location=37.386052,-122.083851&radius=10000&input=madrid&components=country:us&types=(regions)&key=YOUR_API_KEY
