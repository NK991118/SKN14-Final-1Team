Cloud Firestore 라이브러리 및 프레임워크 통합

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

Cloud Firestore는 본 문서에서 다룬 클라이언트 및 서버 라이브러리 외에도 다양한 오픈소스 라이브러리와의 통합이 가능합니다.
이 같은 통합은 주로 Cloud Firestore를 사용하는 개발자가 자주 이용하는 프레임워크로 이를 가져오는 과정에서 이루어집니다.

Cloud Firestore팀과의 협력을 통해 구현된 통합은 다음과 같습니다.

AngularFire

플랫폼: 웹

AngularFire [https://github.com/angular/angularfire2]는 Firebase용 Angular 라이브러리입니다. AngularFire 및 Cloud Firestore를 사용하면 데이터를 문서 및 컬렉션에 추가하고 데이터를 쿼리하며 오프라인 데이터 액세스를 활성화할 수 있습니다.

문서 읽기 [https://github.com/angular/angularfire2/tree/master/docs/firestore]

FirebaseUI

플랫폼: Android, iOS

Android [https://github.com/firebase/FirebaseUI-Android/] 및 iOS [https://github.com/firebase/FirebaseUI-iOS/]용 FirebaseUI 라이브러리를 사용하면 Cloud Firestore 데이터베이스의 데이터를 앱의 UI에 바인딩할 수 있습니다.

iOS 문서 읽기 [https://github.com/firebase/FirebaseUI-iOS/blob/master/README.md]
Android 문서 읽기 [https://github.com/firebase/FirebaseUI-android/blob/master/README.md]

React Native Firebase

플랫폼: Android, Apple

React Native Firebase [https://rnfirebase.io/]는 공식 지원되는 Firebase용 React Native 라이브러리입니다. 단, Firebase 지원이 제공되지 않습니다 React Native에서 Cloud Firestore 시작하기 [https://rnfirebase.io/firestore/usage]에서 React Native 및 Cloud Firestore로 가능한 작업에 대해 자세히 알아보세요.

문서 읽기 [https://rnfirebase.io/docs]

찾으시는 프레임워크가 없나요?
Cloud Firestore를 자주 사용하는 프레임워크로 가져오는 통합 구현에 관심이 있다면 Slack 채널 [https://firebase.community/] 또는 StackOverflow [https://stackoverflow.com/questions/tagged/firebase]에 문의하세요.
