Firestore 감사 로깅 정보

bookmark_borderbookmark

컬렉션을 사용해 정리하기

내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요.

이 문서에서는 Firestore의 감사 로깅을 설명합니다. Google Cloud 서비스는 Google Cloud 리소스 내의 관리 및 액세스 활동을 기록하는 감사 로그를 생성합니다.
Cloud 감사 로그에 대한 자세한 내용은 다음을 참조하세요.

감사 로그 유형 [https://cloud.google.com/logging/docs/audit?hl=ko#types]
감사 로그 항목 구조 [https://cloud.google.com/logging/docs/audit?hl=ko#audit_log_entry_structure]
감사 로그 저장 및 라우팅 [https://cloud.google.com/logging/docs/audit?hl=ko#storing_and_routing_audit_logs]
Cloud Logging 가격 책정 요약 [https://cloud.google.com/stackdriver/pricing?hl=ko#logs-pricing-summary]
데이터 액세스 감사 로그 사용 설정 [https://cloud.google.com/logging/docs/audit/configure-data-access?hl=ko]

참고
감사 로깅을 구성할 때 서비스 이름 datastore.googleapis.com을 사용하여 datastore.googleapis.com 및 firestore.googleapis.com를 모두 구성합니다.구성되면 Firestore API의 로그에 서비스 이름 firestore.googleapis.com가 포함됩니다.DATA_READ 또는 DATA_WRITE 요청을 처리하는 데 걸린 시간을 보려면 AuditLog의 metadata 객체 내 processing_duration 필드를 참고하세요.
processing_duration은 데이터베이스가 요청을 처리하는 데 걸린 시간을 설명합니다. 이 값은 최종 사용자 지연 시간보다 작습니다. 특히 네트워크 오버헤드를 포함하지 않습니다.Listen [https://cloud.google.com/firestore/docs/reference/rpc/google.firestore.v1?hl=ko#google.firestore.v1.Firestore.Listen] 요청의 경우 processing_duration는 반환된 초기 결과 집합의 감사 로그에만 있습니다. 동일한 Listen 대상에 대한 후속 감사 로그에는 없습니다.가져오기, 일괄 삭제 작업, TTL의 개별 쓰기는 감사 로그에 기록되지 않습니다.
서비스 이름

Firestore 감사 로그는 firestore.googleapis.com이라는 서비스 이름을 사용합니다.
이 서비스에 대한 필터: 더 많은 코드 작업을 살펴봅니다.

밝은 코드 테마

어두운 코드 테마

protoPayload.serviceName="firestore.googleapis.com"

권한 유형별 메서드

Firestore에는 Key Visualizer [https://cloud.google.com/firestore/docs/key-visualizer?hl=ko] 진단 도구의 일부로 다음 작업도 포함됩니다. 데이터 액세스 [https://cloud.google.com/logging/docs/audit?hl=ko#data-access] 감사 로그이며 서비스 이름은 firestorekeyvisualizer.googleapis.com입니다. firestore.googleapis.com 서비스에 DATA_READ를 사용 설정하면 사용 설정됩니다.google.cloud.keyvisualizer.KeyVisualizer.GetScangoogle.cloud.keyvisualizer.KeyVisualizer.ListScans
각 IAM 권한에는 type 속성이 포함되며 그 값은 네 가지 값(ADMIN_READ, ADMIN_WRITE, DATA_READ, DATA_WRITE) 중 하나일 수 있는 열거형입니다. 메서드를 호출하면 Firestore에서 메서드를 수행하는 데 필요한 권한의 type 속성에 따라 카테고리가 달라지는 감사 로그를 생성합니다.

DATA_READ, DATA_WRITE, ADMIN_READ의 type 속성 값을 가진 IAM 권한이 필요한 메서드는 데이터 액세스 [https://cloud.google.com/logging/docs/audit?hl=ko#data-access] 감사 로그를 생성합니다.

type 속성 값이 ADMIN_WRITE인 IAM 권한이 필요한 메서드는 관리자 활동 [https://cloud.google.com/logging/docs/audit?hl=ko#admin-activity] 감사 로그를 생성합니다.

이 메서드에 대한 필터::
protoPayload.methodName="google.firestore.v1.Firestore.Listen"

참고:  Listen은 여러 스트리밍 대상을 결합하는 장기 실행 RPC입니다. 각 대상은 문서 키의 쿼리 또는 집합입니다. 각 대상의 스트림에는 초기 결과 집합과 결과 집합에 대한 업데이트, 추가, 삭제 시퀀스가 포함됩니다. 대상은 관련 감사 단위입니다. Firestore는 각 대상을 다음과 같이 감사합니다. 대상이 추가되면 대상 쿼리 또는 문서 키가 설정된 로그 항목을 내보냅니다. 이러한 항목에서 operation.first [https://cloud.google.com/logging/docs/reference/v2/rest/v2/LogEntry?hl=ko#logentryoperation]는 true입니다. 스트림이 이전 리슨 대상 스트림에서 계속되는 경우 감사 로그가 생략됩니다. 이 대상의 마지막 감사 로그 이후 업데이트 수를 보고하는 주기적 업데이트를 내보냅니다. 명시적으로 또는 Listen RPC 종료로 인해 대상이 스트림에서 삭제될 때 로그 항목을 내보냅니다. 이 로그 항목은 이 대상의 마지막 감사 로그 이후의 업데이트 수를 보고합니다. 이러한 항목에서 operation.last [https://cloud.google.com/logging/docs/reference/v2/rest/v2/LogEntry?hl=ko#logentryoperation]는 true입니다.  내보낸 로그 항목에는 동일한 operation.id [https://cloud.google.com/logging/docs/reference/v2/rest/v2/LogEntry?hl=ko#logentryoperation]가 사용됩니다.

요청 호출자 식별

감사 로그 항목에는 로깅 작업을 수행한 ID에 대한 정보가 포함됩니다. 요청 호출자를 식별하려면 AuditLog [https://cloud.google.com/logging/docs/reference/audit/auditlog/rest/Shared.Types/AuditLog?hl=ko] 객체 내에서 다음 필드를 참조하세요.

호출자 ID는 AuthenticationInfo [https://cloud.google.com/logging/docs/reference/audit/auditlog/rest/Shared.Types/AuditLog?hl=ko#AuthenticationInfo] 필드에 보관됩니다. 여기에는 사용자의 principalEmail이 포함될 수 있습니다. 이 정보는 가끔 수정 [https://cloud.google.com/logging/docs/audit?hl=ko#user-id]됩니다.

JSON 웹 토큰(JWT)이 타사 인증에 사용된 경우 thirdPartyPrincipal 필드에는 토큰의 헤더와 페이로드가 포함됩니다.
예를 들어 Firebase 인증 [https://firebase.google.com/docs/auth?hl=ko]으로 인증된 요청의 감사 로그에는 해당 요청의 인증 토큰 [https://firebase.google.com/docs/auth/users?hl=ko#auth_tokens]이 포함됩니다.
AuditLog 항목의 requestMetadata [https://cloud.google.com/logging/docs/reference/audit/auditlog/rest/Shared.Types/AuditLog?hl=ko#requestmetadata] 객체 내에 있는 callerIp 필드에는 호출자의 IP 주소가 포함됩니다.
