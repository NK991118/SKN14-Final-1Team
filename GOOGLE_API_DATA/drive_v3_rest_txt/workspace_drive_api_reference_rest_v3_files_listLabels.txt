파일의 라벨을 나열합니다.
HTTP 요청
GET https://www.googleapis.com/drive/v3/files/{fileId}/listLabels
URL은 gRPC 트랜스코딩 [https://google.aip.dev/127] 문법을 사용합니다.
경로 매개변수
매개변수
fileId
string
파일의 ID입니다.
쿼리 매개변수
매개변수
maxResults
integer
페이지당 반환할 라벨의 최대 수입니다. 설정하지 않으면 기본값은 100입니다.
pageToken
string
다음 페이지에서 이전 목록 요청을 계속하는 토큰입니다. 이 값은 이전 응답의 'nextPageToken' 값으로 설정해야 합니다.
요청 본문
요청 본문은 비어 있어야 합니다.
응답 본문
파일에 적용된 라벨 목록입니다.
성공한 경우 응답 본문은 다음과 같은 구조의 데이터를 포함합니다.
JSON 표현
{
  "labels": [
    {
      object (Label [https://developers.google.com/workspace/drive/api/reference/rest/v3/Label?hl=ko])
    }
  ],
  "nextPageToken": string,
  "kind": string
}
필드
labels[]
object (Label [https://developers.google.com/workspace/drive/api/reference/rest/v3/Label?hl=ko])
라벨 목록입니다.
nextPageToken
string
라벨의 다음 페이지에 대한 페이지 토큰입니다. 목록의 끝에 도달한 경우 이 필드는 없습니다. 어떤 이유로든 토큰이 거부되면 토큰을 삭제하고 결과의 첫 번째 페이지에서 페이지로 나누기를 다시 시작해야 합니다. 페이지 토큰은 일반적으로 몇 시간 동안 유효합니다. 하지만 새 항목이 추가되거나 삭제되면 예상 결과가 달라질 수 있습니다.
kind
string
항상 drive#labelList입니다.
승인 범위
다음 OAuth 범위 중 하나가 필요합니다.
https://www.googleapis.com/auth/drive
https://www.googleapis.com/auth/drive.file
https://www.googleapis.com/auth/drive.meet.readonly
https://www.googleapis.com/auth/drive.metadata
https://www.googleapis.com/auth/drive.metadata.readonly
https://www.googleapis.com/auth/drive.readonly
일부 범위는 제한되어 있으며 앱에서 이를 사용하려면 보안 평가가 필요합니다. 자세한 내용은 승인 가이드 [https://developers.google.com/workspace/guides/configure-oauth-consent?hl=ko]를 참고하세요.
